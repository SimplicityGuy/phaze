## Cross-Phase Integration

Integration checker traced the full pipeline (image → compute agent → routing → push → analyze → report → toggle). All 6 cross-phase seams CONNECTED at the code level; toggle-OFF reverts cleanly to all-local with no un-gated compute path. The checker initially returned GAPS_FOUND on three findings — **all now fixed and merged in PR #161 (commit f301545):**

1. **BLOCKER (resolved):** `docker-compose.cloud-agent.yml` used `command: uv run saq …`, which overrides the Dockerfile CMD; the arm64 (py3.13/--system) image cannot use the `uv` launcher, so the container would never start. Fixed to `python3 -m saq …` + regression test.
2. **W1 (resolved):** `cloud_burst_enabled=true` with no `compute_scratch_dir` silently dead-ended files in ANALYSIS_FAILED. Now a fail-fast `ControlSettings` validator + tests.
3. **W2 (resolved):** control/agent scratch-dir skew surfaced as a silent re-push loop. Now a clear runtime diagnostic naming the path + likely config mismatch.
