# Quick Task 260706-q90: Optional models PVC mount for the Kueue Job pod

## Goal

Add an optional, backward-compatible per-Kueue-backend field `models_pvc_name` so
`build_job_manifest` mounts an operator-provisioned, ReadOnlyMany PVC read-only at the
models dir (`/models`) — a SECOND volume, entirely separate from the existing `phaze-ca`
Secret mount at `/certs`. Unset ⇒ byte-identical manifest (regression-guarded). phaze
authors no PV/PVC and never bakes/downloads weights.

## Constraints (HARD)

- No cert/key/kubeconfig/SA-token material committed anywhere (placeholders only).
- No cert/CA/key baked into any image; KDEPLOY-06 preserved (`/certs` CA mount unchanged).
- The new PVC carries ONLY essentia weights — never secrets, never certs.
- `/models` mountPath MUST equal the agent-env ConfigMap's `PHAZE_MODELS_DIR` (documented invariant).

## Tasks (atomic commits)

1. **Config field** — `src/phaze/config_backends.py` `KubeConfig`: add
   `models_pvc_name: str | None = None` (plain K8s object name, no SecretStr, no `*_file`).
   Test: `tests/analyze/services/test_backends.py` — `[backends.kube]` with `models_pvc_name`
   parses/round-trips through TOML; default is None when omitted.

2. **Manifest mount** — `src/phaze/services/kube_staging.py` `build_job_manifest`: when
   `kube.models_pvc_name` is truthy, append a `models` PVC volume
   (`persistentVolumeClaim.claimName` + `readOnly:True`) and a `/models` readOnly volumeMount
   on the `analyze` container; leave `phaze-ca`/`/certs` exactly as-is. Document the
   `/models == PHAZE_MODELS_DIR` invariant in the docstring.
   Tests: `tests/analyze/services/test_kube_staging.py` — (a) set ⇒ PVC volume+mount present;
   (b) unset ⇒ ONLY the `phaze-ca` volume/mount (regression guard).

3. **Dockerfile comment** — `Dockerfile.job`: note `/models` is supplied by an
   operator-provisioned PVC (or a pre-baked image), never committed to the repo; stays weights-free.

4. **Docs** — `docs/configuration.md` `[backends.kube]` field reference: add a `models_pvc_name`
   row. `docs/k8s-burst.md`: add a "Models provisioning" subsection (operator pre-populates a
   ReadOnlyMany PVC with essentia weights, then sets `models_pvc_name`; pod mounts it read-only;
   phaze authors no PV/PVC — references the claim by name only).

5. **Release** — bump CalVer `2026.7.2` → `2026.7.3` in `pyproject.toml` + `uv.lock`. Publish
   (api + job + sidecars) happens on pushing the bare CalVer tag `2026.7.3` after the PR merges to
   main (CI `tags: ["[0-9]+.[0-9]+.[0-9]+"]` → docker-publish).
