<placement_decision>
The current Dockerfile order is: `FROM` → `WORKDIR /app` → `COPY uv` → `COPY pyproject/uv.lock` + `uv sync` (deps) → `COPY src/alembic` → `uv sync` (project) → user creation → `USER phaze` (line 27).

Insert the apt layer immediately after `WORKDIR /app` (current line 3), BEFORE the `COPY --from=...uv` line. Rationale:
- MUST be before `USER phaze` (line 27) because apt needs root — satisfied with large margin.
- Placing it early (before the uv COPY/sync layers) means the system-deps layer is invalidated only by changes to that RUN line itself, not by source or dependency changes. It caches independently and stays warm across the vast majority of rebuilds (which change `src/` or `uv.lock`).
- Keeps system setup grouped at the top of the build, matching common Docker layering convention (OS deps → tooling → app deps → app code).
</placement_decision>
