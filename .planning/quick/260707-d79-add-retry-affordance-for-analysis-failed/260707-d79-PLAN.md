<objective>
Add an operator-triggered BULK retry for files stuck in `FileState.ANALYSIS_FAILED`
("local — failed"). Today this is a dead end: no UI button, and `recover_orphaned_work`
classifies ANALYSIS_FAILED as a *done* state (`_select_done_analyze_ids`, reenqueue.py),
so even RECOVER skips it. ~428 files are stranded.

Add a new `POST /pipeline/analysis-failed/retry` that re-enqueues `process_file` for every
ANALYSIS_FAILED file through the SAME guarded funnel `deepen_analysis` uses (per-agent
routing → NoActiveAgentError guard → `enqueue_process_file` full-payload+deterministic-key),
but with NORMAL caps (a retry is a fresh re-analysis, not a deepen). Each retried file flips
ANALYSIS_FAILED → FINGERPRINTED (committed before enqueue) so it leaves the red bucket.
Auto-recovery is LEFT UNCHANGED — the manual endpoint is its deliberate, operator-gated
counterpart.

Purpose: give the operator a way to re-run genuinely-retryable failed files without
re-looping un-analyzable ones automatically.
Output: new endpoint + ack fragment, "Retry failed" button on the Analysis Health card,
route tests, and a code comment on `_select_done_analyze_ids` documenting the distinction.
</objective>

src/phaze/routers/pipeline.py, modeled on `deepen_analysis` and `trigger_backfill_cloud`.

Order (matches the Phase-30/Pitfall-3 guards):
1. `files = await get_analysis_failed_files(session)` (already imported from services.pipeline;
   if not, add to the existing `from phaze.services.pipeline import (...)` block). If empty →
   render `retry_failed_response.html` with `count=0`, no enqueue.
2. Resolve the queue ONCE: `routed = await enqueue_router.resolve_queue_for_task("process_file",
   request.app.state, session)`. Catch `enqueue_router.NoActiveAgentError` → render ack with
   `no_active_agent=True`, `count=0`; DO NOT flip state, DO NOT enqueue, DO NOT fall through to
   the default queue (Phase-30 guard). `agent_id = cast("str", routed.agent_id)`.
3. Flip every file: `for f in files: f.state = FileState.FINGERPRINTED` then `await session.commit()`
   BEFORE any enqueue (get_session does NOT auto-commit; Pitfall 3). Choosing FINGERPRINTED — the
   canonical pre-analysis state — makes the file leave the ANALYSIS_FAILED bucket immediately;
   on success `put_analysis` flips it → ANALYZED, on re-failure `report_analysis_failed` →
   ANALYSIS_FAILED, so the terminal bucket is re-entered only if it fails again.
4. Loop `await enqueue_process_file(routed.queue, f, agent_id, settings.models_path)` — NO
   fine_cap/coarse_cap (normal 60/30 caps; a retry is a fresh re-analysis, NOT a deepen). The
   deterministic key dedups any file with a live in-flight job to a no-op. Re-enqueue ALL failed
   (dedup-safe; do NOT silently cap).
5. `logger.info("retry_analysis_failed re-queued files", count=len(files))` for observability,
   then render `retry_failed_response.html` with `count=len(files)`.
