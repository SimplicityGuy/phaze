<objective>
Fix the `trigger_scan` dead-letter bug. `POST /tracklists/scan`
(`src/phaze/routers/tracklists.py::trigger_scan`, line 232) currently enqueues
`scan_live_set` jobs with only `file_id`. `ScanLiveSetPayload` is
`extra="forbid"` and requires `file_id` + `original_path` + `agent_id`, so every
job dead-letters — the v4.0.8 payload-incident class. Phase 40 already fixed the
BULK path in `pipeline.py::_enqueue_scan_jobs`; this per-tracklist route was left
as a follow-up. Build the COMPLETE `ScanLiveSetPayload` per enqueue, mirroring the
Phase-40 producer.

Purpose: Make the Tracklists "Scan" tab actually produce executable jobs instead
of silently dead-lettering every one.
Output: A corrected `trigger_scan` handler plus regression tests asserting the
enqueued payload validates against `ScanLiveSetPayload`.
</objective>

async def _enqueue_scan_jobs(queue: Any, files: list[FileRecord], agent_id: str) -> None:

        payload = ScanLiveSetPayload(file_id=f.id, original_path=f.original_path, agent_id=agent_id)
