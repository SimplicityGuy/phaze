# Phase quick-260609-f96 Plan 01: Fix scan_directory 10s TimeoutError Summary

Per-agent SAQ queues built by `AgentTaskRouter._queue_for` now register the
`apply_project_job_defaults` `before_enqueue` hook, so agent-dispatched jobs
(notably `scan_directory`) inherit the project's 600s `worker_job_timeout`
instead of SAQ 0.26.3's 10s default and are no longer cancelled with
`asyncio.TimeoutError` after exactly 10 seconds.

## What Changed

### Task 1 — Register the hook on per-agent queues (commit c6c7e20)

`src/phaze/services/agent_task_router.py`:
- Added a first-party import of `apply_project_job_defaults` from
  `phaze.tasks._shared.queue_defaults` (placed after the third-party imports,
  respecting isort's `force-sort-within-sections` + `lines-after-imports = 2`).
- Restructured `_queue_for` so the queue is built into a local, the hook is
  registered via `queue.register_before_enqueue(apply_project_job_defaults)`,
  then cached — guaranteeing the hook is registered exactly once per `agent_id`
  and never re-registered on cache hits.
- Mirrored the established Phase 27 UAT Gap 1 comment from `controller.py` /
  `agent_worker.py` and added a `quick-260609-f96` note documenting the live
  diagnosis (agent-dispatched `scan_directory` jobs inheriting the 10s default).

This is the third call site for the hook; the controller queue and the agent
worker's own queue already registered it, but the API/control-plane per-agent
dispatch path did not.

### Task 2 — Regression test (commit db8cbce)

`tests/test_services/test_agent_task_router.py`:
- Added `test_queue_registers_job_defaults_hook`, mirroring the existing
  `@pytest.mark.integration` tests. It builds a queue via
  `router._queue_for("agent-timeout-test")` and asserts
  `apply_project_job_defaults in queue._before_enqueues.values()` (SAQ 0.26.3
  stores before_enqueue callbacks in the `_before_enqueues` dict keyed by
  `id(callback)`).
- Added a docstring explaining it guards against the 10s `scan_directory`
  `TimeoutError` regression.
- Extended the module docstring's "Assertions" list with the new item (#5).
