<objective>
Add a "scanned / deduped / unique" reconciliation to the Phaze pipeline admin UI so the Discovery
stage count (rows in `files`) visibly reconciles with the agent scan total
(`scan_batches.total_files`). The gap is files the agent walked that upserted onto an existing row
under the NFC-normalized composite unique key `(agent_id, original_path)` — duplicates/normalization
collisions, NOT lost work. Today the two numbers sit in different UI cards with no explanation and
read as a bug.

Purpose: Turn an apparent bug into a self-explaining reconciliation, so the operator trusts the
pipeline counts.
Output: A degrade-safe data layer in `services/pipeline.py`, router wiring, and two template
annotations (Discovery DAG node + Recent Scans FILES cell), fully tested at ≥85% coverage.
</objective>
