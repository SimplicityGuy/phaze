<objective>
Metadata extraction currently never advances FileRecord.state. The PUT
/api/internal/agent/metadata/{file_id} handler (put_metadata) upserts the
FileMetadata row and clears the scheduling ledger but never transitions the
file, so files stay DISCOVERED forever: the pipeline UI
(metadata_workspace.html renders the State column straight from f.state) shows
every row as "discovered", and the fingerprint stage — which gates on
state == METADATA_EXTRACTED via get_files_by_state — is permanently blocked.

Fix: add a guarded state advance (DISCOVERED -> METADATA_EXTRACTED) in the SAME
transaction as the FileMetadata upsert, mirroring the sibling analyze handler
(agent_analysis.py:235) and the push guard pattern (agent_push.py:126).

Purpose: Unblock the fingerprint stage and make the pipeline UI reflect reality.
Output: Guarded state advance in put_metadata + regression/guard tests.
</objective>
