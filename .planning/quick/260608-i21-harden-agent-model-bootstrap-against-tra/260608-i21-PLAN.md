<objective>
Harden the essentia model bootstrap against transient network failures so the agent
worker no longer "grinds forward" across a dozen Docker restarts on a fresh /models
volume. Add per-file retry with bounded exponential backoff + jitter, explicit
connect/read timeouts, and stronger atomic/idempotent writes inside `_download_one`.
Improve the bootstrap error message to clearly name the failing file. Only fail after
exhausting retries.

Purpose: A single TLS handshake drop mid-transfer currently propagates out of
`_download_one` -> `download_to` -> `ensure_models_present`, killing the SAQ worker and
inflating RestartCount/alerts. Retrying transient errors in place keeps the download
resilient within a single worker lifetime.
Output: Hardened `_download_one`, a clearer bootstrap message, and tests proving the
retry/atomicity/backoff behaviors with zero real network I/O and zero real sleeps.
</objective>

  <name>Task 1: Add retry, backoff, timeouts, and atomic writes to _download_one</name>
  <files>src/phaze/scripts/download_models.py</files>
  <behavior>
    - A transient error (httpx.TransportError, ssl.SSLError, socket.timeout, TimeoutError, urllib.error.URLError, ConnectionError) on attempts 1..N-1 followed by success on a later attempt → file downloads successfully, os.replace promotes it, no .part remains.
    - A 5xx HTTP status is treated as retryable (via the local _RetryableDownloadError sentinel) and retried; a later 200 succeeds. A 4xx status fails fast immediately — response.raise_for_status() lets httpx.HTTPStatusError propagate UNCAUGHT (it is NOT in the caught tuple), so no retry occurs.
    - All attempts failing transiently → raises an error after exactly _MAX_ATTEMPTS attempts; the message names the dest file and attempt count.
    - A truncated transfer (bytes written != Content-Length when present) raises _RetryableDownloadError and is retried as an incomplete read.
    - Each retry logs at WARNING naming the dest file and "attempt K/_MAX_ATTEMPTS".
    - Backoff between attempts uses time.sleep with bounded exponential delay + random jitter.
    - dest.exists() still short-circuits before any network/sleep.
    - A failed attempt removes its .part file before the next attempt.
  </behavior>
  <action>
    Add module imports: `os`, `random`, `socket`, `ssl`, `time`, `urllib.error`, and `logging` (module logger `logger = logging.getLogger(__name__)`). Define module constants near the top: `_MAX_ATTEMPTS = 5`, `_BACKOFF_BASE_SECONDS = 1.0`, `_BACKOFF_MAX_SECONDS = 30.0`, `_JITTER_SECONDS = 1.0`, and an explicit `_TIMEOUT = httpx.Timeout(connect=15.0, read=60.0, write=60.0, pool=15.0)`. Define a transient-error tuple constant `_TRANSIENT_ERRORS = (httpx.TransportError, ssl.SSLError, socket.timeout, TimeoutError, urllib.error.URLError, ConnectionError)`. Define a module-level local sentinel exception class `_RetryableDownloadError(Exception)` (a one-line class, no fenced code in comments) used to signal application-level retryable conditions that are NOT real transport exceptions.

    Rewrite `_download_one(url, dest)`: keep the `if dest.exists(): return` fast-path and `dest.parent.mkdir(parents=True, exist_ok=True)` first. Loop `for attempt in range(1, _MAX_ATTEMPTS + 1)`. Inside a try, open `httpx.stream("GET", url, follow_redirects=True, timeout=_TIMEOUT)`; on the response distinguish status BEFORE streaming: if `400 <= status < 500` call `response.raise_for_status()` — this raises httpx.HTTPStatusError which (because it inherits from HTTPError, NOT TransportError, and is therefore NOT in _TRANSIENT_ERRORS and NOT _RetryableDownloadError) propagates UNCAUGHT straight out of the function, giving fail-fast with zero retries. If `status >= 500`, `raise _RetryableDownloadError(f"server error {status} for {dest.name}")` so it is caught by the retry handler. Stream bytes to the `<dest>.part` tmp file, tracking total bytes written. After the stream completes, read `Content-Length` from `response.headers` if present and compare to bytes written; on mismatch `raise _RetryableDownloadError(...)`. On full success, `os.replace(tmp, dest)` and `return`.

    Use a single except clause `except (*_TRANSIENT_ERRORS, _RetryableDownloadError) as exc:` (httpx.HTTPStatusError is deliberately absent from this tuple so 4xx stays fail-fast). In the handler: remove the lingering `.part` file (`tmp.unlink(missing_ok=True)`), and if `attempt < _MAX_ATTEMPTS` log a WARNING naming `dest.name` and `f"attempt {attempt}/{_MAX_ATTEMPTS}"`, then sleep `min(_BACKOFF_MAX_SECONDS, _BACKOFF_BASE_SECONDS * 2 ** (attempt - 1)) + random.uniform(0, _JITTER_SECONDS)` via `time.sleep`, and continue. If `attempt == _MAX_ATTEMPTS`, raise a RuntimeError whose message names the file and attempt count, e.g. `f"Failed to download {dest.name} after {_MAX_ATTEMPTS} attempts: {exc}"`, chained `from exc`.

    Do NOT use fenced code in any comment. Do NOT reorder CLASSIFIER_MODELS/GENRE_MODELS. Keep `download_to` unchanged. Update the module docstring's atomicity note to mention `os.replace` and retry.
  </action>
