# Quick Task 260608-i21: Harden Agent Model Bootstrap Against Transient Network Failures Summary

Per-file bounded retry with exponential backoff + jitter, explicit connect/read timeouts, and atomic `os.replace` promotion added to `_download_one`, so a single TLS/handshake/read drop or a 5xx no longer kills the SAQ worker on a fresh `/models` volume; only a genuinely unreachable host fails, and it does so with a per-file named error after exhausting `_MAX_ATTEMPTS` attempts.

## What Was Built

### Task 1 — `_download_one` resilience (`src/phaze/scripts/download_models.py`)
- Bounded retry loop (`_MAX_ATTEMPTS = 5`) with exponential backoff (`_BACKOFF_BASE_SECONDS`, capped at `_BACKOFF_MAX_SECONDS`) plus random jitter (`_JITTER_SECONDS`), all sleeping through `time.sleep` so tests run with zero real delay.
- Explicit `httpx.Timeout(connect=15.0, read=60.0, write=60.0, pool=15.0)` so a stalled transfer surfaces as a retryable `TransportError` rather than hanging the worker.
- Transient transport errors retried via `_TRANSIENT_ERRORS = (httpx.TransportError, ssl.SSLError, socket.timeout, TimeoutError, urllib.error.URLError, ConnectionError)`.
- 5xx responses raise the new module-local `_RetryableDownloadError` sentinel → retried; a later 200 succeeds.
- 4xx fails fast: `response.raise_for_status()` raises `httpx.HTTPStatusError`, which is deliberately NOT in the caught tuple, so it propagates uncaught (exactly one network call).
- `Content-Length` truncation check: bytes written vs declared length mismatch raises `_RetryableDownloadError` (incomplete read) and is retried.
- Atomic promotion: stream to `<dest>.part`, then `os.replace(tmp, dest)` only on full success; a failed attempt `unlink(missing_ok=True)`s the `.part` before retrying.
- `dest.exists()` fast-path still short-circuits before any network or sleep.
- Each retry logs at WARNING naming `dest.name` and `attempt K/_MAX_ATTEMPTS`; final exhaustion raises a `RuntimeError` naming the file + attempt count, chained `from exc`.

### Task 2 — bootstrap message clarity + tests
- `src/phaze/tasks/_shared/model_bootstrap.py`: docstring now documents that the wrapped cause names the specific file + attempt count, and that transient/5xx conditions are retried in place and no longer reach the wrap. Message wrap (`f"Model download failed: {exc}"`) unchanged — IMPORT-BOUNDARY and `RuntimeError` cause-chaining preserved.
- `tests/test_scripts/test_download_models.py`: added
  - `test_download_one_retries_transient_then_succeeds` (ConnectError → ReadError → 200; sleeps == 2)
  - `test_download_one_retries_5xx_then_succeeds` (503 → 200; sleeps == 1, proves 5xx retried not failed fast)
  - `test_download_one_raises_after_exhausting_attempts` (call_count == `_MAX_ATTEMPTS`, RuntimeError names file, no dest/.part)
  - `test_download_one_failed_attempt_leaves_no_truncated_dest` (atomicity)
  - `test_download_one_retries_truncated_read_then_succeeds` (Content-Length mismatch retried → success)
  - tightened `test_download_one_4xx_raises_and_no_dest_written` to assert `route.call_count == 1` (fail-fast).
  - `time.sleep` monkeypatched to a no-op counter; `respx` for zero network I/O.
