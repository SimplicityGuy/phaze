# Milestone v6.0 — Kubernetes Burst Analysis — Audit

**Status: ⚠ gaps_found** — one critical cross-phase integration defect breaks the live K8s end-to-end flow.

## Summary

All 5 phases (52–56) passed individual VERIFICATION, are Nyquist-compliant, and are
security-verified. All 26 requirements (KJOB/KSTAGE/KSUBMIT/KROUTE/KDEPLOY) are coded and
unit-verified against fakes (respx / moto / fake-kube), with KROUTE-06 and KDEPLOY-06 delivered
as bonuses. The full test suite is green (2474 passed).

**However**, the cross-phase integration check found that the Kueue Job manifest produced by
Phase 54 does not supply the runtime environment that the Phase 52 pod entrypoint consumes. The
two phases were each verified in isolation but never integration-tested across the
manifest→pod seam. On a live cluster the deliverable does not function.

## Critical Gap — JOB-ENV-CONTRACT

`build_job_manifest` (`src/phaze/services/kube_staging.py:181-183`) injects only
`PHAZE_AGENT_CA_FILE` into the analyze container. The pod entrypoint
(`src/phaze/job_runner.py`) requires, at minimum:

1. **`PHAZE_JOB_FILE_ID`** — per-Job, unique; cannot come from a static Secret/ConfigMap, so it
   **must be code-injected** by `build_job_manifest`. Today it is read at `job_runner.py:159-162`
   and, when absent, the pod logs `job_runner_missing_file_id` and `sys.exit(EXIT_CONFIG)` (20).
   It is set **only** in test fixtures (`tests/conftest.py:275`).
2. **`PHAZE_ROLE=agent` + `PHAZE_AGENT_API_URL` + `PHAZE_AGENT_TOKEN`** (+ `PHAZE_MODELS_DIR`) —
   `get_settings()` must return `AgentSettings` and `construct_agent_client(cfg)` needs the API
   URL + token (`config.py:889-901` raises otherwise). The manifest has **no `envFrom`** block to
   source these from an operator-created ConfigMap/Secret, and the runbook never tells the
   operator to create one.

**Impact:** every admitted Kueue pod exits 20 before analysis → `reconcile_cloud_jobs` sees
Failed → re-drives to `cloud_submit_max_attempts` → file marked `ANALYSIS_FAILED`. The
out-of-band callback (`PUT /api/internal/agent/analysis/{file_id}`) is never reached.

**Affected requirements (runtime):** KJOB-02, KSUBMIT-01..06, KSTAGE-03, KROUTE-02.

**Why it passed phase verification:** Phase 52's `job_runner` tests inject `PHAZE_JOB_FILE_ID`
and agent env via the conftest fixture; Phase 54's manifest tests
(`test_kube_staging.py:97-148`) assert structure + the CA mount but not the env contract the pod
consumes. The seam was untested.

**Fix (small, well-scoped):**
- Add `{"name": "PHAZE_JOB_FILE_ID", "value": str(file_id)}` to the container env in
  `build_job_manifest`.
- Add an `envFrom` referencing an operator-created ConfigMap (`PHAZE_ROLE`,
  `PHAZE_AGENT_API_URL`, `PHAZE_MODELS_DIR`) + Secret (`PHAZE_AGENT_TOKEN`).
- Document those objects in `docs/k8s-burst.md` (alongside the existing RBAC/CA Secret).
- Add a manifest-env-contract test asserting `PHAZE_JOB_FILE_ID` + the `envFrom` refs are present
  (the test that would have caught this).
