<objective>
HARD-03: Validate the untrusted `agent_id` query param at BOTH HTTP boundaries.

Two GET endpoints accept an unvalidated `agent_id` query string and silently return an empty
result for a malformed id instead of rejecting it. Harden both (operator decision D-07) with the
established agent-id shape so a malformed id returns 422 at the boundary:
1. `routers/tracklists.py::scan_status` (`GET /scan/status`) — `agent_id: str = Query(...)`.
2. `routers/pipeline_scans.py::agent_roots_swap` (`GET /agent-roots`) — bare `agent_id: str`.

Purpose: Close AR-30-03 / Phase-30 REVIEW IN-01 — input-validation hardening at the boundary.
Output: Pattern+max_length validation on both params plus 422 regression tests.
</objective>
