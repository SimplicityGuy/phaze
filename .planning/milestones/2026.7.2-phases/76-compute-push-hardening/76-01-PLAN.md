<objective>
HARD-01: Make the N-compute liveness probe fan-out session-safe by serializing it.

`services/backends.py::_probe_availability` currently fans `_probe_one` out with
`asyncio.gather`. Since Phase 72 (MCOMP-01) retired the single-active-compute assumption,
N>=2 compute backends are legal, and each compute `_probe_one` touches the shared
request-scoped `AsyncSession` via `select_agent_by_id` (`session.execute`). Concurrent use of
one `AsyncSession` is a SQLAlchemy hazard. Fix by awaiting each `_probe_one` sequentially on the
single session so probes are never concurrent.

Purpose: Close WR-01 (74-REVIEW) — convert an empirically-observed race-free claim into a
structural guarantee.
Output: Serialized `_probe_availability`, reworded docstring/comments, and a deterministic
N>=2-compute regression test.
</objective>
