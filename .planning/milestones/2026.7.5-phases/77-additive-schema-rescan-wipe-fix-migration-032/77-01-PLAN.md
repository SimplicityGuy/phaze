Ship the independently-verifiable rescan-wipe fix (MIG-03 / CONTEXT D-08): remove the
`"state": excluded.state` progress-wipe from the two near-identical `ON CONFLICT DO UPDATE`
upsert sites so re-scanning an already-advanced file no longer resets it to `DISCOVERED`.

`set_` clause (both call sites build new records with `state = DISCOVERED`), silently wiping
pipeline progress. This is the first, standalone deliverable of Phase 77 — it has **zero
dependency on migration `032`** and can be verified/merged on its own.
