<objective>
Thread the `skipped` marker into the single-source derivation contract — ONE new `skipped_clause` builder
plus a 5th `Status` member — so a force-skipped file (a) reads as a distinct `skipped` bucket, (b) leaves
all three enrich pending sets, and (c) is treated as domain-complete by recovery, ALL via the existing
composers with zero per-caller edits. Pure composition change; no reader/writer cuts over here.

`eligible_clause` + `domain_completed_clause` + `stage_status_case` propagates to every pending set, both
recovery paths, and the pill matrix from ONE place. Hand-rolling per-caller skip filters is how drift returns.
Output: the SQL twin (`stage_status.py`) + Python twin (`enums/stage.py`), kept in lockstep.
</objective>
