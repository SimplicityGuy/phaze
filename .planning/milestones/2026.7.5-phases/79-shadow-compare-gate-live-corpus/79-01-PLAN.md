Author the ONE shared assertion core for the state↔derived shadow-compare gate (D-01), plus its
hermetic fixture-corpus CI gate. The core defines an `INVARIANTS` registry — one implication per
`FileState` value per design §6.1 (D-04) — and an async `run_shadow_compare` that, for each
invariant, runs a corpus-wide anti-join (`state = X AND NOT <derived-condition>`) and returns a
`Report` of per-invariant counts + capped `file_id` samples (D-05). The derived side reuses Phase-78's

not cover. `FINGERPRINTED` and `LOCAL_ANALYZING` are a code-commented soft allowlist (D-06).

destructive `033` (Phase 90). Getting the invariant set and the implication-not-equality semantics
right here is load-bearing for the whole milestone.
