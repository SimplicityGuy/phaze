<objective>
Ship the operator retry path for terminally-failed metadata: a bulk endpoint
`POST /pipeline/metadata-failed/retry` that re-enqueues every `metadata.failed_at IS NOT NULL` file,
leaves the failure row in place, and returns an HTMX fragment.

Purpose: FAIL-03 closes gap G-01 — a terminally-failed metadata file must never be a permanent
dead-end that blocks the file from ever reaching `propose` (SC#3). The retry leaves the row and
re-enqueues (D-11 — clearing `failed_at` in place would make a zero-metadata file read DONE forever);
`put_metadata`'s clear-on-success (81-03) wipes the marker only when real metadata lands. The endpoint
mirrors `retry_analysis_failed`'s Phase-30-hardened guard ordering (D-12), minus the state flip
(metadata has no terminal FileState).

Output: `get_metadata_failed_files` in `services/pipeline.py`, the endpoint in `routers/pipeline.py`,
a stage-labelled HTMX fragment, and integration tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator -> control-plane API | `POST /pipeline/metadata-failed/retry` is an operator (internal-realm) bulk enqueue over the current failed-metadata set. |
| enqueue -> SAQ queue | the endpoint routes jobs to a per-agent queue; a fallthrough to the consumer-less default queue is the Phase-30 regression class. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-06-01 | Denial of Service | consumer-less default-queue fallthrough (Phase 30, once stranded 11,428 jobs) | mitigate | resolve the per-agent queue ONCE, catch NoActiveAgentError, return without enqueuing or mutating state — never enqueue to the default queue |
| T-81-06-02 | Denial of Service | unbounded fan-out / duplicate in-flight jobs | mitigate | the central `extract_file_metadata:<file_id>` deterministic key dedups any file with a live in-flight job to a no-op; commit before enqueue |
| T-81-06-03 | Tampering | dead-lettered jobs from a file_id-only enqueue | mitigate | reuse `_enqueue_extraction_jobs` which builds the COMPLETE ExtractMetadataPayload (extra='forbid', four required fields) |
| T-81-06-04 | Elevation of Privilege | metadata failure row deleted/cleared prematurely | mitigate | D-11 — leave the row; only put_metadata's clear-on-success wipes failed_at when real metadata lands |
| T-81-06-SC | Tampering | npm/pip/cargo installs | accept | no package installs in this plan (zero new deps for the phase) |
</threat_model>
