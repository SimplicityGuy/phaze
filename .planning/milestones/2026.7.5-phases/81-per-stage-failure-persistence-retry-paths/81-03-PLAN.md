<objective>
Make `report_metadata_failed` persist a durable metadata failure marker (a `metadata` row with
`failed_at` set, payload columns NULL) instead of nothing, via an optional-body endpoint that stays
version-skew-safe, and make `put_metadata` unconditionally clear that marker on a real success.

Purpose: today `report_metadata_failed` clears only the ledger (`agent_metadata.py:99`), so a
terminally-failed metadata file is invisible in derivation and counts — the latent bug this phase
closes (FAIL-02, D-02). The endpoint must accept a bodyless POST from an old agent image (return
200, clear the ledger) or a triage body from a new agent (D-10), and `put_metadata` must clear
`failed_at` on success or a successful retry reads `failed` forever (D-13).

Output: `MetadataFailurePayload` schema, the failure-row writer + clear-on-success in
`agent_metadata.py`, the widened agent HTTP client, the task-side payload composition, and tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent -> control-plane API | `POST /metadata/{file_id}/failed` is agent-authenticated; the optional body carries agent-supplied free text (`error`). |
| free text -> Postgres -> render | `error_message` persists agent free text and is later rendered on the operator UI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-03-01 | Spoofing | forged agent/file identity via body | mitigate | `agent` comes from `Depends(get_authenticated_agent)`; the ledger key is reconstructed from the PATH `file_id` only, never the body (AUTH-01/T-45-05) |
| T-81-03-02 | Tampering | unknown/extra body fields | mitigate | `MetadataFailurePayload` keeps `extra='forbid'` -> a present body with an unknown field 422s |
| T-81-03-03 | Denial of Service | version-skew: old agent bodyless POST 422s -> ledger never clears -> CR-02 unbounded recovery loop | mitigate | `body: ... | None = None` (no Body wrapper) makes a bodyless POST 200 and still clears `extract_file_metadata:<file_id>` |
| T-81-03-04 | Tampering | PG-invalid free text (NUL/surrogates) or oversized error | mitigate | `error` bounded `max_length=2000`; `error_message` truncated to the column bound before persist (prior incident class: NUL/surrogate write crashes) |
| T-81-03-SC | Tampering | npm/pip/cargo installs | accept | no package installs in this plan (zero new deps for the phase) |
</threat_model>
