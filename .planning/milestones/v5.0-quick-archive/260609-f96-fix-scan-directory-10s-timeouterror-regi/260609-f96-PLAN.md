<objective>
Fix `scan_directory` jobs dying with `asyncio.TimeoutError` after exactly 10 seconds when dispatched to remote agents.

Root cause: `AgentTaskRouter._queue_for` (agent_task_router.py:68) builds per-agent SAQ queues but never registers the `apply_project_job_defaults` `before_enqueue` hook. That hook bumps a Job's timeout from SAQ 0.26.3's 10s default up to the project's `worker_job_timeout` (600s) before the Job is persisted to Redis. The controller queue (controller.py:112) and the agent worker's own queue (agent_worker.py:185) both register it; the per-agent dispatch path used by the API/control plane does not. Agent-dispatched jobs therefore land in Redis with the 10s default and SAQ cancels them after 10s.

Purpose: Long-running agent-dispatched tasks (especially `scan_directory` over tens of thousands of files) must run to completion instead of being killed at 10s and later flagged as stalled.

Output: One-line hook registration in `_queue_for` matching the two existing call sites, plus a regression test.
</objective>

```python
def _queue_for(self, agent_id: str) -> Queue:
    if agent_id not in self._queues:
        self._queues[agent_id] = Queue.from_url(
            self._redis_url,
            name=f"phaze-agent-{agent_id}",
        )
    return self._queues[agent_id]
```

Established registration pattern (controller.py:106-112 and agent_worker.py:179-185):
```python
queue = Queue.from_url(..., name=...)
# Phase 27 UAT Gap 1: SAQ 0.26.3's Worker.__init__ does NOT accept `timeout`,
# `retries`, or `keep_result` -- those are per-Job settings. Apply the project's
# policy defaults via a `before_enqueue` hook on the Queue so every enqueued
# Job inherits the longer timeout / retry budget without breaking Worker
# construction. See phaze.tasks._shared.queue_defaults for the hook body.
queue.register_before_enqueue(apply_project_job_defaults)
```

SAQ internal (verified): `Queue.register_before_enqueue(cb)` stores callbacks in
`queue._before_enqueues` -- a dict keyed by `id(cb)`. Assert membership via
`apply_project_job_defaults in queue._before_enqueues.values()`.
