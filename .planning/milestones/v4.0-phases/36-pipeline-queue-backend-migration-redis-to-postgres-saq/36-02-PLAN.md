<objective>
Perform the atomic core of the migration: route all four queue-construction sites through `build_pipeline_queue` (Postgres backend) AND, in the same change, repoint every `queue.redis` cache reader at a dedicated cache-Redis handle — so no site ever reads a `.redis` attribute that PostgresQueue does not have.

Purpose: This is the highest-risk surface. `tasks/proposal.py:66` is an unwrapped `AttributeError` (hard break) and the Phase-35 counter hooks silently die (soft regression) the moment the queue stops being Redis-backed. Construction swap and cache decoupling MUST land together.
Output: Postgres-backed queues at every site; cache plane fully decoupled; updated unit fakes/tests proving counters and proposals still work.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cache plane ↔ broker plane | Redis (cache/rate-limit/counters) must stay isolated from the now-Postgres broker; readers must not borrow the broker's client |
| API/control process → Postgres | each PostgresQueue opens its own psycopg3 pool — fan-out risk against `max_connections` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | Denial of Service (silent failure) | cache readers borrowing `queue.redis` | mitigate | Repoint proposal.py to `ctx["redis"]` and the counter hooks to `job.queue.cache_redis`; a dedicated cache client survives the backend swap so rate-limiting and counters never read a missing attribute |
| T-36-04 | Denial of Service (connection exhaustion) | per-queue psycopg3 pools (controller + per-agent + monitoring) | mitigate | Conservative per-queue sizing (controller 2/8, agent/per-agent 1/4) so total budget (Σ queue pools + SQLAlchemy engine pools) stays under Postgres `max_connections` (RESEARCH Pitfall 4; homelab budget documented in Plan 04) |
| T-36-06 | Tampering (regression) | default-queue producer guard | mitigate | Factory uses named `PostgresQueue.from_url(..., name=...)`; `tests/test_no_default_queue_producers.py` stays green, preventing a silent unnamed-queue producer reintroduction |
</threat_model>
