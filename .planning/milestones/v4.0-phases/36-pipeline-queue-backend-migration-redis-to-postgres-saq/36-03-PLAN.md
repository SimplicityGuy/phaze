SAQ behavior under test (RESEARCH Pattern 2 + Pitfall 3, verified in saq/queue/postgres.py):
- dequeue: `... WHERE now>=scheduled AND priority BETWEEN .. ORDER BY priority, scheduled ... FOR UPDATE SKIP LOCKED` — lower priority int dequeues first; future `scheduled` parks.

- `PostgresQueue.info()` (postgres.py:245) is backend-agnostic — what `saq_web`/`build_saq_app` renders.
