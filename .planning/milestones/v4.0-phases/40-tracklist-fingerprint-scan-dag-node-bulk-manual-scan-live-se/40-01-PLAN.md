<objective>
Add the SECOND, independent tracklist-discovery node to the DAG: a **Fingerprint Scan**
("Identify Set") node whose bulk endpoint enqueues `scan_live_set` (agent-side audio-fingerprint
identification) over files that do not yet have a tracklist, gated disabled until there are
discovered files AND an online file-server agent, and gated "busy" while a scan batch is in flight.

Purpose: Phase 39 turned the tracklist head into a name-**Search** node (controller-routed). This
phase adds its sibling — the fingerprint-scan path, which is a PER-AGENT task (it reads the audio on
the file-server agent and POSTs back a resolved tracklist). Both produce tracklists and run
INDEPENDENTLY over all files with NO fallback. Manual only — NO auto-trigger (auto-enqueue is reserved
for the Phase-42 recovery pass).

Output: a `POST /pipeline/scan-live-sets` HTMX endpoint (per-agent routing, COMPLETE payload,
graceful no-agent empty-state), a degrade-safe `get_scan_busy_count` + `count_active_agents` service
pair, `scanBusy` + `agentOnline` store keys, and the 10th DAG node + its gate on the canvas — all with
co-located TDD tests, full suite green, ≥85% coverage.
</objective>
