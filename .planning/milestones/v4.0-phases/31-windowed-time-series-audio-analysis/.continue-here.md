## 31-05 EXPANDED BRIEF (resilience decision 2026-06-10)
Plan 31-05's timeout/retry tuning is now resilience-driven, NOT "unbounded":
- Set `worker_job_timeout` **bounded-generous** sized from the spike — ~**14400 s (4 h)**, configurable
  via env — covering the largest real set (a 4 h Coachella set ≈ ~2.5–3 h analysis) with margin.
  **NOT 0/unbounded** — a *bounded* timeout is what lets SAQ reclaim a dead/restarted worker's
  in-flight file instead of orphaning it (resilient across container restarts).
- `retries=1` (process_file is idempotent via 31-03 PUT-replace; never 4× a slow-but-valid long file).
- These apply through the existing `apply_project_job_defaults` hook + `config.worker_job_timeout`/
  `worker_max_retries`. Keep the change config-driven.
