<objective>
Run the MANDATORY spike before any production rewrite. Validate, against a REAL >=2h compressed concert file, that segmented `EasyLoader(startTime, endTime, sampleRate)` per-window decode (a) decodes every window without exception, (b) feeds `RhythmExtractor2013(multifeature)` a 30s buffer with no `OnsetDetectionGlobal` overflow, (c) keeps RSS bounded (flat, not growing with window_index), (d) has acceptable coarse-tier TF inference time, and (A1) shows non-quadratic per-window seek cost.

Purpose: 31-RESEARCH.md empirically INVERTED the design spec — essentia's Python *streaming* framework (`MonoLoader -> FrameCutter -> Pool`) accumulates the whole file into the Pool (measured `(21, 1323000)`), reintroducing the OOM. `EasyLoader` segmented decode is the real fix (RSS flat 240->259 MB across 20 decodes). The only remaining unknowns are A1 (real-file VBR seek cost) and A2 (real total inference time). This spike answers them with real numbers and LOCKS the decode strategy that Plan 04's `analyze_file` rewrite implements.

Output: a throwaway `scripts/spike_windowed_analysis.py` (NOT committed) and a decision-log entry in this plan's SUMMARY recording the measured numbers and the locked EasyLoader-vs-hybrid choice.

NOTE: this plan is NOT autonomous — Task 2 is a blocking human-verify checkpoint (the real-file run happens on the homelab agent where the archive lives).
</objective>
