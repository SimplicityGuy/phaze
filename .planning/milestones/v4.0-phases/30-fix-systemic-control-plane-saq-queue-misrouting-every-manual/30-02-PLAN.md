Eight handlers (four JSON `/api/v1/*` + four HTMX `/pipeline/*`) currently grab `request.app.state.queue` (the dead default queue). Route them all through the Plan 01 helper so `process_file`, `extract_file_metadata`, `fingerprint_file` land on a live agent's queue and `generate_proposals` lands on the controller queue.

Purpose: This is the user-facing "Run analysis" button. After this plan it actually feeds a worker.
Output: pipeline.py with zero `app.state.queue` references, plus regression tests asserting the destination queue per endpoint and the 0-active-agent branch.
</objective>
