<objective>
Fix the legacy scan endpoint's auto-enqueue path. `POST /api/v1/scan` passes `app.state.queue` (the dead default) into `ingestion.run_scan`, which enqueues `extract_file_metadata` per discovered file — so on the legacy path those jobs also rot. Select an active agent at trigger time and hand `run_scan` the agent's `phaze-agent-<id>` queue.

Purpose: close the last misrouted producer (the legacy `/scan` -> ingestion path) so no control-plane code feeds the default queue.
Output: scan.py routes the metadata-extraction enqueue per-agent; ingestion.run_scan's contract documented; regression test asserts named-queue targeting + the zero-agent path.
</objective>
