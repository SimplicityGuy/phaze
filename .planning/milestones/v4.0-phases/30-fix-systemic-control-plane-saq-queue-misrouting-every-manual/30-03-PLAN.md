<objective>
Fix the tracklists router's four misrouted enqueues and the coupled scan-status poll. Three are controller-bound (`scrape_and_store_tracklist`, `search_tracklist`, `match_tracklist_to_discogs`); one is per-agent (`scan_live_set`). The subtlety: `scan_status` looks jobs up via `queue.job(job_key)`, which is queue-scoped — once `scan_live_set` moves to a `phaze-agent-<id>` queue, the poll must query that same per-agent queue, so the selected `agent_id` is threaded from `trigger_scan` through the progress partial into the poll.

Purpose: tracklist scraping/matching/fingerprint-scan triggers currently rot on the default queue; restore them to consumed queues without breaking live progress polling.
Output: tracklists.py + scan_progress.html updated, regression tests asserting queue targeting and the agent_id poll round-trip.
</objective>
