<objective>
Build the shared routing foundation every misrouted control-plane enqueue will use, and eliminate the consumer-less unnamed `default` SAQ queue that caused the v4.0.6 incident (11,428 `process_file` jobs stranded in `saq:job:default:*`).

Purpose: A single source of truth maps each task name to the queue an actual worker consumes — `controller` (lux phaze-worker) or `phaze-agent-<id>` (nox phaze-agent-worker). After this plan, no API code path can construct or enqueue to the unnamed default queue, and per-agent dispatch selects a live agent (excluding the permanently-revoked `legacy-application-server`).

Output: `src/phaze/services/enqueue_router.py` (new), a named `controller` queue wired in the lifespan, a public `AgentTaskRouter.queue_for`, and unit tests for the routing + selection logic. The per-site fixes (Plans 02-04) depend on this.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP control-plane (API) -> SAQ/Redis | Operator-triggered enqueues cross into the task queue; a wrong queue name strands jobs (the incident). |
| API -> agents table | Active-agent selection trusts DB state (revoked_at / last_seen_at) to pick a dispatch target. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | Denial of Service / Data-integrity | enqueue to consumer-less default queue | mitigate | Remove the unnamed default queue from the lifespan; `resolve_queue_for_task` maps every known task to a consumed queue and raises `ValueError` on unknown names (fail loud, never silently default). |
| T-30-02 | Spoofing / Elevation | dispatching to a revoked/dead agent | mitigate | `select_active_agent` filters `revoked_at IS NULL` AND `last_seen_at IS NOT NULL`; `legacy-application-server` is excluded by the revoked filter. |
| T-30-03 | Tampering | attacker-influenceable queue name | mitigate | Queue name derives only from `Agent.id` (DB PK with CHECK `^[a-z0-9]+(-[a-z0-9]+)*$`) and a fixed task-name allowlist; no user input reaches the queue name. |
| T-30-04 | Denial of Service | 0 active agents silently "succeeds" | mitigate | `select_active_agent` raises `NoActiveAgentError`; consuming sites (Plans 02-04) surface a clear error/empty-state instead of a silent 200. |
| T-30-SC | Tampering | package installs | accept | No new packages introduced in this phase (pure refactor of existing SAQ/SQLAlchemy code). |
