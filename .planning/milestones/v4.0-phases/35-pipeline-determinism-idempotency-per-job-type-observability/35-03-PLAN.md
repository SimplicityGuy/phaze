<objective>
Build the authoritative per-stage reconcile query the DAG needs (RESEARCH Q5 — the biggest planning impact). The existing `get_pipeline_stats()` groups by the LINEAR `FileRecord.state` (one enum per file) and STRUCTURALLY CANNOT report parallel-stage done-counts: a file that is both fingerprinted AND analyzed carries only one `state` string. Add `get_stage_progress(session)` that counts each stage's OUTPUT TABLE via `COUNT(DISTINCT file_id/tracklist_id)`, plus the matching denominators from the UI-SPEC "Counter / Denominator Display Rule". This is the DB-truth that self-heals the maintained counters on read (D-03).

Purpose: Without a per-output-table count, every parallel DAG node would report wrong done numbers. This query is the reconcile authority (D-03 mandatory) and the data source for the DAG nodes (35-04/35-05).
Output: `get_stage_progress` in `services/pipeline.py` returning per-node done/total, with the Scan/Search node explicitly denominator-less, and per-source failure isolation matching `get_queue_activity`.
</objective>
