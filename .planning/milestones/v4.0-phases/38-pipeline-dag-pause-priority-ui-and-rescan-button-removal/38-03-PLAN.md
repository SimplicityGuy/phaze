<objective>
Plumb per-stage `{paused, priority}` live state into the existing 5s `/pipeline/stats` OOB poll (REQ-38-4) so the Phase 38 controls reflect authoritative server state across every refresh. Three seams: (1) a degrade-safe `get_stage_controls(session)` reader in `services/pipeline.py`; (2) `_build_dag_context` extended with six integer keys; (3) the `base.html` `$store.pipeline` literal seeded with the six keys. The existing `dag.items()` OOB loop in `stats_bar.html` (and the in-place twin in `dag_canvas.html`) propagate the new keys for free.

Purpose: This is the read-only data substrate the controls bind to. It must NEVER 500 the hot poll path — the reader degrades to defaults (paused=False, priority=50) on any failure, identical to the repo-wide `_safe_count` / `get_queue_activity` discipline (T-35-09).
Output: `get_stage_controls`; six int keys in `_build_dag_context`; six store-literal keys; extended store/int/OOB-seed tests + a control-table-unreadable degrade test; README pipeline-dashboard docs updated.

Cross-phase note: `get_stage_controls` imports `PipelineStageControl` from Phase 37. The degrade-safe try/except returns defaults if that model/table is absent, so this plan renders harmlessly before Phase 37 lands; the live values become real only once Phase 37 ships.
</objective>
