**Pause/Resume structure (LOCKED ŌĆö RESEARCH Open Q4):** TWO `x-show`-gated buttons, each with a
**static** `hx-post`:
- `Pause` ŌåÆ `hx-post=".../pause"`, `x-show="!$store.pipeline.<stage>Paused"`
- `Resume` ŌåÆ `hx-post=".../resume"`, `x-show="$store.pipeline.<stage>Paused"`

Do NOT use a single label-flipping button with a bound `:hx-post` ŌĆö the static two-button form keeps each

`$store.pipeline.<stage>Paused = r.paused ? 1 : 0`. Inner button row: `flex items-center justify-between gap-2`;
stepper group: `inline-flex items-center gap-1`.

## State Behavior Contract

| Control | States |
|---------|--------|
| Pause/Resume toggle | **running** (amber "Pause") Ōåö **paused** (green "Resume"). In-flight: `hx-disabled-elt` greys the clicked button for the request duration. |
| Priority stepper | **normal** ┬Ę **at floor** (Ō¢▓ disabled, priority 0 = max urgency) ┬Ę **at ceiling** (Ō¢╝ disabled, priority 100 = least urgent) ┬Ę **in-flight** (clicked button disabled). |
| Degrade | If `pipeline_stage_control` is unreadable, `get_stage_controls` returns defaults ŌåÆ controls render at **running, priority 50**. Never an error, never a broken control; the 5s poll still returns 200. |
| Pre-poll | Full-page load seeds real values in-place via the `dag.items()` loop (no flash). The store's `0` default applies only for the sub-millisecond before Alpine `x-init` runs. |

**Update timing (LOCKED ŌĆö RESEARCH Open Q2): AUTHORITATIVE-ONLY.** On click, do NOT optimistically
mutate the store. Write the store solely from the server JSON response (`@htmx:after-request`) and from
the next 5s poll. This is flicker-free on a single-user LAN tool and avoids the optimistic-vs-poll race.

**Store seed invariant:** `paused` is emitted to the template as int `0`/`1` (via `int(controls[stage]["paused"])`),
never a Python `True`/`False`, to preserve the canvas's "every `dag` value is a server-computed int safe to
interpolate into `x-init`" rule (Pitfall 3 / T-35-11). The store reads `<stage>Paused` for truthiness.

## Interaction Flow

1. Operator views the DAG; each of the 3 agent chips shows its live priority number and pause state (from the store, refreshed every 5s).
2. Click **Ō¢▓ Higher** on Analyze ŌåÆ `POST .../analyze/priority {delta:-10}`, `hx-swap="none"`. HTMX disables the button for the request. Endpoint returns `{stage:"analyze", priority:40, paused:false}`. Alpine writes `$store.pipeline.analyzePriority = 40`. Number updates; Ō¢▓ re-enables (still > 0).
3. Click **Pause** on Metadata ŌåÆ `POST .../metadata/pause`. Returns `{ŌĆ”, paused:true}`. The `Pause` button hides, `Resume` (green) shows. Backlog parks (Phase 37).
4. The 5s poll continues re-pushing all six keys, reconciling any out-of-band change (another tab, a re-parked re-enqueue).

## Layout Contract (CRITICAL ŌĆö forced recompute)

Adding the control row (divider + button row + hint Ōēł **70ŌĆō90px**) grows each agent chip from ~154px to
~**240ŌĆō260px**. The current `NODE_LAYOUT` col-1 stack uses a uniform **182px** gutter ŌĆö the taller chips
WILL overlap the chip below and fail `test_topology_column_one_chips_do_not_overlap` unless recomputed.

**Required (treat as a first-class task, not an afterthought):**

| Item | Current | Required |
|------|---------|----------|
| col-1 gutter | 182px | **Ōēź measured agent-chip height + 16px** (target ~260px) |
| `metadata.y` | 24 | 24 (unchanged) |
| `analyze.y` | 206 | ~300 |
| `fingerprint.y` | 388 | ~576 |
| `scan_search.y` (no controls, ~126px) | 570 | ~852 |
| canvas wrapper `height` (`dag_canvas.html:177`) | 720px | grow to fit (Ōēł1000px) |
| `<svg>` `height` + `viewBox` (`:178`) | 720 | match new wrapper height |
| col-2/col-3 node `y` (proposals/scrape/execute/match) | per map | re-balance so curved edges still land cleanly on the new col-1 anchors |
| `test_topology_column_one_chips_do_not_overlap` `min_chip_height` | current | update to the new measured agent-chip height |

Exact pixel values are a render-and-measure task; the binding constraint is **gutter Ōēź chip height + 16px**
and a green overlap regression test. The Scan/Search node (no controls) and col-2/col-3 nodes shift to
