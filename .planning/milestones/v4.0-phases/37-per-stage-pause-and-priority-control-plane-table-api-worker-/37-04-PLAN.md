<objective>
Ship the three operator-facing control endpoints — `POST /pipeline/stages/{stage}/priority` (delta), `/pause`, and `/resume` — that mutate the `pipeline_stage_control` row (durable intent) and the live `saq_jobs` backlog (via the Plan-02 service helpers) in a single transaction, and return `{stage, priority, paused}` so Phase 38's UI can re-render.

Purpose: This is the API surface for REQ-37-1 (pause), REQ-37-2 (priority delta + clamp), and REQ-37-3 (resume). Input validation (allowlist `stage`, clamp `delta`) is the Tampering/DoS mitigation. The endpoints reuse the existing `/pipeline/*` conventions and sit behind the same reverse-proxy internal-realm auth as the rest of `phaze-api` (no app-layer auth added).

Output: `StagePriorityDelta` schema; `routers/pipeline_stages.py` with the 3 endpoints registered in `main.py`; endpoint unit tests (httpx); README/docs describing the new endpoints and the locked Open-Q defaults.

Open-Q defaults documented here (RESEARCH recommendations, no discuss-phase): priority is a DELTA op with default step ±10 (10 discrete levels across 0–100), body `{delta: int}`, result clamped `[0,100]`, returns the new absolute value; resume un-parks only (does NOT restore priority). (TTL=5s and pause-persists-reboot are documented in Plan 02.)
</objective>
