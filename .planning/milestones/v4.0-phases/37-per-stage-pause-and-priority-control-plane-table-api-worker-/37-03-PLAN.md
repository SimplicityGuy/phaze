SAQ Postgres semantics (37-RESEARCH, verified vs saq/queue/postgres.py):
- dequeue: `WHERE status='queued' AND queue=:q AND now>=scheduled AND priority BETWEEN plow AND phigh ORDER BY priority, scheduled ... FOR UPDATE SKIP LOCKED`; lower priority + earlier scheduled dequeues first.
- `count("queued")` = `WHERE status='queued' AND now>=scheduled` — a parked row (scheduled=SENTINEL) is excluded (Pitfall 1). `count("incomplete")` includes it.
- retry sets `scheduled = now + delay` (near-future, ≠ SENTINEL).
- `scheduled=0` ⇒ `now>=0` ⇒ immediately dequeueable ("run now").

Real-PG engine/test pattern (tests/test_migrations/test_migration_018.py:81-129): build `create_async_engine(MIGRATIONS_TEST_DATABASE_URL)`, `async with engine.begin()`, run raw `text()` SQL, assert counts. `MIGRATIONS_TEST_DATABASE_URL` + helpers from `tests/test_migrations/conftest.py`.
