<objective>
Give the DAG **Scrape** and **Match** nodes real manual triggers. Today both are display-only
chips gated only on `tracklistDone === 0`. This phase adds a bulk **Scrape** trigger
(`scrape_and_store_tracklist` for every tracklist missing a scraped version) and a bulk **Match**
trigger (`match_tracklist_to_discogs` for every tracklist not yet linked to Discogs). Both are
"bulk over pending" — they skip already-done rows and dedup in-flight via the deterministic
`tracklist_id` key — and both route to the CONTROLLER queue (mirroring the per-tracklist
`rescrape_tracklist` / `match_discogs` triggers and the Phase-39 bulk Search trigger).

Purpose: Phase 39 made the Search head a trigger, Phase 40 added the sibling Fingerprint-Scan head.
Both produce tracklists. This phase makes the two downstream tracklist stages (scrape → match)
operator-triggerable in bulk over their pending sets. Manual only — NO auto-trigger (automatic
enqueue is reserved for the Phase-42 recovery pass).

Output: `POST /pipeline/scrape-tracklists` + `POST /pipeline/match-tracklists` controller-routed HTMX
endpoints; degrade-safe `get_scrape_busy_count` / `get_match_busy_count` and the eligible-set helpers
`get_scrape_pending_tracklists` / `get_match_pending_tracklists`; `scrapeBusy` + `matchBusy` store
keys; the two enqueue buttons + their gates on the canvas; a tracklist-unit response partial — all
with co-located TDD tests, full suite green, ≥85% coverage.

Eligible-set queries = the COMPLEMENT of the get_stage_progress done-counts. From
src/phaze/services/pipeline.py::get_stage_progress, scrape.done = COUNT(DISTINCT TracklistVersion.tracklist_id)
and match.done walks discogs_links → tracklist_tracks → tracklist_versions counting DISTINCT
TracklistVersion.tracklist_id. The pending sets are:
```python
# Scrape pending = tracklists with NO row in tracklist_versions
select(Tracklist).where(~exists(select(TracklistVersion.id).where(TracklistVersion.tracklist_id == Tracklist.id)))

# Match pending = tracklists NOT reachable from discogs_links (same join-walk get_stage_progress uses)
matched_subq = (
    select(TracklistVersion.tracklist_id)
    .select_from(DiscogsLink)
    .join(TracklistTrack, DiscogsLink.track_id == TracklistTrack.id)
    .join(TracklistVersion, TracklistTrack.version_id == TracklistVersion.id)
)
select(Tracklist).where(Tracklist.id.not_in(matched_subq))
```
