The real SAQ `QueueInfo` returned by `Queue.info()` (read from the installed dist at
`.venv/lib/python3.14/site-packages/saq/queue/redis.py:119-178`) is a TypedDict-shaped
mapping with exactly these keys:
- `workers`: dict[str, WorkerInfo]  (empty dict `{}` is valid)
- `name`: str  (the queue name — MUST echo `self.name`)
- `queued`: int
- `active`: int
- `scheduled`: int
- `jobs`: list  (job dicts; empty list `[]` is valid when `jobs=False`)

`saq_web` consumes it via `_get_all_info()` => `[await q.info() for q in QUEUES.values()]`
(`saq/web/starlette.py:104-105`) and `q.info(jobs=True)` for a single queue
(`saq/web/starlette.py:67`). The signature it relies on is
`async def info(self, jobs: bool = False, offset: int = 0, limit: int = 10) -> QueueInfo`.

Existing `FakeQueue` (tests/_queue_fakes.py:56-91) already has `name`, `enqueue`, `job`,
`captured`, `captured_policy`. It does NOT have `info` — that is the only gap this plan closes.
