<objective>
Add a visible link from the main pipeline dashboard page (`GET /pipeline/`) to the
SAQ queue-monitor UI mounted at `/saq` (delivered in plan 33-02). Operator request
2026-06-11: the SAQ dashboard should be reachable from the pipeline page, not only by
typing the `/saq` URL directly.
</objective>

<context>
- The SAQ dashboard is mounted at `/saq` in the lifespan (plan 33-02), gated by
  `settings.enable_saq_ui` (default True). It is SAQ's own full-page Starlette app —
  NOT a Jinja page extending `base.html` and NOT an HTMX partial. So the link must be a
  plain navigation anchor that opens the separate app; open it in a new tab
  (`target="_blank" rel="noopener"`) so the operator does not lose the pipeline page.
- The pipeline dashboard template is `src/phaze/templates/pipeline/dashboard.html`. Its
  header is currently a bare `<h1 class="text-2xl font-semibold leading-tight">Pipeline
  Dashboard</h1>` at the top of the `{% block content %}` `space-y-6` wrapper.
- Match the project's existing Tailwind nav-link styling (see `base.html` nav anchors):
  `text-sm font-semibold px-3 py-2 rounded-md` with the muted-gray + hover treatment.
- Existing render test: `tests/test_routers/test_pipeline.py::test_dashboard_page`
  (`GET /pipeline/` → 200, asserts "Pipeline Dashboard" present). Add a sibling test in
  the same file/style.
</context>
