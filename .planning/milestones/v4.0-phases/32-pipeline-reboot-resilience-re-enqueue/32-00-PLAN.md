<objective>
Build the Wave-0 test harness that makes SAQ's deterministic-key dedup no-op behavior unit-testable WITHOUT a live Redis. The existing `FakeQueue` (tests/_queue_fakes.py) always appends and returns a fresh `MagicMock` job — it cannot express "a re-enqueue of an in-flight file is a no-op", which is the load-bearing primitive of this whole phase (RESEARCH §Q5 Wave-0 gap).

Purpose: Wave 2's re-enqueue task asserts that re-enqueuing a still-incomplete file is a no-op. That assertion needs a fake whose `enqueue` returns `None` for a repeat live deterministic key, mirroring `RedisQueue._enqueue`'s `ZSCORE incomplete` check (RESEARCH §Q1).
Output: A `DedupFakeQueue` + `DedupFakeTaskRouter` (additive, zero blast radius on the 6 existing `FakeQueue` consumers) plus a self-test module.
</objective>

From tests/_queue_fakes.py:
- `_JOB_CONTROL_FIELDS = frozenset(Job.__dataclass_fields__)` — `key` IS a member (SAQ Job dataclass field), so a `key=` kwarg is split into `captured_policy`, not `captured`.
- `class FakeQueue(name, capture=None)` — `.enqueue(task_name, **kwargs)` splits kwargs into `captured` (payload) + `captured_policy` (job-control: timeout/retries/key/...); always returns a `MagicMock` job with a unique `.key`; `.job` is an `AsyncMock(return_value=None)`.
- `class FakeTaskRouter(capture=None)` — `.queue_for(agent_id)` returns a cached `FakeQueue(f"phaze-agent-{agent_id}", self.captures)`; records calls on `.queue_for_calls`.

<!-- SAQ real semantics this fake must mirror. From 32-RESEARCH §Q1. -->
- `RedisQueue._enqueue`: if `job.id` (which embeds the deterministic `key`) is already a member of the per-queue `incomplete` sorted set, the Lua script returns nil → `Queue.enqueue` returns `None` (clean no-op, no raise, no overwrite). The key leaves `incomplete` only when the job finishes — so after completion the same key can enqueue again.
- Scope is per-queue: the same key on two different `phaze-agent-<id>` queues = two different `job.id`s = no cross-queue dedup (accepted single-agent edge case).
