<objective>
Build the controller-side re-enqueue task `reenqueue_discovered(ctx)` that resumes the analysis stage after a reboot/queue loss: query Postgres for `FileState.DISCOVERED` files, pick the active agent, and re-enqueue `process_file` for each through the Wave-1 shared helper. The shared deterministic key makes already-in-flight files dedup to a no-op, so the task is safe to run on every boot AND every cron tick (CONTEXT "Trigger" + "Scope" decisions; RESEARCH §Q2/§Q3/§Pattern 2).

Purpose: This is the recovery layer on top of Phase 31's bounded timeout/retries. It must route via `select_active_agent` + `task_router.queue_for(agent.id)` — NEVER `ctx["queue"]` (the consumer-less controller queue, RESEARCH Pitfall 1) — and degrade gracefully when no agent is live (common right after a cold reboot, RESEARCH Pitfall 3).
Output: `tasks/reenqueue.py::reenqueue_discovered` (mirrors `tasks/scan_reaper.py`) returning `{"reenqueued": N, "skipped": M}`, plus its full test suite.
</objective>
