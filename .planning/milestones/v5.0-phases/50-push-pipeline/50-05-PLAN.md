## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| agent → control internal API | token-authed; the only path the Postgres-free agent mutates FileRecord/ledger. |
| control ORM → compute queue | sha256_hash is read from the ORM here and pinned into the payload (untrusted-agent never supplies it). |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-spoofed-callback | Spoofing | forged push callback | mitigate | `get_authenticated_agent` token dependency; file_id from the path, identity never from the body (AUTH-01). |
| T-50-double-enqueue | Tampering | duplicate process_file / push_file from a repeated callback | mitigate | state transition + ledger clear are one transaction; deterministic keys (process_file:<id>, push_file:<id>) collapse repeats. |
| T-50-loop | Denial of Service | endless re-push of a corrupt source | mitigate | push_attempt counter in the ledger payload capped at push_max_attempts → ANALYSIS_FAILED (Pitfall 4). |
| T-50-integrity-pin | Tampering | agent supplying its own expected_sha256 | mitigate | expected_sha256 is read control-side from FileRecord.sha256_hash, never from the agent (D-11). |
| T-50-SC | Tampering | npm/pip/cargo installs | accept | No new packages; supply-chain gate N/A. |
</threat_model>
