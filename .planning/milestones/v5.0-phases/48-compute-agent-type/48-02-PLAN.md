<objective>
Let the operator register a compute agent (CLOUDAGENT-01): add a `--kind` CLI flag that relaxes the `--scan-roots` requirement for compute, and an `AgentSettings.kind` field that relaxes the empty-scan-roots startup gate so a compute worker process boots with no scan roots.

Purpose: v4.0 assumed every agent owns scan roots (it owns FileRecords). A compute (cloud) agent owns no files and no scan roots. Without these two relaxations the operator cannot register a compute agent (argparse rejects) and the worker crash-loops at startup (config ValueError). `kind` is constrained at the CLI (argparse choices) and config (Literal) layers as the outer two of the 3-layer defense whose inner layer (DB CHECK) shipped in Plan 01.
Output: working `phaze agents add --kind compute` path + `AgentSettings.kind` with the gate relaxed, both fully tested.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-48-02 | Tampering | `--kind` CLI flag + `AgentSettings.kind` | mitigate | Outer two layers of the 3-layer enum defense: argparse `choices=("fileserver","compute")` rejects bad values before any session opens; `kind: Literal["fileserver","compute"]` rejects bad config at settings construction. Inner DB CHECK (Plan 01) is the backstop. |
| T-48-03 | Information Disclosure | minted bearer token during registration | mitigate | Preserve D-13: token is `print()`-only and never logged; `configure_logging()` runs first; the `--kind` change touches no token-handling line. A test asserts the token does not reach a logger. |
| T-48-04 | Elevation of Privilege | relaxed scan-roots gate for compute | mitigate | Only the `scan_roots` requirement is relaxed and ONLY when `kind == "compute"`; `agent_api_url` + `agent_token` stay required for all kinds, so a compute agent still authenticates with a bearer over HTTP. Relaxing roots grants no DB/media access (that boundary is held by the import-boundary test in Plan 03 + no media mount in Phase 51). |
| T-48-SC | Tampering | dependency install | accept | No packages installed — only argparse (stdlib) + pydantic-settings already pinned. No supply-chain surface. |
