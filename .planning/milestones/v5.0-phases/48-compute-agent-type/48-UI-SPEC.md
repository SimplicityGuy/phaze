# Phase 48 — UI Design Contract

> Visual and interaction contract for the Agents admin page kind-badge addition (CLOUDAGENT-03).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

**This is a small, additive change to an existing read-only admin page — not a new screen.**

The only net-new visual element in Phase 48 is a **kind badge** that distinguishes a
`kind="compute"` (cloud) agent from a `kind="fileserver"` (local) agent on the Agents
admin page, so the operator can see available cloud capacity at a glance (CLOUDAGENT-03).

Everything else on this page already ships and is **unchanged**:

| Already present (DO NOT re-specify or alter) | Source |
|----------------------------------------------|--------|
| 5-state liveness pill (alive/stale/dead/revoked/never) | `_status_pill.html` (LOCKED, Phase 29) |
| Queue-depth cell (`last_status.queue_depth`) | `agents_table.html:52-58` |
| 5s HTMX self-replacing poll + "never halt" invariant | `agents_table.html` (LOCKED, Phase 29) |
| "Last refreshed Ns ago" footer | `agents_table.html:69-76` (LOCKED) |
| Red `role=alert` "Refresh failed" banner | `agents_table.html:78-99` + `agents.html` (LOCKED, Phase 29 BLOCKER-2) |
| Empty state ("No agents registered yet") | `agents_table.html:23-29` (LOCKED) |

**No new screen, no new route, no new CTA.** Compute-agent registration is CLI-only
(`phaze agents add --kind compute`); this page is read-only. `_load_agents` already does a
bare `SELECT Agent`, so `kind` rides along free on the loaded row — no router/query change.

## Spacing Scale

The project uses Tailwind's default 4px scale. The kind badge reuses the **project-wide pill
geometry verbatim** (`px-2 py-0.5`), matching `_status_pill.html`.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | — |
| sm | 8px | Badge horizontal padding (`px-2`) |
| md | 16px | Table cell horizontal padding (`px-4`, existing) |
| (cell vert) | 12px | Table cell vertical padding (`py-3`, existing) |

**Exceptions:** Badge vertical padding `py-0.5` = **2px** (not a multiple of 4). This is the
LOCKED project-wide pill ladder geometry inherited from `_status_pill.html` /
`scan_status_pill.html` — the kind badge MUST match it for visual consistency. Do not "fix"
it to `py-1`; that would make the kind badge taller than the adjacent status pill.

## Typography

Existing vocabulary (do not expand). The kind badge uses the same size/weight as every other
pill: **12px / semibold**.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Heading | 24px (`text-2xl`) | 600 (`font-semibold`) | tight (`leading-tight`) | Page `<h1>` "Agents" (existing) |

Font weights in use: exactly **2** — regular (400) and semibold (600).

## Color

60/30/10 split for the page (existing, unchanged), plus one categorical hue introduced by the
kind badge.

| Role | Value (light / dark) | Usage |
|------|----------------------|-------|
| Dominant (60%) | `#ffffff` / `phaze-bg #0a0c12` | Page background, table surface |
| Secondary (30%) | `gray-50` / `phaze-panel #10141c`; borders `gray-200` / `phaze-border #232832` | Row hover, panel, table border |
| Accent (10%) | phaze cyan `#00b0d8` (`blue-500/600`) | Nav active, links, focus ring — **NOT used by the kind badge** |
| Destructive | `red-700` / `red-400` | DEAD pill + "Refresh failed" banner (existing); **no destructive action in this phase** |

**Accent reserved for:** navigation active-state, hyperlinks, focus outlines, the logo mark.
The kind badge deliberately does **not** use the cyan accent — a brand-accent badge would
compete with nav/links and blur the "this is a categorical attribute" reading.

### Kind badge palette (NEW — the one net-new color decision)

The badge mirrors the LOCKED pill formula `bg-{c}-100 dark:bg-{c}-950 text-{c}-700 dark:text-{c}-400`
(file-server uses the gray `-800/-300` variant, matching the "no-signal" status pills' darker
neutral). Hue is chosen to be distinct from BOTH the liveness pill ladder (green/amber/red/gray)
AND the brand cyan accent, so the operator never confuses kind with status.

| Kind | Tailwind classes | Why this hue |
|------|------------------|--------------|
| **compute** (cloud capacity) | `bg-indigo-100 dark:bg-indigo-950 text-indigo-700 dark:text-indigo-400` | Indigo is unused by the status ladder and is NOT the cyan brand accent (indigo is violet-leaning, cyan is teal-leaning — clearly separable). It **pops** so the operator's eye lands on cloud capacity. |
| **fileserver** (local, baseline) | `bg-slate-100 dark:bg-slate-800 text-slate-700 dark:text-slate-300` | Quiet cool-neutral. The baseline kind should **recede** so compute stands out. Slate stays in the pill ladder's neutral family without colliding with any semantic status color. |

Contrast: both variants reuse the exact lightness relationship of the already-shipped
`green`/`gray` status pills (which pass WCAG AA), so AA is inherited.

**Color is never the sole signal** — each badge carries an explicit uppercase label AND an
`aria-label` (see Component Contract), so the indigo/slate distinction is redundant, not
load-bearing, for colorblind and screen-reader users.

## Component Contract — Kind Badge (`_kind_badge.html`)

The core deliverable. A new partial `src/phaze/templates/admin/partials/_kind_badge.html`,
included from `agents_table.html` exactly where `_status_pill.html` is included (`:51`), so it
renders identically on first full-page load AND every 5s HTMX poll (avoids the Pitfall-5 flicker).

**Geometry (LOCKED, copy from `_status_pill.html`):**
`text-xs font-semibold px-2 py-0.5 rounded-full`

**Variants:**

| `agent.kind` | Rendered label | Classes (added to geometry) | `aria-label` |
|--------------|----------------|------------------------------|--------------|
| `"compute"` | `COMPUTE` | `bg-indigo-100 dark:bg-indigo-950 text-indigo-700 dark:text-indigo-400` | `Kind: compute` |
| `"fileserver"` | `FILE SERVER` | `bg-slate-100 dark:bg-slate-800 text-slate-700 dark:text-slate-300` | `Kind: file server` |

- **Both kinds render a badge.** Decision: showing a badge only for compute would leave
  fileserver cells empty (reads as "missing data") and force the operator to infer
  "no badge = fileserver." Kind is a first-class attribute for every row — explicit over implicit.
- **No icon.** The design system has no icon library and no pill carries an icon; the badge stays
  text-only for consistency. (A future inline-SVG cloud glyph for compute is explicitly deferred —
  do not add one in Phase 48.)
- **Defensive default:** the DB column is `NOT NULL DEFAULT 'fileserver'` with a CHECK in
  `('fileserver','compute')`, so the template only needs the two branches above. If a value
  somehow falls outside the enum, render the `fileserver` (neutral) branch as the fallback so the
  cell is never blank — mirror the `_status_pill.html` if/elif structure.

### Placement — new "Kind" column

**Decision (resolves the open question flagged in 48-RESEARCH.md §Open Questions 3):**
a new dedicated **"Kind" column**, positioned **after "Agent", before "Status".**

Header order becomes: `Agent | Kind | Status | Queue | Last seen | Scan roots | Actions`.

Rationale:
- The table is already strictly columnar (one attribute per column). Inlining the badge into the
  "Agent" cell — which already stacks name + monospaced id on two lines — would crowd that cell
  and break vertical scannability.
- A dedicated column lets the operator run their eye straight down one column to count cloud
  capacity ("how many COMPUTE agents are up?"), which is the exact CLOUDAGENT-03 goal.
- Placing Kind immediately left of Status pairs the two attributes the operator reads together
  ("which kind, and is it alive?") without the indigo badge and the status pill sharing a cell.

Column header cell matches existing `<th>`: `scope="col" class="px-4 py-3"`, text "Kind".
Body cell matches existing `<td class="px-4 py-3">` and `{% include "admin/partials/_kind_badge.html" %}`.

## Accessibility

- Each kind badge carries an `aria-label` (`Kind: compute` / `Kind: file server`) so screen
  readers announce the attribute, mirroring `_status_pill.html`'s `aria-label="Status: alive"`.
- The new column gets a proper `<th scope="col">` so the badge cell is associated with the "Kind"
  header.
- Meaning is carried by the visible uppercase label, not color alone (colorblind-safe).
- No change to the existing skip-link, `aria-live="polite"` region, or `role="alert"` banner.
