    **Endpoint 1: POST `/{tracklist_id}/match-discogs`** (per D-01, D-03)
    - Enqueue SAQ task: `await queue.enqueue("match_tracklist_to_discogs", tracklist_id=str(tracklist_id))`
    - Return the tracklist_card partial with a `match_queued=True` context flag
    - The template will show "Matching..." indicator when this flag is set

    **Endpoint 2: GET `/{tracklist_id}/tracks/{track_id}/discogs`** (per D-04, D-05)
    - Query DiscogsLink where `track_id == track_id` and `status != "dismissed"`, order by confidence DESC
    - Return `discogs_candidates.html` partial with candidates list
    - If no candidates, return partial with empty state message

    **Endpoint 3: POST `/discogs-links/{link_id}/accept`** (per D-07)
    - Load DiscogsLink by link_id
    - Set its status to "accepted"
    - Set ALL other DiscogsLink rows with the same track_id to status="dismissed" (auto-dismiss siblings)
    - Commit
    - Re-query remaining candidates for this track_id (should be just the accepted one)
    - Return updated `discogs_candidates.html` partial

    **Endpoint 4: DELETE `/discogs-links/{link_id}`** (per D-07)
    - Load DiscogsLink by link_id
    - Set status to "dismissed"
    - Commit
    - Re-query remaining candidates for the same track_id
    - Return updated `discogs_candidates.html` partial (the dismissed row will be gone)

    **Endpoint 5: POST `/{tracklist_id}/bulk-link`** (per D-11, D-12)
    - Load all DiscogsLink candidates for tracks in this tracklist's latest version
    - For each track that has candidates: find the highest-confidence candidate, set it to "accepted", dismiss the rest
    - Commit
    - Reload and return the full tracks partial (track_detail.html) so the whole tracks section refreshes

    Add tests to `tests/test_routers/test_tracklists.py`:

    - Test accept sets status="accepted" and dismisses siblings
    - Test dismiss sets status="dismissed"
    - Test bulk-link accepts top candidate per track
    - Test bulk-link with no candidates returns gracefully
