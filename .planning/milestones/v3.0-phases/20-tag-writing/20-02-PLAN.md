    **Router** `src/phaze/routers/tags.py` -- follow tracklists.py and proposals.py patterns:

    ```python
    router = APIRouter(prefix="/tags", tags=["tags"])
    ```

    Endpoints:

    1. **`GET /tags/`** (per D-04) -- Tag review list page
       - Query EXECUTED files joined with FileMetadata and left-join TagWriteLog (latest per file)
       - For each file, compute proposed tags via `compute_proposed_tags(file_metadata, tracklist, filename)`
       - Find associated tracklist: query Tracklist where file_id matches
       - Determine status: if TagWriteLog exists with status="completed" -> completed; "discrepancy" -> discrepancy; else "pending"
       - Count changed fields (proposed differs from current metadata)
       - Paginate with `page` and `page_size` query params (default 20, reuse Pagination pattern)
       - Stats: count pending, completed, discrepancy files
       - context: `{"request": request, "current_page": "tags", "files": files, "stats": stats, "pagination": pagination}`
       - HTMX partial detection: if `request.headers.get("HX-Request")` truthy, return `tags/partials/tag_list.html`; else return `tags/list.html`

    2. **`GET /tags/{file_id}/compare`** -- Expand tag comparison (per D-05)
       - Load FileRecord + FileMetadata + associated Tracklist
       - Compute proposed via `compute_proposed_tags`
       - Build comparison list: for each of 6 CORE_FIELDS, dict with field name, current value (from FileMetadata or None), proposed value (from computed or None), changed flag (current != proposed)
       - Return `tags/partials/tag_comparison.html` with `file`, `comparison`, and `proposed_tags` context
       - Include display labels: {"artist": "Artist", "title": "Title", "album": "Album", "year": "Year", "genre": "Genre", "track_number": "Track #"}

    3. **`GET /tags/{file_id}/edit/{field}`** -- Inline edit activation (per D-07)
       - Load current proposed value for the field
       - Return `tags/partials/inline_edit.html` with file_id, field, value

    4. **`PUT /tags/{file_id}/edit/{field}`** -- Save inline edit (per D-07)
       - Accept form data with field value
       - Store edited value in session (or pass through -- the edited values are sent with the write action)
       - Return `tags/partials/inline_display.html` with updated value and changed flag
       - NOTE: Inline edits are transient. The comparison panel tracks proposed values client-side. On write, the current DOM values are sent.

    5. **`POST /tags/{file_id}/write`** -- Execute tag write (per D-10)
       - Accept form data with all 6 field values (the potentially-edited proposed values)
       - Load FileRecord, validate state == EXECUTED
       - Determine source: "manual_edit" if any field was changed from computed proposal, else derive from which source contributed most
       - Call `execute_tag_write(session, file_record, tags_dict, source)`
       - Return updated file row HTML partial with new status badge and toast notification
       - Toast: "Tags written to {filename}" on success, error message on failure (per UI-SPEC copywriting)
