Create Alembic migration `009_add_search_vectors.py` with revision `"009"`, down_revision `"008"`.

The `upgrade()` function must execute these raw SQL statements in order:

1. Enable pg_trgm extension:
   ```sql
   CREATE EXTENSION IF NOT EXISTS pg_trgm
   ```

2. Add GENERATED STORED tsvector column to `files` table:
   ```sql
   ALTER TABLE files ADD COLUMN search_vector tsvector
   GENERATED ALWAYS AS (
       to_tsvector('simple', coalesce(original_filename, ''))
   ) STORED
   ```

3. Add GENERATED STORED tsvector column to `metadata` table:
   ```sql
   ALTER TABLE metadata ADD COLUMN search_vector tsvector
   GENERATED ALWAYS AS (
       to_tsvector('simple',
           coalesce(artist, '') || ' ' ||
           coalesce(title, '') || ' ' ||
           coalesce(album, '') || ' ' ||
           coalesce(genre, '')
       )
   ) STORED
   ```

4. Add GENERATED STORED tsvector column to `tracklists` table:
   ```sql
   ALTER TABLE tracklists ADD COLUMN search_vector tsvector
   GENERATED ALWAYS AS (
       to_tsvector('simple',
           coalesce(artist, '') || ' ' ||
           coalesce(event, '')
       )
   ) STORED
   ```

5. Create GIN indexes on search_vector columns:
   ```sql
   CREATE INDEX ix_files_search_vector ON files USING gin(search_vector)
   CREATE INDEX ix_metadata_search_vector ON metadata USING gin(search_vector)
   CREATE INDEX ix_tracklists_search_vector ON tracklists USING gin(search_vector)
   ```

6. Create GIN trigram indexes for ILIKE partial matching fallback:
   ```sql
   CREATE INDEX ix_files_filename_trgm ON files USING gin(original_filename gin_trgm_ops)
   CREATE INDEX ix_metadata_artist_trgm ON metadata USING gin(artist gin_trgm_ops)
   CREATE INDEX ix_tracklists_artist_trgm ON tracklists USING gin(artist gin_trgm_ops)
   ```

The `downgrade()` function must drop all indexes then drop the `search_vector` columns, and drop the pg_trgm extension.

Follow the exact migration pattern from `008_add_tracklist_source_status_confidence.py` for docstring, imports, and revision identifier format.

IMPORTANT: Do NOT add `search_vector` as a mapped column on the ORM models. The GENERATED ALWAYS columns exist only in the database. In SQLAlchemy queries, reference them via `column("search_vector")` from `sqlalchemy` or `text()` fragments. The test conftest uses `Base.metadata.create_all` which won't create GENERATED columns -- tests must handle this (see Task 2).

2. `async def search(session, query, *, artist, genre, date_from, date_to, bpm_min, bpm_max, file_state, page, page_size) -> tuple[list[SearchResult], Pagination]`:
   - Build `plainto_tsquery("simple", query)` for FTS
   - File subquery: SELECT from `files` LEFT JOIN `metadata` ON file_id, LEFT JOIN `analysis` ON file_id. Match against `files.search_vector @@ ts_query OR metadata.search_vector @@ ts_query`. Select id, literal "file" as result_type, original_filename as title, metadata.artist, metadata.genre, files.state, files.created_at as date, ts_rank as rank.
   - Tracklist subquery: SELECT from `tracklists`. Match against `tracklists.search_vector @@ ts_query`. Select id, literal "tracklist" as result_type, event as title, artist, NULL as genre, status as state, date, ts_rank as rank.
   - Apply facet filters to appropriate subqueries: artist filter (ILIKE on metadata.artist for files, ILIKE on tracklists.artist for tracklists), genre filter (ILIKE on metadata.genre, files only), date_from/date_to (created_at for files, date for tracklists), bpm_min/bpm_max (JOIN analysis for files, skip tracklists), file_state (files.state = value, skip tracklists).
   - UNION ALL both subqueries into a combined subquery.
   - Count total results via `select(func.count()).select_from(combined)`.
   - Apply ORDER BY rank DESC, LIMIT/OFFSET for pagination.
   - Return `(list[SearchResult], Pagination)`.

   IMPORTANT: Since `search_vector` is a GENERATED column not on the ORM model, reference it via `column("search_vector").cast(TSVECTOR)` from sqlalchemy imports, or use `text()` fragments within the WHERE clause. Use `func.to_tsvector` and `func.plainto_tsquery` from `sqlalchemy.func`.

   For test compatibility: Since `Base.metadata.create_all` does NOT create GENERATED columns, the test version must use expression-based FTS instead. This works the same as the generated column but computes at query time -- acceptable for test data sizes.
