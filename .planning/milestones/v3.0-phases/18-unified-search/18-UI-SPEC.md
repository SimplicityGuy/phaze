## Component Inventory

### 1. Search Input

- Full-width text input at top of page
- Placeholder: `"Search files and tracklists..."`
- Tailwind: `w-full rounded-md border border-gray-300 px-4 py-2 text-sm focus:ring-2 focus:ring-blue-500 focus:border-blue-500`
- Behavior: Form submit via Enter key or Search button (D-03). NOT live search-as-you-type.
- HTMX: `hx-get="/search/"` with `hx-target="#search-results"` and `hx-swap="innerHTML"` and `hx-push-url="true"`
- Loading indicator: `hx-indicator="#search-spinner"` showing a `"Searching..."` text with opacity transition

### 2. Search Button

- Inline with search input (right side or below)
- Label: `"Search"`
- Tailwind: `bg-blue-600 hover:bg-blue-700 text-white text-sm font-semibold px-4 py-2 rounded-md`

### 3. Advanced Filters Panel (D-04)

- Toggle button: `"Advanced filters"` / `"Hide filters"` text link
- Tailwind for toggle: `text-sm text-blue-600 hover:text-blue-800`
- Panel: Collapsed by default. Alpine.js `x-data="{ showFilters: false }"` with `x-show` and `x-transition`
- Panel container: `mt-2 p-4 border border-gray-200 rounded-lg bg-gray-50`
- Filter fields layout: CSS grid, 3 columns on desktop (`grid grid-cols-3 gap-4`), stack on mobile
- Filter fields:
  - Artist: text input, `name="artist"`, placeholder `"Filter by artist..."`
  - Genre: text input, `name="genre"`, placeholder `"Filter by genre..."`
  - Date range: two date inputs (`name="date_from"`, `name="date_to"`), labels `"From"` / `"To"`
  - BPM range: two number inputs (`name="bpm_min"`, `name="bpm_max"`), labels `"BPM min"` / `"BPM max"`
  - File state: `<select>` dropdown with options: All, Ingested, Analyzed, Proposed, Approved, Rejected, Executed
- All filter inputs use same styling as search input: `rounded-md border border-gray-300 px-3 py-2 text-sm`
- All filters are included in the HTMX form submission via `hx-include`

### 4. Summary Counts (D-06, initial state)

- Displayed when no query has been entered
- Layout: horizontal flex with gaps, centered
- Example: `"200,142 files"` and `"45 tracklists"` as two stat blocks
- Stat block: large number (`text-2xl font-semibold text-gray-900`) with label below (`text-sm text-gray-500`)

### 5. Results Table (D-01)

- Dense table matching proposals table pattern
- Table container: no border, full width
- Header row: `text-xs font-semibold text-gray-500 uppercase tracking-wide` with `border-b border-gray-200`
- Columns:

| Column | Width | Content | Tailwind |
|--------|-------|---------|----------|
| Type | `w-24` | Entity type pill badge | `text-xs font-semibold px-2 py-0.5 rounded-full` |
| Name | flex-1 | Filename or event name, truncated at 60 chars | `text-sm text-gray-900 truncate` |
| Artist | `w-40` | Artist name, truncated at 30 chars | `text-sm text-gray-700 truncate` |
| Genre | `w-28` | Genre or `"--"` for tracklists | `text-sm text-gray-500` |
| State | `w-24` | State pill badge (reuse existing pattern) | Same as proposal/tracklist badges |
| Date | `w-28` | `YYYY-MM-DD` format | `text-sm text-gray-400` |

- Row: `border-b border-gray-100 hover:bg-gray-50`
- Row cells: `px-3 py-3`
- Row click: navigates to the entity detail page (files -> `/proposals/?q={filename}`, tracklists -> `/tracklists/#{id}`)

### 6. No Results State (D-07)

- Centered block: `text-center py-16`
- Heading: `text-xl font-semibold text-gray-700 mb-2`
- Body: `text-gray-500`

### 7. Pagination

- Reuse existing pagination pattern from `proposals/partials/pagination.html`
- Three sections: left (showing range), center (page numbers), right (page size selector)
- All query parameters (q, artist, genre, date_from, date_to, bpm_min, bpm_max, file_state) preserved in pagination links
- Page sizes: 25, 50, 100
- Default page size: 50

### 8. Loading Indicator

- Element ID: `search-spinner`
- Content: `"Searching..."` text
- Tailwind: `text-sm text-gray-400 py-4 text-center`
- Uses HTMX indicator pattern: `htmx-indicator` class (opacity 0 -> 1 on `htmx-request`)

## Interaction Contracts

### Search Flow

1. User lands on `/search/` -- sees page heading, search box, and summary counts (D-06)
2. User types query and presses Enter or clicks Search button (D-03)
3. HTMX swaps `#search-results` with results table (or no-results message)
4. URL updates via `hx-push-url="true"` to `/search/?q={query}`
5. User can expand "Advanced filters" panel, fill in facets, and re-submit

### Filter Flow

1. User clicks "Advanced filters" toggle
2. Alpine.js reveals filter panel with `x-transition`
3. User fills in any combination of facet filters
4. User presses Enter or clicks Search -- all filters included in request
5. Results reflect combined query + filter narrowing
6. Filter state preserved in URL query parameters and pagination links

### Pagination Flow

1. After search, pagination appears below results table
2. Page navigation via HTMX replaces `#search-results` content
3. All query parameters (q + all filters) preserved in pagination URLs
4. Page size selector allows switching between 25/50/100 rows

### Navigation

- Search tab is FIRST (leftmost) in nav bar (D-05)
- `current_page = 'search'` for active tab highlighting
- Skip link target: `#search-results` (update from `#proposals-table` in base.html block override)

### Accessibility

- Search input has `role="search"` on containing `<form>`
- Results table uses proper `<table>`, `<thead>`, `<tbody>` semantics
- Badge text is readable (not color-only -- text labels present)
- Skip link overridden to target `#search-results`
- Filter panel uses `aria-expanded` attribute on toggle button
- Loading state communicated via `aria-busy="true"` on results container
