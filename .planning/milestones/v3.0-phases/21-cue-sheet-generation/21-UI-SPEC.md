## Component Inventory

### CUE Management Page (`/cue`)

**Stats Header** (3-column grid, matches `tags/list.html` pattern):

| Card | Background | Text Color | Label |
|------|-----------|------------|-------|
| Eligible | `bg-blue-50` | `text-blue-700` | "Eligible" |
| Generated | `bg-green-50` | `text-green-700` | "Generated" |
| Missing Timestamps | `bg-yellow-50` | `text-yellow-700` | "Missing Timestamps" |

**Tracklist List Table:**
- Each row shows: tracklist artist/event, track count, CUE status badge, action button
- CUE status badges:
  - No CUE: `bg-gray-100 text-gray-600` with text "No CUE"
  - Generated: `bg-emerald-100 text-emerald-700` with text "v1" / "v2" etc.
  - Not eligible: `bg-yellow-100 text-yellow-700` with text "Not Eligible"
- Action column: "Generate CUE" button or "Regenerate" link

**Batch Generation:**
- "Generate All Eligible" button at top of list, styled as primary CTA: `bg-green-600 hover:bg-green-700 text-white font-semibold text-xs px-3 py-1.5 rounded-md`
- HTMX POST to batch endpoint, targets list container for full refresh
- Progress feedback via OOB toast showing count: "Generated 12 CUE files"

### Tracklist Detail Inline Button

**Generate CUE Button** (in tracklist_card.html action row, alongside "Match to Discogs"):
- Condition: `tracklist.status == 'approved' and tracklist.file_id`
- Style: `text-xs bg-green-600 hover:bg-green-700 text-white font-semibold px-3 py-1.5 rounded-md`
- HTMX: `hx-post="/cue/{tracklist_id}/generate"` targeting tracklist card for outerHTML swap
- On success: card re-renders with CUE status badge + OOB toast

**CUE Status Badge** (inline on tracklist card, next to source/status badges):
- Only shown when CUE has been generated
- Style: `text-xs font-semibold px-2 py-0.5 rounded-full bg-emerald-100 text-emerald-700`
- Text: "CUE v1" / "CUE v2" etc.

### Navigation Tab

- Position: after "Tags" tab, before "Audit Log" tab
- Label: "CUE Sheets"
- Active state: `text-blue-600` (matches all other tabs)
- Inactive state: `text-gray-600 hover:text-gray-900` (matches all other tabs)
- `current_page` value: `'cue'`

### Toast Notifications

Follow existing pattern from `tags/partials/tag_row.html`:
- Container: `#toast-container` (already in base.html)
- Style: `bg-gray-800 text-white text-sm px-4 py-2 rounded shadow-lg`
- Auto-dismiss: 5 seconds via Alpine.js `x-init="setTimeout(() => show = false, 5000)"`
- Delivery: HTMX OOB swap `hx-swap-oob="beforeend:#toast-container"`

## Interaction Contracts

### Generate CUE (Single Tracklist)

| Step | Trigger | Target | Swap | Response |
|------|---------|--------|------|----------|
| 1. Click "Generate CUE" | `hx-post="/cue/{id}/generate"` | `#tracklist-{id}` | `outerHTML` | Updated tracklist card with CUE badge |
| 2. Toast notification | OOB swap | `#toast-container` | `beforeend` | "CUE file generated: filename.cue" |

**Error states:**
- File not EXECUTED: button disabled or hidden; if POST attempted, return toast: "File must be executed before generating CUE"
- No timestamps available: return toast: "No tracks have timestamps -- cannot generate CUE"
- Filesystem write failure: return toast: "Failed to write CUE file: [error detail]"

### Generate All Eligible (Batch)

| Step | Trigger | Target | Swap | Response |
|------|---------|--------|------|----------|
| 1. Click "Generate All Eligible" | `hx-post="/cue/generate-batch"` | `#cue-list-container` | `innerHTML` | Updated list with new CUE statuses |
| 2. Toast notification | OOB swap | `#toast-container` | `beforeend` | "Generated N CUE files" |

### CUE Management Page Load

| Step | Trigger | Target | Swap | Response |
|------|---------|--------|------|----------|
| 1. Navigate to /cue | full page load | -- | -- | Full page with stats + tracklist list |
| 2. Pagination | `hx-get="/cue?page=N"` | `#cue-list-container` | `innerHTML` | Updated list partial |

## State Matrix

| Tracklist State | File State | Has Timestamps | CUE Exists | UI Behavior |
|----------------|------------|----------------|------------|-------------|
| approved | EXECUTED | yes | no | Show "Generate CUE" button (green) |
| approved | EXECUTED | yes | yes (v1+) | Show "Regenerate CUE" button (text link) + "CUE v{N}" badge |
| approved | EXECUTED | no | no | Show "Not Eligible" badge (yellow), no button |
| approved | not EXECUTED | any | no | Show "Not Eligible" badge (yellow), no button |
| not approved | any | any | no | No CUE controls shown |
