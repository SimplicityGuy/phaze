  <name>CUE Sheet Generator Service</name>
  <files>src/phaze/services/cue_generator.py, tests/test_services/test_cue_generator.py</files>
  <behavior>
    ## Timestamp Conversion (CUE-02, D-01)
    - seconds_to_cue_timestamp(0.0) -> "00:00:00"
    - seconds_to_cue_timestamp(5.5) -> "00:05:37" (5s = 375 frames, 0.5s = 37 frames, total 412 frames; 412 % 75 = 37, 412 // 75 = 5s, 5 // 60 = 0m, 5 % 60 = 5s)
    - seconds_to_cue_timestamp(332.45) -> "05:32:33" (332.45 * 75 = 24933.75, int = 24933; 24933 % 75 = 33, 24933 // 75 = 332s, 332 // 60 = 5m, 332 % 60 = 32s)
    - seconds_to_cue_timestamp(3661.0) -> "61:01:00" (MM can exceed 59)
    - Frame values never reach 75 (range 00-74)

    ## Timestamp Parsing
    - parse_timestamp_string("1:30:00") -> 5400.0 (HH:MM:SS)
    - parse_timestamp_string("5:32") -> 332.0 (MM:SS)
    - parse_timestamp_string("332.45") -> 332.45 (raw seconds)
    - parse_timestamp_string(None) -> None (returns None for missing)

    ## CUE Content Generation (CUE-01, D-02, D-03)
    - generate_cue_content produces valid CUE with FILE, TRACK, INDEX commands
    - Tracks ordered by position, numbered 01, 02, etc.
    - Tracks with no timestamp (None) are omitted entirely (D-03)
    - FILE command uses audio filename only, not path (D-11)
    - First line is REM COMMENT "Generated by Phaze"

    ## Discogs REM Comments (CUE-03, D-08, D-09)
    - Tracks with genre/label/year get REM GENRE, REM LABEL, REM YEAR after TRACK line
    - Tracks without Discogs data get no REM lines
    - REM values in double quotes

    ## FILE Type Mapping
    - "mp3" -> "MP3"
    - "wav" -> "WAVE"
    - "aiff" -> "AIFF"
    - "flac" -> "WAVE"
    - "ogg" -> "WAVE"
    - "m4a" -> "WAVE"
    - "opus" -> "WAVE"
    - Unknown extension -> "WAVE" (safe default)

    ## Version Suffix Naming (D-12)
    - First CUE: "audio.cue"
    - Second CUE when "audio.cue" exists: "audio.v2.cue"
    - Third CUE when "audio.cue" and "audio.v2.cue" exist: "audio.v3.cue"

    ## File Writing (CUE-02, D-13)
    - Written file starts with UTF-8 BOM bytes (EF BB BF)
    - Content matches generate_cue_content output
    - write_cue_file returns the Path of the written file

    ## Special Characters (Pitfall 4)
    - Double quotes in filenames are stripped before embedding in FILE command
  </behavior>
  <implementation>
    Create `src/phaze/services/cue_generator.py` with:

    1. `CueTrackData` dataclass with fields: position (int), title (str | None), artist (str | None), timestamp_seconds (float | None), genre (str | None), label (str | None), year (int | None)

    2. `_FILE_TYPE_MAP: dict[str, str]` mapping extensions to CUE FILE type keywords:
       - "mp3" -> "MP3", "wav" -> "WAVE", "wave" -> "WAVE", "aiff" -> "AIFF", "aif" -> "AIFF"
       - "flac" -> "WAVE", "ogg" -> "WAVE", "m4a" -> "WAVE", "opus" -> "WAVE"

    3. `seconds_to_cue_timestamp(total_seconds: float) -> str`:
       - total_frames = int(total_seconds * 75)
       - frames = total_frames % 75
       - total_secs = total_frames // 75
       - minutes = total_secs // 60
       - seconds = total_secs % 60
       - return f"{minutes:02d}:{seconds:02d}:{frames:02d}"

    4. `parse_timestamp_string(ts: str | None) -> float | None`:
       - None -> None
       - "HH:MM:SS" -> hours*3600 + minutes*60 + seconds (3 parts)
       - "MM:SS" -> minutes*60 + seconds (2 parts)
       - Otherwise -> float(ts) (raw seconds)

    5. `generate_cue_content(audio_filename: str, file_type: str, tracks: list[CueTrackData]) -> str`:
       - Strip double quotes from audio_filename
       - Build lines list starting with REM COMMENT "Generated by Phaze"
       - FILE "{audio_filename}" {_FILE_TYPE_MAP.get(file_type.lower(), "WAVE")}
       - Filter tracks: skip any with timestamp_seconds is None (per D-03)
       - Sort remaining by position
       - For each track (numbered starting at 1):
         - TRACK {nn:02d} AUDIO
         - If genre: REM GENRE "{genre}" (4-space indent)
         - If label: REM LABEL "{label}" (4-space indent)
         - If year: REM YEAR "{year}" (4-space indent)
         - If title: TITLE "{title}" (4-space indent)
         - If artist: PERFORMER "{artist}" (4-space indent)
         - INDEX 01 {seconds_to_cue_timestamp(timestamp_seconds)} (4-space indent)
       - Join with newline, trailing newline
       - TRACK and its children indented with 2 spaces; REM/TITLE/PERFORMER/INDEX indented with 4 spaces

    6. `next_cue_path(audio_path: Path) -> Path`:
       - base_cue = audio_path.parent / f"{audio_path.stem}.cue"
       - If not exists, return base_cue
       - Scan parent dir for pattern {stem}.v{N}.cue, find max N
       - Return parent / f"{stem}.v{max_version + 1}.cue"
       - If base_cue exists but no .vN.cue files, return .v2.cue (base counts as v1)

    7. `write_cue_file(content: str, audio_path: Path) -> Path`:
       - path = next_cue_path(audio_path)
       - Open with encoding="utf-8-sig" and write content
       - Return path
  </implementation>
