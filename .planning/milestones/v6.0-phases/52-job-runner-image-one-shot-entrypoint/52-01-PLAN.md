<objective>
Define the two contracts the one-shot entrypoint consumes: (1) a shared, Postgres-free
`analysis_wire` module holding the mood/style feature-to-dict converters so both the SAQ
`process_file` task and the new `job_runner` import them from one place; (2) a
`request_download_url(file_id) -> (url, expected_sha256)` method on `PhazeAgentClient` plus
the response schema it deserializes.

This is interface-first work: Plan 02 (`job_runner`) builds against these contracts, so they
must exist and be exact before that plan runs. The presign endpoint's SERVER side is Phase 53;
here we only define + unit-test the CLIENT method (respx-mocked) and the consumed response
shape (KJOB-02, RESEARCH Q3 / Pitfall 3).

Purpose: Eliminate the "scavenger hunt" ŌĆö Plan 02 receives ready-made helpers and a typed
client method instead of re-deriving them.
Output: `analysis_wire.py`, an updated `functions.py` import, the `request_download_url`
method, a `PresignDownloadResponse` schema, and client unit tests.
