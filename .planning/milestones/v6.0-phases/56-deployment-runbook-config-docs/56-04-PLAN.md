| cluster Secret → pod env | The bearer token rides a Secret; must not be logged or over-scoped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-RBAC | Elevation of Privilege | runbook RBAC Role | mitigate | namespaced Role with the exact verb floor (jobs create/get/delete; workloads list/get/watch; localqueues get); no cluster-wide grants; asserted by test_rbac_covers_call_graph |
| T-56-TOKEN | Information Disclosure | bearer-token Secret | mitigate | token in a core/v1 Secret consumed via PHAZE_AGENT_TOKEN_FILE (SecretStr, never logged); minted per the agents-add flow |
| T-56-NAME | Tampering | kube object names | mitigate | runbook uses DNS-1123-safe placeholder names; runtime Job names are deterministic UUID-based (no operator free-text enters object names) |
| T-56-VER | Integrity/Availability | apiVersion drift | mitigate | the lockstep rule (manifest == cluster == PHAZE_KUBE_WORKLOAD_API_VERSION) + v1beta2 upgrade note (Pitfall 1) |
| T-56-SC | Tampering | npm/pip/cargo installs | accept | zero new external packages |
</threat_model>
