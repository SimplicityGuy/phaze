Create the `services/audfprint/` and `services/panako/` directories with the following files:

**services/audfprint/Dockerfile.audfprint** (per D-02: Python 3.13 + uv):
- Base: `python:3.13-slim`
- System deps: `apt-get install -y --no-install-recommends ffmpeg git`
- Clone audfprint: `git clone --depth 1 https://github.com/dpwe/audfprint.git /app/audfprint`
- Copy uv from `ghcr.io/astral-sh/uv:latest`
- Copy `pyproject.toml` and run `uv sync --frozen --no-dev --no-install-project`
- Copy `app.py`
- Set `ENV UV_NO_SYNC=1`
- Create non-root user `audfprint`, set `USER audfprint`
- Create `/data/fprint` dir owned by audfprint user for database storage
- Expose 8001, CMD: `["uv", "run", "uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8001"]`

**services/audfprint/pyproject.toml**:
- Python 3.13 project with dependencies: `fastapi>=0.135.2`, `uvicorn>=0.34.0`, `numpy`, `scipy`, `docopt`, `joblib`, `psutil`, `librosa`, `standard-aifc`, `standard-sunau`
- Name: `audfprint-service`

**services/audfprint/app.py** (per D-07, D-08, D-09):
- FastAPI app with 3 endpoints:
  - `GET /health` -> returns `{"status": "healthy", "engine": "audfprint"}`
  - `POST /ingest` accepts JSON `{"file_path": str}`. Uses `asyncio.to_thread(subprocess.run, ...)` to call `python audfprint/audfprint.py add --dbase /data/fprint/fprint.pklz {file_path}`. Returns `{"status": "ingested", "file_path": file_path}` on success, 500 with stderr on failure. If the database file does not exist, create it first with `python audfprint/audfprint.py new --dbase /data/fprint/fprint.pklz`.
  - `POST /query` accepts JSON `{"file_path": str}`. Uses `asyncio.to_thread(subprocess.run, ...)` to call `python audfprint/audfprint.py match --dbase /data/fprint/fprint.pklz {file_path}`. Parse stdout for matches. Return `{"matches": [{"track_id": str, "confidence": float}]}` where confidence is normalized 0-100 (per D-10). Empty list if no match.
- Use `asyncio.Lock` for write operations (ingest) to prevent concurrent writes to the serialized database file (Research Pitfall 3). Reads (query) can be concurrent.
- Pydantic models for request/response bodies: `IngestRequest(file_path: str)`, `IngestResponse(status: str, file_path: str)`, `QueryMatch(track_id: str, confidence: float)`, `QueryResponse(matches: list[QueryMatch])`, `HealthResponse(status: str, engine: str)`.
- Type hints on all functions, double quotes, 150 char line length.

**services/panako/Dockerfile.panako** (per D-03: JRE slim + Python wrapper):
- Multi-stage build:
  - Stage 1 (`builder`): `eclipse-temurin:21-jdk-jammy`, install git, clone Panako, run `./gradlew shadowJar`
  - Stage 2: `python:3.13-slim`
    - Install ffmpeg
    - Copy JRE from `eclipse-temurin:21-jre-jammy` at `/opt/java/openjdk`
    - Set `PATH="/opt/java/openjdk/bin:$PATH"`
    - Copy built Panako JAR from builder stage
    - Copy uv, pyproject.toml, run `uv sync`
    - Copy `app.py`
    - Set `ENV UV_NO_SYNC=1` and `ENV HOME=/data/fprint` (Panako needs writable home for LMDB)
    - Create non-root user `panako`, create `/data/fprint` dir
    - Expose 8002, CMD: `["uv", "run", "uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8002"]`

**services/panako/pyproject.toml**:
- Python 3.13 project with dependencies: `fastapi>=0.135.2`, `uvicorn>=0.34.0`
- Name: `panako-service`

**services/panako/app.py** (per D-07, D-08, D-09):
- FastAPI app with 3 endpoints:
  - `GET /health` -> returns `{"status": "healthy", "engine": "panako"}`
  - `POST /ingest` accepts JSON `{"file_path": str}`. Uses `asyncio.to_thread(subprocess.run, ...)` to call `java -jar /app/panako.jar store {file_path}`. Returns `{"status": "ingested", "file_path": file_path}`.
  - `POST /query` accepts JSON `{"file_path": str}`. Uses `asyncio.to_thread(subprocess.run, ...)` to call `java -jar /app/panako.jar query {file_path}`. Parse semicolon-separated stdout (Research: "index; total; query path; query start; query end; match path; match ID; match start; match end; score; time factor; freq factor; match percentage"). Normalize to 0-100 confidence using match percentage field. Return `{"matches": [{"track_id": str, "confidence": float}]}`.
- Same Pydantic models as audfprint: `IngestRequest`, `IngestResponse`, `QueryMatch`, `QueryResponse`, `HealthResponse`.
- Type hints on all functions, double quotes, 150 char line length.

**README.md for each service**: Brief description, how to build, API endpoints, volume requirements. Keep concise (under 50 lines).

Add two new services and two new volumes to `docker-compose.yml` (per D-01, D-04, D-05):

**audfprint service:**
```yaml
audfprint:
  build:
    context: .
    dockerfile: services/audfprint/Dockerfile.audfprint
  volumes:
    - "${SCAN_PATH:-/data/music}:/data/music:ro"
    - audfprint_data:/data/fprint
  healthcheck:
    test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8001/health')"]
    interval: 10s
    timeout: 5s
    retries: 3
  restart: unless-stopped
```

**panako service:**
```yaml
panako:
  build:
    context: .
    dockerfile: services/panako/Dockerfile.panako
  volumes:
    - "${SCAN_PATH:-/data/music}:/data/music:ro"
    - panako_data:/data/fprint
  healthcheck:
    test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8002/health')"]
    interval: 10s
    timeout: 5s
    retries: 3
  restart: unless-stopped
```

**volumes section** -- add alongside existing pgdata:
```yaml
volumes:
  pgdata:
  audfprint_data:
  panako_data:
```

Both services: NO ports mapping (per D-05, internal network only). The worker service calls them by Docker service name (e.g., `http://audfprint:8001`).

Add `depends_on` to worker service so it waits for fingerprint containers:

```yaml
  depends_on:
    postgres:
      condition: service_healthy
    redis:
      condition: service_healthy

      condition: service_healthy
    panako:
      condition: service_healthy
```
