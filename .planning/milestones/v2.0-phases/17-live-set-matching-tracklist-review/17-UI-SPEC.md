## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between badge text and container, confidence badge padding, icon spacing |
| sm | 8px | Gap between action buttons (gap-2), compact padding, checkbox-to-label gap |
| md | 16px | Card padding (px-4 py-4), element spacing, default gaps, inline edit input padding |
| lg | 24px | Section spacing (space-y-6), stats header padding (p-6) |
| xl | 32px | Page vertical padding (py-8 in main) |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: none

Source: Phase 15 UI-SPEC spacing scale. Existing `tracklists/` templates use `px-4 py-4` for cards, `p-6` for stats header, `space-y-6` for layout, `gap-2` / `gap-3` for inline elements.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (bg-white) | Page background, card backgrounds, inline edit input background |
| Secondary (30%) | `#F9FAFB` (bg-gray-50) | Stats header background, hover states (hover:bg-gray-50), scan tab file list rows |

Accent reserved for:
- "Scan Selected Files" primary CTA button (bg-blue-600)
- Active filter tab text and border (text-blue-600 border-blue-600)
- "Approve Tracklist" button (bg-blue-600)
- Inline edit focus ring and border (focus:ring-blue-500 border-blue-300)
- Undo link in toast (text-blue-300)

Additional semantic colors (reused from Phase 15):
- `#16A34A` (text-green-600): Confidence scores >= 90%, "Approved" status badge background (bg-green-100 text-green-700)
- `#CA8A04` (text-yellow-600): Confidence scores 70-89%, "Proposed" status badge background (bg-yellow-100 text-yellow-700)
- `#DC2626` (text-red-600): Confidence scores < 70%, "Rejected" status badge background (bg-red-100 text-red-700)
- `#7C3AED` (bg-purple-100 text-purple-700): "Fingerprint" source badge
- `#2563EB` (bg-blue-100 text-blue-700): "1001Tracklists" source badge

Source: Phase 15 UI-SPEC color contract. D-09 specifies green/yellow/red confidence tiers.

### Inline Edit Field (D-12)

Display mode (default):
```html
<td class="py-2 pr-3 text-gray-900 cursor-pointer hover:bg-blue-50"
    hx-get="/tracklists/tracks/{id}/edit/{field}"
    hx-target="this"
    hx-swap="innerHTML">
    {value}
</td>
```

Edit mode (swapped in by HTMX):
```html
<input type="text"
       name="{field}"
       value="{value}"
       class="text-sm border border-blue-300 rounded px-2 py-1 w-full
              focus:outline-none focus:ring-1 focus:ring-blue-500 bg-white"
       hx-put="/tracklists/tracks/{id}/edit/{field}"
       hx-target="closest td"
       hx-swap="innerHTML"
       hx-trigger="blur, keyup[keyCode==13]"
       autofocus>
```

- Input: `text-sm border border-blue-300 rounded px-2 py-1 w-full focus:outline-none focus:ring-1 focus:ring-blue-500 bg-white`
- Transition: instant swap, no animation (keeps editing fast)

## Interaction Contract

### HTMX Interactions

| Trigger | Method | URL Pattern | Target | Swap |
|---------|--------|-------------|--------|------|
| Scan tab click | -- | -- | Alpine.js toggle (no HTMX) | Show `#scan-panel`, hide `#tracklists-list` |
| Proposed tab click | GET | `/tracklists/?filter=proposed&page=1` | `#tracklists-list` | innerHTML |
| Trigger scan | POST | `/tracklists/scan` | `#scan-panel` | innerHTML (shows progress) |
| Poll scan status | GET | `/tracklists/scan/status?job_id={id}` | `#scan-progress-{id}` | innerHTML (hx-trigger="every 3s") |
| Expand fingerprint card | GET | `/tracklists/{id}/tracks` | `#tracks-{id}` | innerHTML (returns fingerprint_track_detail) |
| Inline edit (get input) | GET | `/tracklists/tracks/{track_id}/edit/{field}` | caller `<td>` | innerHTML |
| Inline edit (save) | PUT | `/tracklists/tracks/{track_id}/edit/{field}` | closest `<td>` | innerHTML |
| Delete track | DELETE | `/tracklists/tracks/{track_id}` | closest `<tr>` | outerHTML (swap to empty) |
| Approve tracklist | POST | `/tracklists/{id}/approve` | `#tracklist-{id}` | outerHTML |
| Reject tracklist | POST | `/tracklists/{id}/reject` | `#tracklist-{id}` | outerHTML |
| Bulk reject low confidence | POST | `/tracklists/{id}/reject-low?threshold={n}` | `#tracks-{id}` | innerHTML |
| All existing Phase 15 interactions | -- | (unchanged) | -- | -- |

### Alpine.js Interactions

| Component | x-data | Behavior |
|-----------|--------|----------|
| Filter tabs (extended) | `{ activeTab: '{{ active_filter }}', showScan: false }` | When "Scan" clicked: `showScan=true`, hide `#tracklists-list`, show `#scan-panel`. Other tabs: `showScan=false`, show list, hide scan panel. |
| Scan file selection | `{ selectedFiles: [], selectAll: false }` | Checkboxes bound to `selectedFiles` array. "Select All" toggles all visible. Disabled state on "Scan Selected Files" when `selectedFiles.length === 0`. |
| Tracklist card (extended) | `{ expanded: false }` | Unchanged from Phase 15. |
| Toast | `{ show: true }` | Unchanged from Phase 15. 10-second auto-dismiss. |

### State Transitions

| User Action | Visual Feedback | Backend Effect |
|-------------|----------------|----------------|
| Click "Scan" tab | File selection panel replaces tracklist list | None (client-side toggle) |
| Select files and click "Scan Selected Files" | Button shows "Scanning..." spinner; scan progress panel appears | arq `scan_live_set` task enqueued per selected file |
| Scan completes | Progress indicator replaced with "Scan complete. N tracklists created." link | Tracklist + TracklistVersion + TracklistTrack rows created with `source='fingerprint'`, `status='proposed'` |
| Expand fingerprint tracklist card | Track table with confidence badges and inline edit loads | GET tracks from latest version |
| Click editable field (artist/title/timestamp) | Cell content swapped to input field with blue border | None until save |
| Blur or press Enter on edit input | Input swapped back to display text with updated value | PUT saves field to database |
| Click "Remove" on track row | Row fades out and is removed from table | DELETE track from TracklistTrack |
| Click "Approve Tracklist" | Status badge changes from "Proposed" (yellow) to "Approved" (green); action buttons update | Tracklist.status set to 'approved' |
| Click "Reject Tracklist" | Status badge changes to "Rejected" (red); card grayed out | Tracklist.status set to 'rejected' |
| Click "Reject Low Confidence" | Tracks below threshold removed from table; success toast | TracklistTrack rows below threshold deleted |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
