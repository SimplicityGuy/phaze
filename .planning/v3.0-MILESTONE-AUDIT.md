## E2E User Flows

### Flow 1: Search to Discovery
1. User visits `/search/` -- sees summary counts (files, tracklists, Discogs)
2. Enters query -- gets unified results with File (blue), Tracklist (green), Discogs (purple) badges
3. Applies facet filters -- results narrow by artist, genre, date, BPM, state
4. Clicks result -- navigates to file or tracklist detail
**Status:** Complete (code-verified, human confirmation needed for browser rendering)

### Flow 2: Discogs Linking
1. User views tracklist detail -- clicks "Match to Discogs"
2. SAQ task processes all tracks against discogsography API
3. User expands track rows -- sees candidates with confidence scores
4. User accepts/dismisses candidates (accept auto-dismisses siblings)
5. User can bulk-link all tracks in one action
6. Accepted links appear in unified search with purple badges
**Status:** Complete (code-verified, human confirmation needed for live service)

### Flow 3: Tag Writing
1. User visits `/tags/` -- sees EXECUTED files with pending tags
2. Expands file to see side-by-side comparison (current vs proposed)
3. Can inline-edit proposed values
4. Clicks "Write Tags" -- tags written to destination copy
5. System verifies by re-reading file -- logs to TagWriteLog
6. Row updates in-place showing completed status
**Status:** Complete (code-verified, human confirmation for HTMX OOB swap)

### Flow 4: CUE Generation
1. User visits `/cue/` -- sees eligible tracklists with source badges
2. Fingerprint-sourced tracklists sort first (indigo badge)
3. User clicks "Generate CUE" -- .cue file written with UTF-8 BOM
4. CUE includes correct MM:SS:FF timestamps and REM Discogs metadata
5. Button changes to "Regenerate CUE" with version badge
6. Can also generate from tracklist card directly
7. Batch generation available for all eligible tracklists
**Status:** Complete (code-verified, human confirmation for live filesystem)
