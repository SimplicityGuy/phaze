```python
# In tests/test_routers/test_proposals.py or tests/conftest.py:
import uuid
from phaze.models.file import FileRecord, FileState
from phaze.models.proposal import RenameProposal, ProposalStatus

async def create_test_proposal(
    session: AsyncSession,
    *,
    original_filename: str = "test_file.mp3",
    proposed_filename: str = "Artist - Track.mp3",
    confidence: float = 0.85,
    status: str = ProposalStatus.PENDING,
    reason: str = "Test reasoning",
) -> RenameProposal:
    """Create a FileRecord + RenameProposal pair for testing."""
    file_record = FileRecord(
        id=uuid.uuid4(),
        sha256_hash="a" * 64,
        original_path=f"/music/{original_filename}",
        original_filename=original_filename,
        current_path=f"/music/{original_filename}",
        file_type="music",
        file_size=1000000,
        state=FileState.PROPOSAL_GENERATED,
    )
    session.add(file_record)
    await session.flush()

    proposal = RenameProposal(
        id=uuid.uuid4(),
        file_id=file_record.id,
        proposed_filename=proposed_filename,
        confidence=confidence,
        status=status,
        context_used={"artist": "Test Artist", "event_name": "Test Event"},
        reason=reason,
    )
    session.add(proposal)
    await session.commit()
    return proposal
```

Write all tests from the behavior list above. Use `pytest.mark.asyncio` on all test functions. Use the `client` and `session` fixtures from conftest.py.

Key patterns:
- For list tests: seed data with `create_test_proposal(session, ...)`, then `response = await client.get("/proposals/")`
- For HTMX tests: pass `headers={"HX-Request": "true"}`
- For approve/reject: `response = await client.patch(f"/proposals/{proposal.id}/approve")`
- For bulk: `response = await client.patch("/proposals/bulk", data={"action": "approve", "proposal_ids": [str(id1), str(id2)]})`
- Verify DB state after mutations: `await session.refresh(proposal)` then assert `proposal.status == "approved"`
- For pagination: create 60 proposals in a loop with unique filenames
- For search: create proposals with specific filenames, search for a substring
- For sort: create proposals with different confidence values, verify order in response

Each test must assert:
1. HTTP status code (200 for success, 404 for not found)
2. Content-type header contains "text/html"
3. Response body contains expected strings (e.g., proposed filenames, status text, specific HTML elements)
4. Database state (for mutation tests)

**Important test notes:**
- The `client` fixture from conftest.py uses `ASGITransport` which does NOT invoke lifespan (per Phase 04 decision). No Redis mock needed.
- Template resolution: Jinja2Templates uses `pathlib.Path(__file__).parent.parent / "templates"` which resolves correctly from the package location, not CWD.
- Use `await session.get(RenameProposal, proposal.id)` to re-read after mutations.
- Each FileRecord needs a UNIQUE `original_path` (unique index exists).
