# Phase 7 — UI Design Contract

> Visual and interaction contract for the approval workflow UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS via CDN, no build step) |
| Preset | not applicable |
| Component library | none (hand-built with Tailwind utility classes) |
| Icon library | Heroicons via CDN (https://unpkg.com/heroicons@2.1.5/) inline SVG |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline badge padding |
| sm | 8px | Table cell vertical padding, compact element spacing |
| md | 16px | Table cell horizontal padding, form element spacing, card padding |
| lg | 24px | Section padding (stats bar, filter bar), page horizontal gutter |
| xl | 32px | Page vertical padding between major sections |
| 2xl | 48px | Top-of-page spacing above stats bar |
| 3xl | 64px | Not used in this phase |

Exceptions: Table row height minimum 44px for touch target accessibility (keyboard navigation rows).

Tailwind mapping: Use `p-1`(4px), `p-2`(8px), `p-4`(16px), `p-6`(24px), `p-8`(32px), `py-12`(48px). Do not use arbitrary values.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 12px | 600 (semibold) | 1.5 | `text-xs font-semibold leading-normal` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

Body at 14px chosen for data-dense table layout (scanning hundreds of filenames). Display at 28px reserved for page title only. Label at 12px for column headers, status badges, and count badges.

Weights: 400 (regular) for body text and table cell data. 600 (semibold) for headings, labels, column headers, buttons, and badges. No other weights.

### Destructive Actions

| Action | Approach |
|--------|----------|
| Reject single proposal | Instant action with 5-second undo toast (D-05). No confirmation dialog. |
| Bulk reject proposals | Instant action with 5-second undo toast showing count. No confirmation dialog. |

Rationale: Reject is reversible (status change, not data deletion). Undo toast provides safety net without slowing the review workflow. Source: D-05.

## Visual Hierarchy

**Primary visual anchor:** the proposal table (the work area where review happens).

**Eye flow:** stats bar (overview at a glance) → filter tabs (scope the view) → search box (find specific files) → proposal table (review and act) → pagination (navigate) → bulk action bar (batch operations, appears on selection).

## Component Inventory

Components the executor must build for this phase. All server-rendered Jinja2 templates with HTMX attributes.

| Component | Template File | HTMX Behavior | Alpine.js |
|-----------|--------------|----------------|-----------|
| Base layout | `templates/base.html` | None (full page shell) | None |
| Stats bar | `templates/partials/stats_bar.html` | OOB swap on approve/reject to update counts | None |
| Filter tabs | `templates/partials/filter_tabs.html` | `hx-get` loads table with status filter param | None |
| Search box | `templates/partials/search_box.html` | `hx-get` with `hx-trigger="keyup changed delay:300ms"` | None |
| Proposal table | `templates/partials/proposal_table.html` | Target for filter/search/pagination swaps | `x-data` for row selection state, keyboard nav |
| Table row | `templates/partials/proposal_row.html` | `hx-get` for expand; `hx-patch` for approve/reject | `x-data` for selected checkbox |
| Row detail panel | `templates/partials/row_detail.html` | Lazy-loaded via `hx-get` on expand click | None |
| Pagination | `templates/partials/pagination.html` | `hx-get` swaps table content | None |
| Bulk action bar | `templates/partials/bulk_actions.html` | `hx-patch` for bulk approve/reject | `x-show` visible when selections > 0 |
| Toast | `templates/partials/toast.html` | OOB swap injected on actions; undo via `hx-patch` | `x-data` for auto-dismiss (5s timeout) |
| Keyboard nav | (inline in proposal_table) | None | `x-data` with `@keydown` handlers |

### Page Size Default

50 items per page. Balances table density with page load performance for 200K proposals.

### Search Debounce

300ms debounce on search input keystrokes. Prevents excessive server requests while typing.

### Confidence Thresholds

| Level | Range | Color |
|-------|-------|-------|
| High | >= 0.8 | green-600 |
| Medium | >= 0.5, < 0.8 | yellow-600 |
| Low | < 0.5 | red-600 |

### Undo Mechanism

Immediate database write on approve/reject. Undo triggers a second `hx-patch` that reverts status to PENDING. Toast auto-dismisses after 5 seconds. After dismissal, undo is no longer available (status change is final).

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Arrow keys navigate rows, Enter/Space to expand, A/R shortcuts only when row focused |
| Focus visible | All interactive elements show visible focus ring (ring-2 ring-blue-500) |
| Color not sole indicator | Status badges include text labels ("Approved", "Rejected", "Pending"), not just color |
| Button labels | All buttons have explicit text labels (no icon-only buttons) |
| Table semantics | Use `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` for screen reader compatibility |
| ARIA live region | Toast container uses `aria-live="polite"` for screen reader announcement |
| Page title | `<title>` updates to reflect current filter (e.g., "Pending Proposals - Phaze") |
| Skip link | Hidden "Skip to proposals table" link at top of page |
