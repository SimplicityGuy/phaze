# Phase 29 — UI Design Contract

> Visual and interaction contract for the new `/admin/agents` page, the
> `admin/` template namespace, the agent status-pill component (5 states),
> the relative-time helper, and the "Agents" top-nav link. Generated by
> gsd-ui-researcher; verified by gsd-ui-checker.

Phase 29 is the **first admin/operator-facing page family** in the project.
It extends the existing Phaze design system (Tailwind v4 + Inter/Jura fonts
+ custom palette in `templates/base.html`) and introduces **no new visual
tokens** — every spacing, color, font-size, and pill-geometry decision below
is inherited from existing templates (`templates/pipeline/`,
`templates/execution/`, `templates/tracklists/`).

What Phase 29 **does** introduce, and what this contract locks:

1. The `templates/admin/` directory convention (page shell + `partials/`
   subdir) — establishes the namespace for future Phase 30+ admin pages.
2. A **5-state agent status pill** (`alive`, `stale`, `dead`, `revoked`,
   `never`) — the project's first multi-state liveness pill. Existing pills
   are 3-4 states (scan, proposal, execution); the agent pill ladder maps
   onto already-used color hues so no new palette additions land.
3. A **relative-time helper** (`23s ago`, `4m ago`, `2h ago`, `3d ago`) —
   first general-purpose use; the helper is locked to a single signature so
   future phases can reuse without re-deriving format rules.
4. A **never-halting HTMX poll** (mirrors Phase 27 D-08 cadence, but the
   `outerHTML` target is the poll element itself and the trigger is always
   present — no terminal-state markup).
5. An **opinionated empty state** + **failure-tolerant refresh** posture
   (last-known state stays on screen; transient HTMX failures do not blow
   up the page).
6. The **"Agents" top-nav link** positioned **far right** (operator/ops
   convention; CONTEXT.md D-discretion locks this rationale).

CONTEXT.md decisions D-11..D-14 are LOCKED. This UI-SPEC translates them
into prescriptive HTML/Tailwind classes the executor can apply directly.

## Status Pill Component (5 States)

The agent status pill is the central new component in Phase 29. It is
computed server-side from `agents.revoked_at`, `agents.last_seen_at`, and
`now()` per CONTEXT.md D-12 thresholds. Status thresholds and the priority
ladder are LOCKED.

| State | Trigger (CONTEXT.md D-12) | Pill text (UPPER) | Tailwind classes | `aria-label` |
|-------|---------------------------|-------------------|-------------------|--------------|
| revoked | `revoked_at IS NOT NULL` (takes precedence over all `last_seen_at` math) | `REVOKED` | `text-xs font-semibold px-2 py-0.5 rounded-full bg-gray-100 dark:bg-gray-800 text-gray-700 dark:text-gray-300` | `Status: revoked` |
| never | `revoked_at IS NULL` AND `last_seen_at IS NULL` | `NEVER` | (same as revoked) `bg-gray-100 dark:bg-gray-800 text-gray-700 dark:text-gray-300` | `Status: never seen` |
| alive | `now() - last_seen_at < 90s` | `ALIVE` | `text-xs font-semibold px-2 py-0.5 rounded-full bg-green-100 dark:bg-green-950 text-green-700 dark:text-green-400` | `Status: alive` |
| stale | `90s ≤ now() - last_seen_at < 300s` | `STALE` | `text-xs font-semibold px-2 py-0.5 rounded-full bg-amber-100 dark:bg-amber-950 text-amber-700 dark:text-amber-400` | `Status: stale` |
| dead | `now() - last_seen_at ≥ 300s` | `DEAD` | `text-xs font-semibold px-2 py-0.5 rounded-full bg-red-100 dark:bg-red-950 text-red-700 dark:text-red-400` | `Status: dead` |

**Pill text is uppercase and short (≤7 chars)** — mirrors the project-wide
pill text style across `tracklists/partials/status_badge.html`,
`pipeline/partials/scan_status_pill.html`, and
`execution/partials/agents_table.html`.

**Pill geometry is non-negotiable:** every pill across Phaze uses
`text-xs font-semibold px-2 py-0.5 rounded-full`. Phase 29 does not deviate.

**Reusable partial:** the pill MUST live in
`templates/admin/partials/_status_pill.html` (the leading underscore is the
Phaze convention for partials that other partials include — see
`templates/_partials/`). The partial accepts one context variable
`status: Literal["alive","stale","dead","revoked","never"]` and renders the
matching `<span>`. The agents table row includes it via
`{% include "admin/partials/_status_pill.html" %}`. This isolates the
status-pill HTML so future pages reuse it without copy-paste.

**Server-side classification:** the route handler attaches `agent._status`
to each row before rendering (transient ORM attribute, same pattern as
Phase 27's `_agent_name` / `_elapsed_seconds`). The classifier function
lives in `phaze.services.agent_liveness.classify(agent: Agent, now: datetime) -> str`
and consumes `phaze.constants.AGENT_LIVENESS_ALIVE_SECONDS = 90`,
`AGENT_LIVENESS_STALE_SECONDS = 300` so tests and renderer share one
source of truth.

**Accessibility:** color alone is not enough. Every pill carries both the
status text inside the `<span>` AND a redundant `aria-label="Status: <state>"`
matching the table above. Screen readers announce both. Mirrors the
established pattern in `execution/partials/agents_table.html` line 44.

## Relative-Time Helper (`23s ago`)

CONTEXT.md D-discretion locks the helper module location:
`phaze.utils.humanize.relative_time(dt: datetime, *, now: datetime | None = None) -> str`.

Pure Python, no extra dep. The optional `now=` kwarg makes the helper
deterministic for unit testing (mirror Phase 27 `_elapsed_seconds` pattern).

**Output rules (LOCKED):**

| Delta from now | Output | Example |
|----------------|--------|---------|
| `dt is None` | `"never"` | — (matches the `never` pill state) |
| `delta < 0` (future timestamp; clock skew) | `"just now"` | — |
| `0 ≤ delta < 60s` | `"{int(delta)}s ago"` | `"23s ago"` |
| `60s ≤ delta < 3600s` | `"{int(delta / 60)}m ago"` | `"4m ago"` |
| `3600s ≤ delta < 86400s` | `"{int(delta / 3600)}h ago"` | `"2h ago"` |
| `delta ≥ 86400s` | `"{int(delta / 86400)}d ago"` | `"3d ago"` |

**Format invariants:**

- No leading zero (`"5s ago"`, not `"05s ago"`).
- No plural-s suffix (`"1s ago"`, not `"1 second ago"`). Compact + glanceable.
- Single-character unit suffix (`s`/`m`/`h`/`d`), space before `ago`.
- Always int-truncates toward zero (not rounds). `89.7s` → `"89s ago"`,
  NOT `"1m ago"`.
- No fractional units (`90s` → `"1m ago"`, not `"1.5m ago"`).

**Where used in Phase 29:** the `Last seen` column on the agents table.

**Why not `humanize` (the PyPI lib):** not in `pyproject.toml`; CONTEXT.md
notes "no new pip dependencies." A 12-line helper is cheaper than a new
dep + version pin + supply-chain review.

**Test surface:** `tests/test_utils/test_humanize.py` covers all five
output rules + the `None` case + the negative-delta (future) case + the
boundary cases (`59s`, `60s`, `3599s`, `3600s`, `86399s`, `86400s`). Live
under D-22's test list.

## Interaction Contract

### Polling

| Property | Value | Source |
|----------|-------|--------|
| HTMX trigger | `hx-trigger="every 5s"` | CONTEXT.md D-13 (LOCKED) |
| HTMX swap mode | `hx-swap="outerHTML"` | CONTEXT.md D-13 (LOCKED) |
| HTMX target endpoint | `/admin/agents/_table` | CONTEXT.md D-11 — partial route |
| HTMX target element | the polling element IS the swap target (the partial replaces itself) | mirrors Phase 27 D-08 pattern from `pipeline/dashboard.html` line 17 |
| Terminal halt | **NEVER** — the partial always re-emits its own `hx-trigger` attribute on every render | Differs from Phase 27's scan-progress partial (which omits `hx-trigger` on terminal state); the agent monitor polls indefinitely |
| HTMX request indicator | `hx-indicator` is NOT used — the swap is invisible at the user's scale (sub-100ms LAN round-trip) and a flashing indicator every 5 seconds would be visually noisy | — |
| Cadence justification | 5s matches `pipeline/dashboard.html` `#pipeline-stats` cadence; operators viewing both pages get a synchronized refresh feel | — |

### Self-Contained Partial Pattern

The HTMX poll target MUST be the **outer container of the partial itself**,
not a child element. Concrete shape:

```html
<!-- templates/admin/partials/agents_table.html -->
<section id="agents-table-section"
         hx-get="/admin/agents/_table"
         hx-trigger="every 5s"
         hx-swap="outerHTML"
         data-refreshed-at="{{ refreshed_at_iso }}"
         aria-labelledby="agents-table-heading"
         class="border border-gray-200 dark:border-phaze-border rounded-lg p-4">
  <h2 id="agents-table-heading" class="sr-only">Registered agents</h2>
  ...table or empty-state...
  <p class="text-xs text-gray-500 dark:text-gray-400 mt-2"
     x-data="{ refreshedAt: new Date('{{ refreshed_at_iso }}'),
                 now: new Date(),
                 get secondsAgo() { return Math.floor((this.now - this.refreshedAt) / 1000); } }"
     x-init="setInterval(() => now = new Date(), 1000)">
    Last refreshed <span x-text="secondsAgo + 's ago'"></span>
  </p>
</section>
```

Because `hx-swap="outerHTML"` replaces the `<section>` itself, the new
`<section>` arrives with its own `hx-trigger` — polling continues. This is
the LOCKED pattern (mirrors Phase 27 stats-bar but with `outerHTML`
self-replacement rather than `innerHTML` child-replacement; chosen because
`outerHTML` lets the partial own its entire lifecycle including the
`data-refreshed-at` timestamp).

### Error / Failure-Tolerant Refresh

HTMX poll failures (5xx, network drop, timeout) MUST NOT blow up the page.
The recovery posture:

1. **Default HTMX behavior is sufficient for transient failures.** When a
   poll request returns a 5xx or network error, HTMX 2.x leaves the existing
   DOM in place by default and emits an `htmx:responseError` /
   `htmx:sendError` event. The next 5-second tick fires normally.
2. **Surface failures in the footer.** A small HTMX event listener
   (declared once in `templates/admin/agents.html`) listens for
   `htmx:responseError` and `htmx:sendError` on `#agents-table-section`,
   stores the failure timestamp in `localStorage` (key
   `phaze-admin-agents-last-failure`), and on next successful swap clears it.
   The footer Alpine component reads this key on init to decide whether to
   render "Refresh failed at HH:MM:SS; showing last-known state" (red) or
   "Last refreshed Ns ago" (gray).
3. **No retry storm, no exponential backoff.** The 5-second cadence is the
   recovery cadence. If the API is down for 60 seconds, the user sees 12
   silent failure ticks + a single red footer message — quiet, not noisy.
4. **No browser alert / no modal.** Operator-only page on a private LAN;
   transient HTTP errors are part of normal operation during application-server
   restarts.

**Why this posture:** Phase 28 D-16 already established "fire-and-forget,
log + continue" for the heartbeat caller. The admin page mirrors that
posture client-side — the operator sees stale data with a clear "stale"
indicator rather than a broken page.

### Server-Side Behavior

| Endpoint | Method | Response |
|----------|--------|----------|
| `/admin/agents` | GET | Full page render: `base.html` extension + `admin/agents.html` + nested `admin/partials/agents_table.html` |
| `/admin/agents/_table` | GET | The partial alone: `admin/partials/agents_table.html` |
| HX-Request header detection | both routes | per STATE.md "Search UI: HTMX partial detection via truthy HX-Request header check" — `/admin/agents` MAY also serve the partial when `HX-Request: true` is set (planner's choice; the dedicated `/_table` route is the primary mechanism for polling) |

**The `_table` route returns the partial unconditionally** — it never
serves the full page. This is the cleanest design for HTMX polling and
matches Phase 27's `/pipeline/scans/agent-roots` partial-only endpoint.

## Accessibility

| Concern | Resolution |
|---------|------------|
| Color-blind users distinguishing alive/stale/dead | The pill text (`ALIVE`/`STALE`/`DEAD`/`REVOKED`/`NEVER`) is rendered inside the `<span>` AND in `aria-label`. Color is decorative, not load-bearing. |
| Screen-reader users hearing status changes during 5-s poll | The polling `<section>` has `aria-live="polite"` (already a pattern in `scan_progress.html` line 25). Screen readers announce changes without interrupting. |
| Keyboard users skipping past the table | Skip link at top of page: `<a href="#agents-table-section" class="sr-only focus:not-sr-only ...">Skip to agents table</a>` — mirrors `base.html` line 117 and `pipeline/dashboard.html` line 4. |
| Empty state read by screen readers | Empty state is regular text inside the `<section>` — no `role="status"` because the state is "no data on first render" not "data just changed." Screen readers read it as part of the normal heading hierarchy. |
| Table semantics | Standard `<table>` with `<thead>` / `<tbody>` / `<th scope="col">`. The table has a `<caption class="sr-only">Registered agents</caption>` for screen readers (mirrors `execution/partials/agents_table.html` line 25). |
| Focus management on HTMX swap | Default HTMX behavior preserves focus on swap. Since the table has no interactive cells in Phase 29 (no buttons, no links beyond the agent name tooltip), there's nothing to lose focus on. |
| Tooltip on agent name (the agent ID surface) | Use `title="{agent.id}"` on the agent name `<span>` — desktop hover only. The agent ID is ALSO rendered as visible `font-mono text-xs` subtext below the name (mirrors `execution/partials/agents_table.html` lines 39-40) so the ID is reachable for keyboard / mobile / screen-reader users without relying on hover. |
| Dark mode contrast | Every color choice above declares the `dark:` variant. The amber pill uses `dark:bg-amber-950 dark:text-amber-400` which clears WCAG AA contrast on `#0a0c12` (phaze-bg) — same ladder as every other pill in the project. |

## Empty State

Triggered when the controller's `agents` query returns zero rows (fresh
install, no agent rows yet).

**Markup pattern** (place inside the polling `<section>`, replaces the table):

```html
<div class="text-center py-8">
  <p class="text-sm font-semibold text-gray-900 dark:text-gray-100 mb-1">No agents registered yet</p>
  <p class="text-sm text-gray-500 dark:text-gray-400">
    Provision an agent token via psql, then run <code class="font-mono bg-gray-100 dark:bg-phaze-panel px-1 rounded">just up-agent</code> on the file server.
    Once the worker boots, it will appear here within a few seconds.
  </p>
</div>
```

The empty state IS polled (same `hx-trigger="every 5s"` on the outer
`<section>`) — once an agent registers, the next swap renders the table
without a page reload. Mirrors `recent_scans_table.html` empty-state
treatment.
