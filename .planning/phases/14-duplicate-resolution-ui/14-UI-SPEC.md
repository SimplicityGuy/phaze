## Interaction Contracts

### Card Expand/Collapse (D-01)

1. User clicks "Compare" button or card header on a group card.
2. HTMX `hx-get="/duplicates/{group_hash}/compare"` fetches comparison table partial.
3. Partial is swapped into `#compare-{group_hash}` target inside the card.
4. Button text toggles to "Collapse" via Alpine.js state.
5. Second click hides the comparison table (Alpine.js `x-show`, no server round-trip for collapse).

### Single Group Resolution (D-04, D-05, D-07)

1. User views expanded comparison table with radio buttons.
2. One file is pre-selected (auto-scored best per D-08).
3. User optionally changes radio selection.
4. User clicks "Resolve Group".
5. HTMX `hx-post="/duplicates/{group_hash}/resolve"` with selected file ID in form data.
6. Server marks non-canonical files as `DUPLICATE_RESOLVED` in FileRecord.state.
7. Response: card disappears (empty swap on `#group-{group_hash}`) + undo toast via OOB swap.
8. Stats header updated via OOB swap to decrement counts.

### Bulk Resolution (D-06)

1. User clicks "Accept All" in stats header.
2. HTMX `hx-post="/duplicates/resolve-all"` resolves all unresolved groups on current page.
3. All group cards disappear. Bulk undo toast appears with 10-second window.
4. "Undo All" reverses all resolutions from this bulk action.
5. Stats header updated via OOB swap.

### Undo Flow (D-07)

1. User clicks "Undo" in toast within 10 seconds.
2. HTMX `hx-post="/duplicates/{group_hash}/undo"` reverses the state change.
3. Group card reappears in the list. Stats header updated via OOB swap.
4. Toast dismissed.

### Pagination

Follow existing proposals pagination pattern: page_size=20, offset-based, HTMX partial reload of group list. Pagination controls at bottom of page.

### Keyboard Navigation

| Key | Action |
|-----|--------|
| Arrow Down | Move focus to next group card |
| Arrow Up | Move focus to previous group card |
| Enter | Toggle expand/collapse on focused card |
| R | Resolve focused group (if expanded, uses current radio selection) |

## States Matrix

| View | State | What Renders |
|------|-------|-------------|
| Duplicates page | Has unresolved groups | Stats header + paginated group cards |
| Duplicates page | No duplicate groups exist | Empty state (D-11) |
| Duplicates page | All groups resolved | Empty state with different subtext: "All duplicate groups have been resolved." |
| Group card | Collapsed (default) | Hash, file count, pre-selected rationale, "Compare" button |
| Group card | Expanded | Collapsed header + comparison table with radio buttons + "Resolve Group" button |
| Comparison table | Pre-selected row | Blue highlight (`bg-blue-50 ring-2 ring-blue-500`) on auto-scored best |
| Comparison table | User changed selection | Blue highlight moves to user-selected row |
| After resolve | Group removed | Card disappears, undo toast appears for 10 seconds |
| After bulk resolve | All cards removed | All cards disappear, bulk undo toast for 10 seconds |
| After undo | Group restored | Card reappears in list, toast dismissed |
| Stats header | Updating | OOB swap updates counts after each resolve/undo |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Radio group semantics | `<fieldset>` with `<legend class="sr-only">` "Select canonical file for group {hash}" per comparison table |
| Radio labels | Each radio has `aria-label` "Select {truncated_path} as canonical" |
| Card expand/collapse | Button has `aria-expanded="true/false"`, `aria-controls="compare-{hash}"` |
| Toast announcements | Toast container has `aria-live="polite"` (already on `#toast-container` in base.html) |
| Best value highlighting | Green text alone is not sufficient -- also uses `font-semibold` for non-color indicator |
| Keyboard navigation | Arrow keys for card focus, Enter for expand, R for resolve -- announced via keyboard hint |
| Skip link | On duplicates page, skip link targets `#duplicates-list` |
| Color contrast | Green-700 on white passes WCAG AA. Blue-600 on white passes AA. Gray-500 on gray-50 passes AA for large text. |
| Focus indicators | Default browser focus ring on radio buttons and buttons. Focused card gets `ring-2 ring-blue-500` |
