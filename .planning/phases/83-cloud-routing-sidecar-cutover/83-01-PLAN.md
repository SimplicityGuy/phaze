<objective>
Create the single shared go-forward writer of `cloud_job.status='awaiting'` — `hold_awaiting_cloud()` in
`services/backends.py`. This closes the D-01 discovery (no such writer exists today) and gives the hold
path and both over-cap spill paths one reused writer instead of three hand-written copies (D-02).

Purpose: Every `AWAITING_CLOUD` file must carry a `cloud_job(status='awaiting')` sidecar row so the hard
shadow invariant `AWAITING_CLOUD ⇒ cloud_job(status='awaiting')` holds for all go-forward holds. This is
the foundation Wave 2/3 build on (the hold-path cutover and the spill re-stamps reuse this helper).

Output: `hold_awaiting_cloud()` helper; unit tests proving the upsert shape, the `'awaiting' ∉ IN_FLIGHT`
invariant (D-03), and that `LocalBackend.dispatch` keeps its flip and writes no row (D-13).
</objective>
