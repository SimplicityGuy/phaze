<objective>
Build the pure kr8s seam — the single home of every Kubernetes API call — mirroring `s3_staging.py`:
a config-validated async client factory, the suspended-Job manifest builder, and one-operation-per-
function verbs (submit/list/get/get-workload/delete) the submit task and reconcile cron call. Plus the
shared fake-kube test substrate (respx fixture + canned Job/Workload factories) every later test reuses.

Purpose: KSUBMIT-01 (the suspended-Job spec + deterministic name + 409-idempotent create), KSUBMIT-05
(`backoffLimit: 0` neutralizes pod-level retry), KSUBMIT-06 (idempotent delete). Keeping kr8s behind a
thin pure seam (no ORM) makes the reconcile/submit state machine testable HTTP-free (Layer 1).
Output: kube_staging.py + tests/kube_fakes.py + the `kube_respx` fixture + seam tests.
</objective>

## Trust Boundaries

|----------|-------------|
| control plane → kube API server | the seam authenticates with kube creds and submits/lists/deletes objects over Tailscale/WireGuard |
| file_id → kube object name | the deterministic Job name is derived from a server-generated UUID |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-06 | Tampering | Job name injection | mitigate | name = `phaze-analyze-<file_id>` where file_id is a server UUID (DNS-1123 safe); no operator free-text enters kube object names |
| T-54-07 | Information disclosure | kube creds in the client factory | mitigate | creds come from `_FILE`-resolved SecretStr; the `_api` factory never logs them |
| T-54-08 | Elevation of privilege | pod-level/infra retry | mitigate | `backoffLimit: 0` neutralizes Job retry so the control plane solely owns the retry budget (KSUBMIT-05) |
| T-54-09 | Denial of service | repeated delete on a missing Job | mitigate | delete_job swallows NotFound → idempotent, safe to re-run after a partial reconcile tick (KSUBMIT-06) |
</threat_model>
