<objective>
Implement the copy-verify-delete execution service with write-ahead audit logging and arq batch job.

Purpose: This is the critical safety layer for an irreplaceable 200K file collection. Files are copied, SHA256-verified, then originals deleted -- never direct rename/move. Every operation is logged before execution for crash traceability (EXE-02). The arq job processes all approved proposals sequentially within a single batch job, storing progress in Redis for the SSE endpoint (Plan 02).

Output: Execution service module, arq task module, Alembic migration, comprehensive tests.
</objective>

    Create `src/phaze/services/execution.py` with these functions:

    1. `compute_sha256(file_path: Path) -> str` -- chunked (8192 bytes) SHA256 hash computation. Pure function, no DB. Per research pattern.

    2. `async def log_operation(session, proposal_id, operation, source_path, destination_path) -> ExecutionLog` -- creates ExecutionLog entry with status=IN_PROGRESS, sha256_verified=False. Commits immediately (write-ahead per EXE-02). Per research Pattern 1.

    3. `async def complete_operation(session, log_entry, *, sha256_verified=False, error_message=None) -> None` -- updates log entry status to COMPLETED (no error) or FAILED (with error). Updates sha256_verified. Commits.

    4. `async def get_approved_proposals(session) -> list[RenameProposal]` -- select all proposals with status=APPROVED, eagerly load file relationship via selectinload. Order by created_at for deterministic processing.

    5. `async def execute_single_file(session, proposal, file_record) -> bool` -- the core copy-verify-delete flow:
       - Compute destination: `Path(file_record.current_path).parent / proposal.proposed_filename` (per D-03 rename in-place)
       - Guard: if destination.exists(), fail immediately (pitfall 5 from research)
       - Step 1 COPY: log_operation("copy"), shutil.copy2(source, destination), complete_operation. On OSError: mark failed, return False.
       - Step 2 VERIFY: log_operation("verify"), compute_sha256(destination) vs file_record.sha256_hash. On mismatch: complete_operation(error), delete destination (per D-05), set state=FAILED, return False. On match: complete_operation(sha256_verified=True).
       - Step 3 DELETE: log_operation("delete"), source.unlink(). On OSError: complete_operation(error) but still update FileRecord (copy is good).
       - Update FileRecord.current_path to str(destination), state=EXECUTED. Commit. Return True.
       - Per D-06: no automatic retry. Per D-07: caller continues on failure.
