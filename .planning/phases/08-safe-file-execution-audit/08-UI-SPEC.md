# Phase 8 — UI Design Contract

> Visual and interaction contract for safe file execution and audit UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS via CDN, no build step) |
| Preset | not applicable |
| Component library | none (hand-built with Tailwind utility classes) |
| Icon library | Heroicons via CDN inline SVG (consistent with Phase 7) |
| Font | Inter via Google Fonts CDN (`?family=Inter:wght@400;600`) |

Source: Inherited from Phase 7 design contract. CLAUDE.md stack (HTMX + Jinja2 + Tailwind CSS CDN + Alpine.js). No React, no shadcn, no build pipeline.

**New CDN dependency for this phase:**

| Dependency | URL | Purpose |
|------------|-----|---------|
| htmx-ext-sse 2.2.4 | `https://cdn.jsdelivr.net/npm/htmx-ext-sse@2.2.4` | HTMX SSE extension for live progress streaming (D-08) |

Add to `base.html` `<head>` after the HTMX script tag with SRI hash.

## Spacing Scale

Inherited from Phase 7 without changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, inline badge padding |
| sm | 8px | Table cell vertical padding, compact element spacing |
| md | 16px | Table cell horizontal padding, card padding |
| lg | 24px | Section padding (stats bar, progress panel), page horizontal gutter |
| xl | 32px | Page vertical padding between major sections |
| 2xl | 48px | Top-of-page spacing above heading |
| 3xl | 64px | Not used in this phase |

Exceptions: Table row height minimum 44px for touch target accessibility (same as Phase 7).

Tailwind mapping: `p-1`(4px), `p-2`(8px), `p-4`(16px), `p-6`(24px), `p-8`(32px), `py-12`(48px). No arbitrary values.

## Typography

Inherited from Phase 7 without changes:

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 12px | 600 (semibold) | 1.5 | `text-xs font-semibold leading-normal` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

Body at 14px for data-dense audit log table. Display at 28px for "Audit Log" page title. Label at 12px for column headers, status badges, and operation badges.

Weights: 400 (regular) for body text and table cell data. 600 (semibold) for headings, labels, column headers, buttons, and badges. No other weights.

| Column: error | Error |

| Pagination | Showing {start}-{end} of {total} operations |

| Operation badge: copy | Copy |

| Operation badge: delete | Delete |

### Navigation

| Element | Copy |
|---------|------|
| Nav link: proposals | Proposals |
| Nav link: audit log | Audit Log |

### Destructive Actions

| Action | Approach |
|--------|----------|
| Execute approved (batch file operations) | Confirmation dialog before execution. Copy: "This will rename {N} approved files using copy-verify-delete. Original files are preserved until verification succeeds. Continue?" Confirm button: "Execute {N} Files". Cancel button: "Cancel". |

Rationale: File operations are significant and affect the actual filesystem. Unlike approve/reject (which are status changes), execution moves real files. A confirmation dialog is warranted. Source: D-01 (human-in-the-loop), D-02 (all approved at once).

## Visual Hierarchy

### Proposal Review Page (modified)

**Primary visual anchor:** the proposal table (unchanged from Phase 7).

**Eye flow:** stats bar -> filter tabs -> search box -> proposal table -> pagination -> bulk action bar. **New element:** "Execute Approved" button sits in the stats bar area (top-right of the stats bar grid), giving it visibility without competing with the table review workflow.

### Audit Log Page (new)

**Primary visual anchor:** the audit log table (the operational record).

**Eye flow:** page title ("Audit Log") -> filter tabs (scope by status) -> audit table (review operations) -> pagination (navigate).

## Accessibility

Inherited from Phase 7, with additions for Phase 8:

| Requirement | Implementation |
|-------------|----------------|
| Execute button label | `aria-label="Execute all approved proposals"` on the button |
| Confirmation dialog | Native `hx-confirm` attribute (uses browser confirm dialog). Accessible by default. |
| Progress region | `aria-live="polite"` on the SSE progress container so screen readers announce updates |
| Audit log table | `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` for screen reader compatibility |
| Status not color-only | All status badges include text labels ("Completed", "Failed", "In Progress", "Pending") |
| Verified not color-only | Checkmark/X icons include `aria-label` text ("SHA256 verified" / "SHA256 not verified") |
| Navigation | `<nav aria-label="Main navigation">` wrapping the Proposals/Audit Log links |
| Skip link | Existing "Skip to proposals table" link. Audit log page adds "Skip to audit table" link. |
| Page title | `<title>` updates per page: "Proposal Review - Phaze" and "Audit Log - Phaze" |
| Focus visible | All interactive elements show visible focus ring (ring-2 ring-blue-500) |
