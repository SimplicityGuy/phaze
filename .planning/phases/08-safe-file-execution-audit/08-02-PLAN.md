    **Step 1: Add sse-starlette dependency**
    Run `uv add "sse-starlette>=3.3.4"`. This provides EventSourceResponse for SSE streaming.

    **Step 2: Create audit log query service** (`src/phaze/services/execution_queries.py`)

    Follow the proposal_queries.py pattern exactly:

    1. `async def get_execution_stats(session: AsyncSession) -> dict[str, int]` -- single query counting total, completed, failed, in_progress using func.count + case. Return dict with keys: total, completed, failed, in_progress.

    2. `async def get_execution_logs_page(session: AsyncSession, *, status: str | None = None, page: int = 1, page_size: int = 50) -> tuple[list[ExecutionLog], Pagination]` -- paginated, filtered query. Order by executed_at DESC. Reuse the Pagination dataclass from proposal_queries.py (import it). Filter by ExecutionLog.status when status param is not None and not "all".

    **Step 3: Create execution router** (`src/phaze/routers/execution.py`)

    Follow the proposals.py router pattern:

    1. `POST /execution/start` -- "Execute Approved" trigger (per D-01, D-02):
       - Accept Request and session dependency
       - Access arq pool via `request.app.state.arq_pool`
       - Generate batch_id = uuid4().hex
       - Enqueue job: `await arq_pool.enqueue_job("execute_approved_batch", batch_id)`
       - Return HTML fragment: the progress display partial with batch_id for SSE connection
       - Use `hx-confirm` attribute on the button (not server-side confirmation) per UI-SPEC

    2. `GET /execution/progress/{batch_id}` -- SSE progress stream (per D-08):
       - Import EventSourceResponse from sse_starlette
       - Access Redis via `request.app.state.arq_pool` (ArqRedis extends redis.asyncio.Redis, supports hgetall)
       - Create async generator that polls Redis `exec:{batch_id}` hash every 1 second
       - Yield SSE events with event="progress" containing HTML span with progress text per UI-SPEC copywriting contract:
         - No data yet: "Waiting for execution to start..."
         - Running with no failures: "{completed}/{total} files processed"
         - Running with failures: "{completed}/{total} files processed ({failed} failed)"
       - When status == "complete": yield final event with event="complete" containing completion message per UI-SPEC:
         - All success: "Execution complete. All {total} files renamed successfully."
         - With failures: "Execution complete. {completed} succeeded, {failed} failed."
         - Include "View Audit Log" link: `<a href="/audit/" class="text-blue-600 hover:underline ml-2">View Audit Log</a>`
       - Return EventSourceResponse(generator)
       - IMPORTANT: Use `sse-close="complete"` on the HTMX element (in template) so connection closes when complete event received (pitfall 4)

    3. `GET /audit/` -- Audit log page (per D-09):
       - Accept Request, session, optional status filter, page, page_size query params
       - Call get_execution_logs_page and get_execution_stats
       - Context: logs, pagination, stats, current_status
       - HTMX request: return audit_table.html partial
       - Full request: return audit_log.html full page
