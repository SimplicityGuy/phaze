<objective>
Establish the alembic-driven test substrate Phase 24 needs to actually exercise migrations 012 and 013. The existing `tests/conftest.py` uses `Base.metadata.create_all` (sidesteps Alembic entirely — RESEARCH Pitfall 2), so no migration in the repo has ever been verified by pytest. This plan creates the new `tests/test_migrations/` package with a `migrated_engine` fixture that runs `alembic.command.upgrade()` against a dedicated test DB plus step-helpers (`upgrade_to`, `downgrade_to`) that subsequent plans use to assert on intermediate revisions.

Purpose: All Wave 1-3 plans depend on this substrate to write `tests/test_migrations/test_012_upgrade.py`, `test_013_upgrade.py`, and `test_downgrade.py`. Without it, the rest of Phase 24 verifies the model layer but never validates the actual migration files that ship.

Output: An empty `tests/test_migrations/__init__.py`, a `tests/test_migrations/conftest.py` exporting the `migrated_engine` fixture plus step helpers, both files committed and importable by pytest.
</objective>
