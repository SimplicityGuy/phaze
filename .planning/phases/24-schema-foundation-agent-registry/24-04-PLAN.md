<objective>
Land migration 013 — the constraint-tightening half of the two-step shape. Enforces NOT NULL on `files.agent_id` and `scan_batches.agent_id` (both backfilled by Plan 03's migration 012), drops the legacy single-column `uq_files_original_path` unique index, creates the composite `uq_files_agent_id_original_path` unique index over `(agent_id, original_path)`. Implements the downgrade with the mandatory D-16 duplicate-detection guard: if the same `original_path` exists under multiple agents, raise `RuntimeError` instead of silently corrupting data.

Writes the integration tests for migration 013 (DATA-02 + DATA-03 NOT NULL behavior) AND a separate `test_downgrade.py` covering clean roundtrip downgrades plus the D-16 dupe-detection error path. Includes the mandatory `[BLOCKING]` `alembic upgrade head -> downgrade base -> upgrade head` smoke task — the Python/Alembic equivalent of a schema-push gate that catches downgrade bugs build/typecheck cannot see (types come from `__table_args__`, not the live DB).

Purpose: This is the step that makes the new attribution invariant enforceable end-to-end. Without 013, Phase 25+ code could leave `agent_id` NULL and rely on application validation; with 013, the DB itself refuses NULL attribution and rejects same-path collisions per-agent. The downgrade guard is irreplaceable: silent dedup of files in an irreplaceable personal music collection is forbidden (CONTEXT D-16).

Output: New `alembic/versions/013_enforce_agent_id_not_null_and_swap_uniqueness.py`; new `tests/test_migrations/test_013_upgrade.py` covering rows #17–#21 from VALIDATION.md; new `tests/test_migrations/test_downgrade.py` covering rows #22–#24; one final `[BLOCKING]` smoke task running the operator-visible `just db-upgrade` / `just db-downgrade` cycle.
