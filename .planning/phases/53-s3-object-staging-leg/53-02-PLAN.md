<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent/pod → control plane | token-authed POST to mint presigned URLs |
| control plane → S3 backend | the only place that holds bucket credentials (DIST-01/KSTAGE-01) |
| presigned URL → S3 | a time-boxed capability handed to an untrusted transferer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-05 | Elevation of Privilege | presigned GET/PUT URL scope+TTL | mitigate | file_id-scoped key; GET TTL is the short s3_presign_get_ttl_sec minted just-in-time (KSTAGE-03); PUT TTL bounded |
| T-53-06 | Information Disclosure | sha256 from request body could disable integrity | mitigate | expected_sha256 sourced ONLY from FileRecord.sha256_hash server-side; never from the request (AUTH-01) |
| T-53-07 | Spoofing | unauthenticated presign request | mitigate | get_authenticated_agent token dependency on the route; 401 on missing/invalid token |
| T-53-08 | Tampering | file_id smuggled in a body to redirect presign | mitigate | file_id on URL PATH only; no request body accepted |
| T-53-09 | Information Disclosure | S3 credentials in logs/errors | mitigate | aioboto3 confined to s3_staging.py; SecretStr; no secret values logged |
| T-53-10 | Denial of Service | object leak (orphaned staged objects) | mitigate | ensure_bucket_lifecycle_ttl backstop (KSTAGE-04) + delete_staged_object capability |
</threat_model>
