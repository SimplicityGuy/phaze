<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator config → control plane | S3 endpoint/bucket/creds provided via env + `_FILE` secrets |
| control plane → S3 backend | endpoint_url is operator-controlled (SSRF surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Information Disclosure | S3 credentials in config | mitigate | `SecretStr` + `_FILE` convention; creds on ControlSettings only, never AgentSettings (KSTAGE-02); never logged |
| T-53-02 | Spoofing/SSRF | operator-provided s3_endpoint_url | mitigate | field_validator requires well-formed http(s) URL with netloc; rejects file:// and scheme-less values |
| T-53-03 | Denial of Service | unbounded presign/lifecycle TTL or part-size config | mitigate | gt/lt + ge bounds on every int field (fail-fast at startup) |
| T-53-04 | Tampering | npm/pip installs (aioboto3, moto) | mitigate | both verified legitimate on pypi.org (see legitimacy note); `[tool.uv] exclude-newer` cooldown enforced |
| T-53-SC | Tampering | supply-chain of new deps | mitigate | aioboto3 (terricain) + moto (getmoto) are well-established PyPI packages; pinned floors under the 7-day cooldown |
</threat_model>
