<objective>
Close CR-01: guard the unguarded `report_scan_terminal` call on the `scan_live_set` no-match
early-return. Today (scan.py:95-100) the no-match branch calls `await api.report_scan_terminal(...)`
with NO exception handling, then returns `{"status": "no_matches"}`. If the controller is down/5xx
after retries, `report_scan_terminal` raises `AgentApiServerError` (or an auth/client error): the
`no_matches` return never executes, SAQ records a FAILED attempt and retries, and on the
retries-exhausted terminal attempt the ack STILL raises — so the `scan_live_set:<file_id>` ledger
row is NEVER cleared and recovery re-enqueues this file on every pass forever. `scan_live_set` is
classified live-keys-only (no domain predicate) precisely because the ack was supposed to clear on
EVERY outcome; the unguarded no-match path breaks that invariant.

Purpose: honor L-02 ("ledger cleared on completion AND terminal failure for every agent stage") for
the scan no-match outcome, and stop the controller-down-during-no-match recovery loop.

Output: a try/except around the no-match `report_scan_terminal` that mirrors the match-path
exception handler verbatim (re-raise on a retryable attempt so the row survives the real retry;
swallow + `logger.warning(..., exc_info=True)` on the terminal attempt so the no-match COMPLETE
still returns), plus tests proving the discipline.
</objective>
