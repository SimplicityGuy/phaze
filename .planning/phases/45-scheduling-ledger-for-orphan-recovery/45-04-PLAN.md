<objective>
Add the one-time idempotent startup backfill (locked decision #3): seed the scheduling_ledger from
the live `saq_jobs` queued/active rows at first boot after the migration, so in-flight work (and any
residual incident cohort still present) stays recoverable immediately — no blind window. This is a
control-side runtime reconcile, NOT an Alembic data migration (Alembic must never read/write the
SAQ-owned saq_jobs table).

Purpose: Between the 022 migration landing and the WRITE hook starting to populate the ledger, jobs
already in saq_jobs have no ledger row. The backfill closes that gap once; it is idempotent (ON
CONFLICT DO NOTHING keyed by the deterministic key) so it is safe to run every boot and becomes a
cheap no-op after the transition cohort drains.

Output: backfill_ledger_from_saq_jobs in reenqueue.py + its startup wiring + integration test.
</objective>
