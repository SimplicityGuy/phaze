<objective>
Close CR-02: give `extract_file_metadata` and `fingerprint_file` a terminal-failure ledger clear.
Today their ONLY ledger clear is in the success PUT handler (`put_metadata` / `put_fingerprint`).
There is NO `/failed` callback for either stage. The phase relied on the domain-completed predicate
(`is_domain_completed` in reenqueue.py) as the net, but that predicate is "absent-from-pending == done"
and CANNOT fire for a terminally-failed file:
  - extract_file_metadata: `get_metadata_pending_files` returns ALL music/video files regardless of
    state → a terminally-failed file is still in the pending set → never domain-completed →
    re-enqueued on every recovery pass forever.
  - fingerprint_file: `get_fingerprint_pending_files` returns METADATA_EXTRACTED files PLUS
    FingerprintResult(status="failed") rows → both keep a terminally-failed file in the pending set →
    re-enqueued forever.
This is an unbounded recovery re-enqueue loop — the exact defect the ledger was introduced to prevent.

The fix mirrors the analyze stage exactly (shipped in 45-02): add a `POST /{file_id}/failed`
terminal-failure endpoint for each stage that clears the ledger row control-side, a corresponding
agent-client method, and an agent-worker try/except in each task that invokes the ack ONLY on the
retries-exhausted terminal attempt (`job is not None and not job.retryable`) then re-raises —
mirroring `process_file` (functions.py:179-189). This honors locked operator decision #1 (terminal
`failed` CLEARS the ledger, no poison re-queue) and the existing analyze/scan precedent. After the
clear, the domain-completed predicate becomes belt-and-suspenders rather than the (broken) primary net.

Purpose: satisfy L-02 ("ledger cleared on completion AND terminal failure for EVERY agent stage").
Output: two new `/failed` endpoints + response schemas, two new client methods, two agent-worker
guards, handler/task tests, and a recovery regression test proving a terminally-failed
metadata/fingerprint ledger row is cleared and NOT re-enqueued.
</objective>
