<objective>
Build the durable scheduling ledger foundation: the SchedulingLedger model, the reversible
Alembic 022 migration, a control-only ledger service (upsert / clear / read + routing
classifier), the `get_live_job_keys` query, and the WRITE + controller-CLEAR hook wiring at
the existing `before_enqueue` / `after_process` chokepoints. This is Wave 1 — every later plan
imports this model and these helpers.

Purpose: Establish "a record exists that a stage was scheduled for an item" (the missing fact
behind the incident) at the single universal chokepoint, while preserving the Postgres-free
agent boundary (the WRITE only fires control-side; the agent queue has no `ledger_sessionmaker`
so the hook degrades to a logged no-op) and clearing controller-stage entries on success AND
terminal failure with ONE hook (locked decision #1, controller half).

Output: scheduling_ledger table + model, migration 022, ledger service, get_live_job_keys,
the wired WRITE hook + controller-stage CLEAR hook, and full unit + migration tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| control-plane producer -> before_enqueue hook -> Postgres ledger | every keyed enqueue upserts a control-side ledger row |
| SAQ worker -> after_process hook -> Postgres ledger | controller-stage terminal outcome clears its ledger row |
| _shared deterministic_key.py -> phaze.database | the agent worker imports _shared; the DB handle must stay behind a runtime getattr, never a module import |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | Tampering | job.kwargs persisted verbatim into ledger.payload | accept | kwargs already validated by the keyed producers' extra="forbid" schemas before enqueue; the ledger stores the same trusted control-side payload. JSONB stored, never executed. |
| T-45-02 | Elevation/Boundary | deterministic_key.py dragging phaze.database into the agent import graph | mitigate | DB access only via getattr(job.queue, "ledger_sessionmaker", None) + a function-LOCAL lazy import that executes only when the handle is present (control-side). tests/test_task_split.py is in this plan's verify gate and MUST stay green. |
