<objective>
Add an optional per-job cap override to ProcessFilePayload and thread it end-to-end:
producer (enqueue_process_file) -> serialized payload -> agent worker (process_file) ->
analyze_file. This is the backend lever the "deepen analysis" endpoint (Plan 03) will pull —
NO DB migration, a pydantic schema field only.

Purpose: A sampled file must be re-analyzable with a higher/unbounded window budget. cap<=0
reaches the shipped _stride_to_cap no-op (analyze ALL windows). Defaults of None preserve the
existing AgentSettings 60/30 behavior for every current caller.
Output: ProcessFilePayload cap fields, extended enqueue_process_file, payload-aware
process_file, full regression tests.
</objective>

No new unbounded path. |
| T-44-03 | Spoofing | producer routing to a consumer-less queue | mitigate | This plan does NOT enqueue directly; it only extends the single enqueue_process_file funnel and its caller. Routing is the caller's responsibility (Plan 03 uses enqueue_router). Deterministic key process_file:<file_id> preserved (dedup intact). |
| T-44-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; nothing to audit. |
