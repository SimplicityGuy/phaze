<interfaces>
CRITICAL — saq_jobs has NO started/touched SQL column (PATTERNS.md banner). Columns are only:
  key TEXT PK, lock_key SERIAL, job BYTEA, queue TEXT, status TEXT, priority SMALLINT, group_key TEXT, scheduled BIGINT, expire_at BIGINT
SAQ stores started/touched (epoch MILLISECONDS) INSIDE the serialized job BYTEA blob (saq/job.py:132-133),
and sets job.started = dequeue time on the active transition (saq/queue/postgres.py:692). There is NO pure-SQL
age filter on a started column. This plan reads age in PYTHON by deserializing the SMALL set of active
process_file job blobs (see Task 2 action). scheduled BIGINT is set to dequeue/now on active and is queryable but
does NOT equal started after a retry, so it is NOT a reliable running-age source — use the blob's started.
