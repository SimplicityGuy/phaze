<objective>
HARD-02: Make the `push_attempt` ledger read-modify-write atomic under concurrent `/mismatch`.

`routers/agent_push.py::report_push_mismatch` reads `SchedulingLedger.payload["push_attempt"]`,
computes `+1`, and writes it back. Two concurrent `/mismatch` for the same file can both read the
same value and lose an increment (AR-73-02 / T-73-13 / WR-04). Fix by adding `.with_for_update()`
to the ledger SELECT so the RMW is serialized at the row level in Postgres.

Purpose: Close AR-73-02 / T-73-13 / WR-04 — no lost increment, cap still trips.
Output: Row-locked SELECT plus a real-Postgres concurrent regression test.
</objective>
