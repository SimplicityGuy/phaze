    In `src/phaze/routers/tags.py`, in the `write_file_tags` endpoint (around line 317, after the form_data parsing loop), add a fallback: if `tags` dict is empty after parsing form_data, compute the proposed tags server-side and use those instead. This makes the collapsed-row button functional since it cannot include comparison panel inputs. The specific change:

    After the existing `for field in CORE_FIELDS:` loop that builds `tags` from `form_data` (lines 306-316), and BEFORE the source determination block (line 319), add:

    ```python
    # Fallback: if no tag values submitted (e.g., collapsed row button without comparison panel),
    # use server-computed proposed tags
    if not tags:
        tracklist = await _get_tracklist_for_file(session, file_id)
        computed = compute_proposed_tags(file_record.file_metadata, tracklist, file_record.original_filename)
        tags = {k: v for k, v in computed.items() if v is not None}
        source = "proposal"
    ```

    Then move the existing `tracklist = await _get_tracklist_for_file(...)` and `computed = compute_proposed_tags(...)` calls (currently at lines 319-320) inside an `else:` block, so the tracklist query is not duplicated when tags are non-empty. The resulting structure should be:

    ```python
    if not tags:
        tracklist = await _get_tracklist_for_file(session, file_id)
        computed = compute_proposed_tags(file_record.file_metadata, tracklist, file_record.original_filename)
        tags = {k: v for k, v in computed.items() if v is not None}
        source = "proposal"
    else:
        tracklist = await _get_tracklist_for_file(session, file_id)
        computed = compute_proposed_tags(file_record.file_metadata, tracklist, file_record.original_filename)
        has_edits = any(str(tags.get(f, "")) != str(computed.get(f, "")) for f in CORE_FIELDS if f in tags or f in computed)
        source = "manual_edit" if has_edits else "proposal"
    ```
