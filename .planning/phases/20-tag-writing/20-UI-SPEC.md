|--------|-------|---------|

## Interaction Contract

### Expand/Collapse Tag Comparison

- **Trigger:** Click filename or expand button in file row
- **Mechanism:** `hx-get="/tags/{file_id}/compare"` targeting a `<tr>` below the file row
- **Swap:** `hx-swap="innerHTML"` on the detail row `<td colspan="5">`
- **Toggle:** Alpine.js `x-data="{ expanded: false }"` with `@click="expanded = !expanded"`

### Inline Edit Proposed Values

- **Trigger:** Click on proposed value span in comparison panel
- **Mechanism:** `hx-get="/tags/{file_id}/edit/{field}"` returns `<input>` element
- **Save:** `hx-put="/tags/{file_id}/edit/{field}"` on blur or Enter key
- **Swap:** `hx-target="closest td"` `hx-swap="innerHTML"`
- **Input classes:** `text-sm border border-blue-300 rounded px-2 py-1 w-full focus:outline-none focus:ring-1 focus:ring-blue-500 bg-white`

Source: Exact pattern from `tracklists/partials/inline_edit_field.html`.

### Write Tags (Primary CTA)

- **Trigger:** Click "Write Tags" button on an expanded file row
- **Mechanism:** `hx-post="/tags/{file_id}/write"` with current proposed values
- **Response:** Replace the file row with updated status (completed/discrepancy/failed)
- **Feedback:** Toast notification with outcome message
- **Button classes:** `bg-blue-600 hover:bg-blue-700 text-white text-sm font-semibold px-4 py-2 rounded-md`
- **Disabled state:** `bg-gray-300 text-gray-500 cursor-not-allowed` when no changes proposed

### HTMX Partial Detection

All endpoints check `HX-Request` header. If truthy, return partial template. If falsy, return full page. Matches existing pattern from search and tracklists (D from STATE.md accumulated decisions).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Write Tags" |
| Secondary action | "Write All Pending" (bulk action for visible page) |
| Empty state heading | "No files ready for tag writing" |

| Discrepancy notice | "Tags written with {n} discrepancy. Re-read values differ from what was sent -- usually encoding normalization. Review the audit log for details." |
| Success toast | "Tags written to {filename}" |
| Comparison header | "Tag Comparison" |
| Changed field label | Field name rendered in `font-semibold text-blue-600` when proposed differs from current |
| Empty field placeholder | "\u2014" (em dash) in `text-gray-400` |
| No changes indicator | "No changes" in `text-gray-400 italic` when proposed matches current for all fields |
| Write blocked tooltip | "Only executed files with destination copies can have tags written" |

Source: D-04 through D-09 from CONTEXT.md. Empty/error patterns follow `proposals/list.html` structure.
