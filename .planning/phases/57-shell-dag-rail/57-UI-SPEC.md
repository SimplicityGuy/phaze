# Phase 57 — UI Design Contract

> Visual and interaction contract for the v7.0 load-bearing three-column "Hybrid Console" shell.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Authority:** This contract CODIFIES already-locked decisions — it does not invent direction.
> The canonical visual/behavioral target is `docs/superpowers/specs/2026-06-28-ui-redesign-assets/prototype.html`.
> The brand/theme machinery is preserved VERBATIM from `src/phaze/templates/base.html` (SHELL-04).
> When the prototype's raw mockup hexes (`#0b1220`, `#3b82f6`, …) conflict with the production C3
> tokens in `base.html`, **the production tokens win** — this is "evolution, not reskin."

## Scope Boundary (read first)

This spec covers the **shell chrome only** — the load-bearing IA that Phases 58–62 build on.

| In scope (contract below) | Out of scope (deferred — note only) |
|---------------------------|--------------------------------------|
| Three-column shell layout (header · DAG rail · `#stage-workspace` · right pane) | Redesigned rich stage workspaces (Analyze lane cards, Discover/Metadata/Fingerprint views) — **Phase 58** |
| DAG rail as nav spine (node states, order, counts, HTMX wiring, `aria-current`) | Functional ⌘K palette contents (search/commands wiring) — **Phase 61** |
| Header status strip (agent dots + Agents link, D-05 minimal) | Per-file full-record slide-in detail; Agents page rebuild — **Phase 61** |
| ⌘K affordance (button + keybind + **empty skeleton** modal, D-04) | First-run empty/scan state — **Phase 61** |
| Theme/brand preservation (C3 / Jura / blue / wave / auto-dark-light) | Responsive narrow rail-collapse-to-icons; density/a11y depth — **Phase 62** |
| Focus/ARIA baseline (skip-link, focus-to-heading, `aria-current`, modal focus) | Dead-template removal (CUT-02) — **Phase 62** |
| Fragment-correctness as a UI contract (content-only swaps; chrome persists) | — |

Phase 57 stage workspaces and the right pane **EMBED EXISTING legacy content as fragments**
(the bridge, D-01). This spec governs the chrome AROUND that bridged content, not the bridged
content's internal styling.

## Spacing Scale

Base contract — the 4-point scale (used for layout gaps, section/card padding):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | icon gaps, inline padding |
| sm | 8px | compact element spacing, rail node gap (`gap-2`) |
| md | 16px | default element spacing, header gap (`gap-4`), pane padding (`p-5` chrome → 20px exception) |
| lg | 24px | section padding, workspace header padding (`px-6` → 24px) |
| xl | 32px | layout gaps |
| 2xl | 48px | major section breaks |
| 3xl | 64px | page-level spacing, first-run logo (deferred to 61) |

**Exceptions (permitted — established in the existing dense-console codebase; this is a
single-user desktop admin tool, not a marketing surface):**

- **Tailwind half-steps** (2/6/10/14/20px → `*-0.5`, `*-1.5`, `*-2.5`, `*-3.5`, `*-5`) are
  allowed for dense chrome: rail node padding `px-3 py-2.5` (12/10px), rail group label
  `tracking-[0.2em]`, status dots, header control gaps `gap-2.5` (10px). These appear verbatim
  in the prototype and match `base.html`'s existing nav density.
- **Fixed load-bearing dimensions** (do NOT round to the 4-scale — they define the layout grid):
  | Element | Dimension | Source |
  |---------|-----------|--------|
  | Left DAG rail width | **280px** (`w-[280px]`) | prototype `<aside>` |
  | Right file pane width | **350px** (`w-[350px]`) | prototype `<aside id="pane">` |
  | Header height | **56px** (`h-14`) | prototype `<header>` |
  | Footer height | **28px** (`h-7`) | prototype `<footer>` (optional in 57) |
  | ⌘K modal width | **640px** (`w-[640px] max-w-[92vw]`) | prototype `#cmd` |
  | Control heights | 32 / 36 / 44px (`h-8` / `h-9` / `h-11`) | prototype buttons/inputs |
  | Status dots | 6 / 8 / 10px (`w-1.5` / `w-2` / `w-2.5`) | prototype dots |

Center workspace, rail, and pane each `overflow-y-auto` independently; the shell is
`h-screen overflow-hidden flex flex-col` (full-viewport app frame, not a scrolling document).

## Typography

Two brand-locked families, preserved verbatim from `base.html` (SHELL-04): **Jura** for
headings/eyebrows/labels (uppercase, letter-spaced), **Inter** for body/UI text.

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Heading | Jura | 18px (`text-lg`) | 500 | 1.2 | stage-workspace `<h1>` (uppercase, `tracking-[0.15em]`); focus target after swap |
| Body | Inter | 14px (`text-sm`) | 400 | 1.5 | default UI text, rail node labels, table cells, pane text |
| Mono counts | monospace | 12px (`text-xs`) | 400 | 1.4 | rail live-counts + capacity numerals (`mono` → `ui-monospace, 'SF Mono', Menlo`) so digit columns align — a display treatment, not a new family |
| Label | Jura | 11px (`text-[11px]`) | 500 | 1.4 | section eyebrows, rail group labels (`uppercase tracking-[0.2em]`), "PHAZE" wordmark (`text-sm tracking-[0.25em]`), footer |

Exactly **four sizes** in Phase 57's shell chrome: 18 / 14 / 12 / 11px. The **Display / 24px**
(`text-2xl`) scale is **reserved for and defined by Phase 61's first-run / empty-state heading**,
where it first renders — it appears in no Phase 57 header/rail/pane element, so it is not declared
here.

**Weights — exactly two declared for Phase 57 chrome (both preserved from `base.html`):**

- **400 (regular)** — Inter body, the default (rail node labels, table cells, pane text, the
  ⌘K placeholder, mono counts).
- **500 (medium)** — Jura headings/eyebrows/labels and brand wordmark (the brand-locked heading
  weight; `base.html` loads Jura 300/500), and the Jura-tracked controls (the "+ Scan" CTA, the
  "Agents" link). Do not introduce Jura 700/bold in the shell.

> **Note:** Inter 600 (semibold) exists in `base.html` and will appear in bridged stage content /
> Phase 58+ workspaces; Phase 57 chrome does not introduce it, so it is not part of this phase's
> declared weight set.

The C3 dark palette is primary. All values are existing `@theme` tokens in `base.html` — the
shell uses them via Tailwind utilities; it does NOT introduce the prototype's raw mockup hexes.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `phaze-bg` **#0a0c12** (dark) / `white` (light) | body background + center `#stage-workspace` canvas |
| Secondary (30%) | `phaze-panel` **#10141c** (dark) / `gray-50` (light); divider `phaze-border` **#232832** / `gray-200` | header, DAG rail, right pane, cards, chrome surfaces + all borders |
| Accent (10%) | `blue-500` **#00b0d8** / `blue-400` **#1abbdb** (dark) | see reserved list below |
| Semantic — success | `emerald-400` / `emerald-300` | "done"/complete stage dots, agent **online** dot, complete status text |
| Semantic — attention | `amber-400` / `amber-300` | Review & Apply group (pending-approval), agent **degraded**, k8s quota-wait (lane detail is Phase 58) |
| Destructive | `rose-400` / `red-500` | **none used in Phase 57** — the shell is navigation chrome + a skeleton modal; no destructive action exists here |

**Accent reserved for (explicit — never "all interactive elements"):**

1. **Active rail node** indicator — left inset bar `inset 3px 0 0` blue-500 + tint `bg-blue-500/10`
   (prototype `.rail-node.sel`; map `rgba(59,130,246,.12)` → `blue-500/10`).
2. **Primary action buttons** — the rail header **"+ Scan"** and any primary CTA (`bg-blue-600`, hover `bg-blue-500`, white text).
3. **Focus rings** — keyboard focus outline on rail nodes, the ⌘K button, and the modal input (`ring-2 ring-blue-500`).
4. **The wave logo mark** (existing brand SVG, blue-500/blue-400 — preserved verbatim).
5. **The "live/analyzing" dot** — `blue-400 animate-pulse` on the Analyze node (the only pulsing dot).
6. **⌘K hint chip** accent and the search affordance focus state.

Hover (non-accent): rail nodes and header controls use a subtle raised surface
`hover:bg-white/5` (dark) — the prototype's `#16213a` maps to this, NOT to an accent tint.
Idle/empty dots use `gray-600`. **Offline** agent dot = `gray-600` (dark) / `gray-400` (light).

## Header Status Strip — agent dot states (D-05, minimal)

The strip shows agent online/total status + an **Agents** link. Fed by the single existing
`/pipeline/stats` 5s poll fanned out via `hx-swap-oob` behind the `oob_counts` gate — **no new
poll loop, no new store** (consume `$store.pipeline`, e.g. `agentOnline`). No lane-capacity
cards (deferred to the Analyze workspace, Phase 58).

| State | Dot color | When |
|-------|-----------|------|
| online | `emerald-400` | agent heartbeat fresh (live count from `$store.pipeline.agentOnline`) |
| degraded | `amber-400` | agent reachable but unhealthy / quota-wait (reserve; minimal in 57) |
| offline | `gray-600` (dark) / `gray-400` (light) | no fresh heartbeat |

Dot size `w-2 h-2 rounded-full`. The **Agents link** is a Jura-tracked control
(`AGENTS · <n>`) linking to the existing `/admin/agents` route (below-the-line rail also links
it). Count text binds to the store; the dot/count refresh OOB on the poll — never via a second
`setInterval`.

**HTMX wiring (every navigable node):**

+ re-syncs the active rail node (re-applies `aria-current`/selected class). `audit` and `agents`
are below-the-line links to their own existing routes (not `/s/` stages).

## ⌘K Command Palette — skeleton modal (D-04)

Phase 57 ships the **affordance + keybinding + open/close/focus contract** with an **empty,
placeholder body**. No search or command wiring (that is Phase 61). The `/search` → ⌘K rename
redirect lands users here (research recommends `RedirectResponse("/?palette=1")` + Alpine
reading the `palette` query param on load to auto-open).

**Trigger:**
- Header **⌘K button** (Jura/blue-accented search affordance: `flex-1 max-w-md h-9 rounded-lg`,
  `phaze-panel` surface, `phaze-border`, muted placeholder + a `⌘K` mono chip on the right).
- **Keybinding:** `⌘K` (mac) / `Ctrl+K` (win/linux) — `@keydown.window` on the shell root,
  `preventDefault`.
- **ESC** closes; backdrop click closes.

**Visual (C3 / Jura-blue, from prototype `#cmd`):**
- Backdrop: `fixed inset-0 z-50 bg-black/60`.
- Panel: centered top `left-1/2 top-24 -translate-x-1/2 w-[640px] max-w-[92vw]`,
  `bg phaze-panel`, `border phaze-border`, `rounded-2xl shadow-2xl overflow-hidden`.
- Header row (`h-14`, `border-b phaze-border`): search icon + text input
  (`bg-transparent outline-none`, placeholder per copy below) + a `esc` mono hint chip.
- **Body (Phase 57 = skeleton):** a single muted placeholder line (copy below). No result rows,
  no command list, no fetch.

**Open / close / focus contract (Alpine-driven, no `@alpinejs/focus` yet — that dep lands in 61):**
- `x-data` holds `open` boolean; `role="dialog"` `aria-modal="true"` `aria-label="Command palette"`.
- On open: input receives focus (`x-init`/`$nextTick` focus, or `autofocus` on `x-show`).
- On close (ESC, backdrop, or programmatic): **return focus to the ⌘K header button**.
- ESC handled by `@keydown.escape`. Basic focus management only; full focus-trap depth is Phase 62.

## Focus / ARIA Baseline (CUT-01 minimum for this phase)

- **Skip link** → `#stage-workspace` (the stable swap target), styled as the existing
  sr-only-focus pattern in `base.html:174–176`.
- **Focus-to-heading after swap:** on `htmx:afterSwap` of `#stage-workspace` (and on
  `htmx:historyRestore`), move focus to the stage `<h1>` (`tabindex="-1"`) so keyboard/SR users
  land in the new content.
- **`aria-current="page"`** on the active rail node; re-applied by the `historyRestore` handler.
- **Modal:** `role="dialog"`, `aria-modal="true"`, focus the input on open, ESC closes, focus
  returns to the ⌘K trigger on close.
- **Fragment correctness:** stage responses are content-only (`#stage-workspace` innerHTML);
  a swapped fragment MUST NEVER contain `<html>`/`<head>`/`{% extends "base.html" %}` — the shell
  chrome (header, rail, pane, theme machinery) persists across every swap. This is both an
  architectural and an accessibility contract (no duplicate landmarks/skip-links injected).
