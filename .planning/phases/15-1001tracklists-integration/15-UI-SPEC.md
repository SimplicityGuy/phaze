# Phase 15 — UI Design Contract

> Visual and interaction contract for the 1001Tracklists Integration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS via CDN, no build step) |
| Preset | not applicable |
| Component library | none (HTMX + Alpine.js + Jinja2 partials) |
| Icon library | none (text labels and Unicode only, matching existing pages) |
| Font | Inter (400, 600) via Google Fonts CDN |

Source: `base.html` -- Tailwind Browser 4, Inter font, HTMX 2.0.7, Alpine.js 3.15.9.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between badge text and container, icon spacing, badge vertical padding |
| sm | 8px | Gap between action buttons (gap-2), compact padding |
| md | 16px | Card padding (px-4 py-4), element spacing, default gaps |
| lg | 24px | Section spacing (space-y-6), stats header padding (p-6) |
| xl | 32px | Page vertical padding (py-8 in main) |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: none

Source: Existing `duplicates/` templates use `px-4 py-4` for cards, `p-6` for stats header, `space-y-6` for page layout, `gap-2` / `gap-4` for inline elements.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (bg-white) | Page background, card backgrounds |
| Secondary (30%) | `#F9FAFB` (bg-gray-50) | Stats header background, hover states (hover:bg-gray-50) |
| Accent (10%) | `#2563EB` (bg-blue-600) | Primary CTA buttons, active nav link text, undo link in toast |
| Destructive | `#DC2626` (text-red-600) | Low confidence scores (<70%), unlink confirmation |

Accent reserved for:
- "Search Tracklists" primary CTA button (bg-blue-600)
- "Search 1001Tracklists" button on unmatched file rows (bg-blue-600)
- Active navigation link text (text-blue-600)
- Undo link in toast (text-blue-300)
- High confidence score indicator (text-green-600 for 90%+, NOT accent blue)

Additional semantic colors (from existing patterns):
- `#16A34A` (text-green-600): Confidence scores >= 90% (auto-linked threshold)
- `#CA8A04` (text-yellow-600): Confidence scores 70-89% (needs review)
- `#DC2626` (text-red-600): Confidence scores < 70% (low confidence)
- `#1E293B` (bg-slate-800): Toast background

Source: `group_card.html` uses bg-white cards with border-gray-200. `stats_header.html` uses bg-gray-50. `toast.html` uses bg-slate-800. RESEARCH.md Pattern 4 (HTMX Card) uses green/yellow/red for confidence tiers.

## Component Inventory

### Page: Tracklists List (`tracklists/list.html`)

Extends `base.html`. Structure mirrors `duplicates/list.html` exactly.

| Component | Template Path | Interaction |
|-----------|--------------|-------------|
| Stats header | `tracklists/partials/stats_header.html` | Static; shows total tracklists, matched count, unmatched count |
| Filter tabs | `tracklists/partials/filter_tabs.html` | HTMX GET swaps card list; tabs: Matched / Unmatched / All (D-22) |
| Tracklist card | `tracklists/partials/tracklist_card.html` | Alpine.js expand/collapse; HTMX lazy-loads track detail on first expand (D-20) |
| Track detail | `tracklists/partials/track_detail.html` | HTMX GET on expand; shows numbered track list with artist, title, label, timestamp |
| Card list | `tracklists/partials/tracklist_list.html` | HTMX target for pagination and filter swaps |
| Pagination | `tracklists/partials/pagination.html` | HTMX GET swaps card list; follow proposals pattern |
| Search results panel | `tracklists/partials/search_results.html` | HTMX response to manual search; ranked results with "Link Tracklist" button per result (D-09) |
| Undo toast | `tracklists/partials/toast.html` | Alpine.js 10-second auto-dismiss; "Undo" link (D-14, D-23) |

## Interaction Contract

### HTMX Interactions

| Trigger | Method | URL Pattern | Target | Swap |
|---------|--------|-------------|--------|------|
| Filter tab click | GET | `/tracklists/?filter={matched|unmatched|all}&page=1` | `#tracklists-list` | innerHTML |
| Pagination click | GET | `/tracklists/?filter={current}&page={n}` | `#tracklists-list` | innerHTML |
| Expand card | GET | `/tracklists/{id}/tracks` | `#tracks-{id}` | innerHTML |
| Link search result | POST | `/tracklists/{id}/link` | `#tracklists-list` | innerHTML |
| Unlink tracklist | POST | `/tracklists/{id}/unlink` | `#tracklists-list` | innerHTML |
| Re-scrape tracklist | POST | `/tracklists/{id}/rescrape` | `#tracklist-{id}` | outerHTML |
| Search 1001Tracklists | GET | `/tracklists/{id}/search` | `#search-panel-{id}` | innerHTML |
| Manual search (unmatched) | POST | `/tracklists/search?file_id={id}` | `#search-panel-{id}` | innerHTML |
| Undo auto-link | POST | `/tracklists/{id}/undo-link` | `#tracklists-list` | innerHTML |

### Alpine.js Interactions

| Component | x-data | Behavior |
|-----------|--------|----------|
| Tracklist card | `{ expanded: false }` | Toggle expand/collapse; trigger HTMX load on first expand (same pattern as duplicates group_card) |
| Undo toast | `{ show: true }` | Auto-dismiss after 10 seconds via `setTimeout(() => show = false, 10000)` with fade-out transition |
| Filter tabs | `{ activeTab: 'all' }` | Track active tab for styling; HTMX handles actual filtering |

### State Transitions

| User Action | Visual Feedback | Backend Effect |
|-------------|----------------|----------------|
| Click "Search 1001Tracklists" on unmatched file | Loading indicator; search results panel appears | arq task enqueued for 1001tracklists search |
| Click "Link Tracklist" on search result | Card updates with linked tracklist; success toast | Tracklist linked to file, confidence stored |
| Auto-link fires (90%+ match) | Undo toast appears for 10 seconds (D-14, D-23) | Tracklist auto-linked, `auto_linked=True` |
| Click "Undo" on toast | Toast dismissed; card reverts to unlinked state | Link removed, file_id set to NULL |
| Click "Unlink" | Card updates to show unlinked state | file_id cleared on tracklist |
| Click "Re-scrape" | Spinner on button; card refreshes with updated data | New TracklistVersion created |
| Click "Find Better Match" | Search results panel opens below card | New search triggered for this file |
| Expand card | Track detail loads inline with transition | GET tracks from latest version |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Skip link | `<a href="#tracklists-list" ...>Skip to tracklists list</a>` |
| ARIA expand | `aria-expanded` on card expand button, `aria-controls` pointing to track detail container |
| Live region | Toast container uses `aria-live="polite"` (already in base.html) |
| External link | `target="_blank" rel="noopener noreferrer"` on "View on 1001tracklists" links |
| Keyboard hint | `Keyboard: arrows to navigate, Enter to expand` footer text (matching duplicates pattern) |
| Tab filter | Filter tabs are regular links (not role="tab") for simplicity, consistent with existing patterns |
