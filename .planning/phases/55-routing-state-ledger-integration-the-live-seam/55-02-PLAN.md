<objective>
Add the `cloud_phase` admission-progression column to the `cloud_job` sidecar table (D-04 / KROUTE-03)
and wire its writers: `submit_cloud_job` seeds the initial value, and the Phase 54 reconcile cron
(`reconcile_cloud_jobs._reconcile_one`) co-writes the progression as it maps Job/Workload conditions.
This is a Wave-1 schema foundation, independent of the config rename (Plan 01) — it touches a disjoint
file set so the two run in parallel.

`cloud_phase` is the Kueue **admission progression** (queued_behind_quota → admitted → running →
finished), kept ORTHOGONAL to the existing `inadmissible` fault flag — RESEARCH is explicit that
admission progress and the fault flag are different concerns. The FileRecord state machine is
untouched (KROUTE-03 — files reuse PUSHING/PUSHED). The KROUTE-06 dashboard cards that READ this
column ship in Plan 05.

Purpose: gives the operator a per-phase admission view and the cards a backing column.
Output: migration 027, `CloudPhase` enum + column in the model, the seed in submit, the co-writes in reconcile.
</objective>
