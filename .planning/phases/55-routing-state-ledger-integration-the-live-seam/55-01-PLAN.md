<objective>
Hard-replace the `cloud_burst_enabled` boolean master toggle with a single `cloud_target:
Literal["local","a1","k8s"]` selector (default `"local"` == cloud off), per CONTEXT D-02 /
KROUTE-01. This is the FOUNDATION every other Phase 55 plan keys off — it lands in Wave 1 so the
live-seam branch (Plan 03) and backfill (Plan 04) can branch on `cloud_target`.

The single load-bearing subtlety: the two existing `_enforce_*_when_cloud_enabled` validators are
NOT both "cloud on" — they are per-target (`compute_scratch_dir` is the **a1** rsync-scratch
concern; `s3_bucket`/`s3_endpoint_url` is the **k8s** staging concern). Splitting them by
`cloud_target` — and adding a NEW `_enforce_kube_config_when_k8s` validator (pulls the K8s portion
of KDEPLOY-02 forward) — is the correct non-over-coupled rewrite. Collapsing to a single
`!= "local"` gate would silently change a1 fail-fast semantics (RESEARCH Pitfall 3).

Purpose: One setting selects off/a1/k8s; no layered toggle. The live v5.0 a1 path must be byte-for-byte
unchanged when `cloud_target="a1"`.
Output: `cloud_target` field + 3 validators in `config.py`; every production call site re-keyed; the
existing `cloud_burst_enabled` tests migrated to `cloud_target`; a new `test_cloud_target.py`.
</objective>
