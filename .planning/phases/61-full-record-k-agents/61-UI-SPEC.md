# Phase 61 — UI Design Contract

> Visual and interaction contract for the four new v7.0 surfaces: the per-file **full-record slide-in**,
> the functional **⌘K command palette**, the **Agents page** (heartbeating agents + ephemeral compute lanes),
> and the **first-run empty state**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Authority:** This contract CODIFIES already-locked decisions — it does not invent direction.
> The design system is INHERITED VERBATIM from `57-UI-SPEC.md` (C3 "Evolved phaze": Jura tracked-uppercase
> headers, Inter body, blue accent, wave logo, dark `phaze-bg` theme + light toggle). The canonical
> visual/behavioral target is `docs/superpowers/specs/2026-06-28-ui-redesign-assets/prototype.html`
> (record modal, ⌘K palette, Agents cards, first-run) and design §8/§9/§10.
> When this spec diverges from the prototype, the divergence is **required by a locked CONTEXT decision**
> (flagged inline) — the two divergences are the **agent-roots-only empty state** (D-08 drops the prototype's
> free-text path input) and the **two-section Agents page** (D-06 splits the prototype's single 3-card grid).

| Body | Inter | 14px (`text-sm`) | 400 | 1.5 | palette rows, record facts, agent card body, empty-state body copy, table cells. *12px `text-xs` monospace is used in tight numeric contexts (record path, sha256, palette `esc` chip, scan counts, timeline medians, diff code chips) — a display variant of Body, not a distinct type-scale entry (matches Phase 57's inlined 12px treatment).* |
| Eyebrow / Label | Jura | 11px (`text-[11px]`) | 500 | 1.4 | record section eyebrows (`uppercase tracking-[0.2em]`); Jura-tracked control labels (APPROVE, START SCAN, FULL ↗); **⌘K group headers** (Files/Tracklists/Artists/Commands, `text-[11px] uppercase tracking-wider text-gray-600`) |

**Chart series strokes** in the multi-lane timeline are a data-viz treatment (not text): BPM `#60a5fa`,
Key `#34d399`, Energy `#fbbf24` (blue/emerald/amber-400), `stroke-width 1.5`. Permitted exception, like the 12px mono variant.

**Weights — exactly two declared for this phase's new chrome:**

- **400 (regular)** — Inter body, the default (palette rows, record facts, agent body, empty-state body, history).
- **500 (medium)** — Jura headings/eyebrows/labels and all Jura-tracked controls (Display heading, record `<h2>`,
  section eyebrows, ⌘K group headers, APPROVE / START SCAN / OPEN FULL RECORD labels).

> **Note:** Inter **600 (semibold, `font-medium`)** already renders in the 58+ workspaces for emphasized names
> (agent-card `name`, record/pane file-name). This phase reuses that inherited treatment for the same purpose;

## Surface 1 — Full-record slide-in (RECORD-01)

(no `extends base.html`). `fixed inset-0 z-40`; backdrop `absolute inset-0 bg-black/60` (click closes);

**Sections (top→bottom, per design §8 + prototype `recordHTML`):**

| # | Section | Content | Reuse |
|---|---------|---------|-------|

| 2 | **Facts grid** | `grid-cols-4 gap-3` key/value tiles: Format · Duration · sha256 · **Lane badge** (🖥️ local / ☁️ A1 / ⎈ k8s). Tile = `kv()`: `rounded-lg bg-phaze-panel border p-…`, `text-[11px]` label + truncating value. | — |

Builds on Phase 31/57.1 windowed data. | `AnalysisResult` + `AnalysisWindow` |
| 4 | **Metadata diff** | `grid-cols-2` — left card = metadata before→after; each field a `_diff_row.html`-style row (struck-rose BEFORE → emerald AFTER). | **`_diff_row.html`** pattern |

| 6 | **Pending approvals for this file** | Amber box (`bg-amber-500/[0.06] border border-amber-500/25 p-4`), Jura amber eyebrow; **inline-approvable** rows reusing `_diff_row.html` + the SAME proposals/tags approve/undo endpoints (Phase 60). Empty → the "up to date" copy below. | **`_diff_row.html`** + proposals/tags routes |
| 7 | **History** | Jura eyebrow "History" + a mono timeline of per-file events (discovered · metadata+fingerprint · analyze dispatched…). Read-only from `ExecutionLog` + `TagWriteLog` (or `/audit/` scoped to the file — Discretion). | `ExecutionLog` / `TagWriteLog` |

**Live-update discipline (D-02 — snapshot body, counts-only OOB):** the record body renders **once on open**
(static structure — no self-refresh request, no new poll loop). Only a few bits update off the **same
`/pipeline/stats` 5s fanout** via `hx-swap-oob` behind the `oob_counts` gate: **the lane badge, this file's
stage chips, and the pending-approval count.** New OOB ids for the open record register in the existing
registry behind the gate. **Never re-render the operator's in-progress subtree** — an approval selection /
inline edit inside the open record must survive the poll (mirrors Phase 60's R-2 discipline).

**Keyboard / focus / ARIA (baseline — `@alpinejs/focus` lands here):**

- `role="dialog"` `aria-modal="true"` `aria-label` bound to the file name (e.g. `"File record: {name}"`).
- **`x-trap.inert.noscroll`** on the open panel (traps focus, `inert`-s the shell behind it, locks body scroll).
- On open: move focus to the record `<h2>` (`tabindex="-1"`); after the HTMX fragment swaps in, run
  `Alpine.initTree()` on the record container so `x-trap` + `x-data` initialise.
- **Esc** closes; backdrop click closes; the `✕` closes. On close, **return focus to the invoking file row /
  trigger** (record the opener; fall back to `#stage-workspace`).

**Error state:** a record fetch for a missing/de-duplicated file resolves **404** with a friendly fragment
(copy below), rendered inside the panel; the close/focus-return contract still applies.

## Surface 2 — ⌘K command palette (RECORD-02)

**Makes the Phase 57 skeleton (`cmdk_modal.html`) functional.** Keeps its open/close/`$nextTick`-focus/
`?palette=1`/Esc-→-`#cmdk-trigger` contract; adds grouped results, commands, arrow-nav, and `x-trap`.

**Layout (inherited from the 57 skeleton + prototype `#cmd`):** `fixed inset-0 z-50`; backdrop `bg-black/60`;
panel `absolute left-1/2 top-24 -translate-x-1/2 w-[640px] max-w-[92vw] bg-phaze-panel border rounded-2xl
shadow-2xl overflow-hidden`. Header row `h-14 border-b`: search icon + `x-ref="input"` + `esc` mono chip.
Results body `max-h-[50vh] overflow-y-auto p-2 text-sm`.

**Grouped results + commands (D-03/D-05):** four labeled groups, each header a Jura Eyebrow/Label
(`text-[11px] uppercase tracking-wider text-gray-600`) line; rows are `rail-node`-style (`rounded-lg px-3 py-2
flex items-center gap-3`, leading emoji + flex-1 label + trailing muted status).

| Group | Source | Row → action |
|-------|--------|--------------|
| **Files** | `search_queries.search()` (result_type=file) | 🎵 name + status hint (e.g. "analyzing · k8s") → **open the full-record slide-in** |
| **Tracklists** | `search_queries.search()` (result_type=tracklist) | 📜 name + "{n} tracks" → open its tracklist workspace |
| **Artists** | **read-only `SELECT DISTINCT` on `FileMetadata.artist` / `Tracklist.artist`** (D-05 — the one sanctioned additive read query) | 👤 artist + optional file count → navigate `files?artist=X` (existing `artist=` param) |
| **Commands** | static | `⊕ Scan` (existing scan trigger via `enqueue_router`) · `→ Jump to a stage` (`/s/<stage>`) · `✓ Jump to a review queue` (a Phase 60 gate) · `⎈ Open Agents` |

Discogs-release results: **Discretion** — either surface as a group or keep to the three named groups + Commands.

**Full keyboard nav (D-04):**

- Typing debounce-fetches results via an HTMX GET that **swaps the results body only** (not the header/input);
  on new results the active index resets to the **first selectable row**.
- **↑ / ↓** move a single **active row across ALL groups** (roving index over the flat ordered list; group
  headers are skipped, not selectable). The active row wraps at the ends and auto-scrolls into view.
- **Enter** activates the active row (open record · run command · navigate).
- **Esc** closes and **returns focus to `#cmdk-trigger`** (existing contract).

**ARIA (listbox pattern):** the input is `role="combobox"` `aria-expanded` `aria-controls="{listbox}"`
`aria-activedescendant="{active row id}"`; the results container is `role="listbox"`; each selectable row is
`role="option"` with `aria-selected` on the active one; group headers are `role="presentation"` (or
`aria-hidden`). The dialog wrapper keeps `role="dialog" aria-modal="true" aria-label="Command palette"`.

**Focus-trap:** replace the skeleton's basic focus management with **`x-trap.inert.noscroll`** on the open panel.

**Empty / no-results states:** with an empty query, show the **Commands** group (optionally recent files);
with a non-matching query show the "no matches" copy below (no error styling — muted text).
