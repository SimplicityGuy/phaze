<objective>
Add a boot-time, static, id-tagged fail-fast when two compute backends bind the same `agent_ref`
(D-04) — a cross-entry invariant that lives in the container validator `_validate_registry`, mirroring
the existing `Counter`-based duplicate-bucket-id guard (REG-05 idiom). Per D-05 this is a STATIC config
check only: an `agent_ref` naming a not-yet-registered agent is NOT a boot error (agents register
dynamically via check-in), so the validator performs NO DB existence check — that path degrades to a
runtime hold (Plan 03).

Purpose: Retiring the `≤1-compute` fail-fast (Plan 02) means two compute entries pointing at the SAME
agent would silently double-bind; this replaces the retired blanket raise with a precise duplicate guard,
keeping the "fail loud on genuine mis-config" property while allowing N distinct compute agents.
Output: a ~4-line Counter guard in `_validate_registry` + positive/negative registry tests.
</objective>
