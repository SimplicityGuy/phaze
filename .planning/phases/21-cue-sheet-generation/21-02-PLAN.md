    **Helper: `_get_cue_stats(session)`**
    Query counts:
    - `eligible`: Tracklists WHERE status='approved' AND file_id IS NOT NULL, joined to FileRecord WHERE state='executed', having at least one TracklistTrack with non-null timestamp
    - `generated`: Count of eligible tracklists whose file's current_path has a matching .cue file on disk (use `pathlib.Path(current_path).with_suffix('.cue').exists()`)
    - `missing_timestamps`: Tracklists WHERE status='approved' AND file_id IS NOT NULL AND file.state='executed' but NO tracks have timestamps

    **Helper: `_get_eligible_tracklists(session, page, page_size)`**
    Query approved tracklists with file_id not null, join FileRecord where state=EXECUTED. Eager load latest TracklistVersion -> tracks. Return paginated list with CUE status computed per tracklist.

    **Helper: `_build_cue_tracks(tracks, discogs_links_by_track_id)`**
    Convert TracklistTrack list + DiscogsLink dict into list[CueTrackData]:
    - For each track: parse_timestamp_string(track.timestamp) for timestamp_seconds
    - Look up accepted DiscogsLink by track.id for genre (discogs_label as label proxy), label (discogs_label), year (discogs_year)
    - Note: DiscogsLink does not have a genre field. Use discogs_label for REM LABEL and discogs_year for REM YEAR. Skip REM GENRE unless the DiscogsLink model has genre data (it does not -- so genre will be None for all tracks). Per D-09, no placeholder REM lines.

    **Helper: `_get_cue_version(file_path)`**
    Check filesystem for existing .cue files matching the audio file stem. Return version number (0 if no CUE exists, 1 if base.cue exists, N for highest .vN.cue).

    **Endpoint: `GET /cue/`** (per D-05)
    - Query stats and eligible tracklists (page 1, 20 per page)
    - For each tracklist, compute CUE status: "none", "generated" (with version), "not_eligible"
    - If HX-Request header truthy: return cue_list.html partial
    - Else: return full cue/list.html with current_page="cue"

    **Endpoint: `POST /cue/{tracklist_id}/generate`** (per D-04, D-07)
    - Load tracklist with eager file, latest version -> tracks
    - Validate: tracklist.status == "approved" (D-06), file exists, file.state == FileState.EXECUTED (D-10)
    - Validate: at least one track has non-null timestamp
    - Load accepted DiscogsLinks for all track IDs in the version
    - Build CueTrackData list via _build_cue_tracks
    - Call generate_cue_content(Path(file.current_path).name, file.file_type, cue_tracks)
    - Call write_cue_file(content, Path(file.current_path))
    - Return updated tracklist row partial + OOB toast: "CUE file generated: {written_path.name}"
    - On validation failure: return OOB toast with appropriate error message from UI-SPEC copywriting contract

    **Endpoint: `POST /cue/generate-batch`** (per D-05)
    - Query all eligible tracklists (approved, file EXECUTED, has timestamps)
    - For each, run same generation logic as single generate
    - Return updated cue_list.html + OOB toast: "Generated {count} CUE files"
