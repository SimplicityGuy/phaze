<objective>
Make reconcile cap-safe and single-owner. (1) `KueueBackend.reconcile` acquires the drain's advisory lock `5_000_504` per-row (the granularity `pg_advisory_xact_lock` requires, since reconcile commits per row) so a reconcile row-mutation and a drain tick are mutually exclusive (SCHED-02). (2) The reconcile cron loop is replaced by per-backend dispatch `for b in resolve_backends(cfg): await b.reconcile(...)`, removing the global un-scoped `cloud_job` query — compute rows stay owned only by their `/pushed` callback (SCHED-05). (3) The at-cap terminal flips from `ANALYSIS_FAILED` to spill-back to `AWAITING_CLOUD` (SCHED-03/D-04) — local is the guaranteed safety net, so a cloud-flaky file never hard-fails.

Purpose: Close the SCHED-02 lock-ordering correctness gap, remove the double-owner vector (SCHED-05), and stop hard-failing processable files on cloud flakiness (SCHED-03).
Output: cap-safe backend-scoped reconcile + spill-back + tests.
</objective>
