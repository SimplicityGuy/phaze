<objective>
Re-key both push callbacks off the RECORDED `cloud_job.backend_id` (record-don't-rederive) so a file's terminalization, scratch dir, and compute queue are attributed to the agent it was dispatched to — the MCOMP-06 no-cross-attribution property. Two landmines become explicit tasks: (1) `/mismatch` must re-stamp `dest_*` onto its re-driven payload, and (2) the D-07 reporter check applies where the reporter is a compute agent.

Purpose: `/pushed` fixes Pitfall 4 (route `process_file` to the file's OWN compute agent, not `select_active_agent`). `/mismatch` adds D-07 reporter validation (its reporter IS the compute agent) + the Landmine-1 destination re-stamp.

Output: backend_id-scoped `/pushed`, reporter-validated + re-stamped `/mismatch`.

**IMPORTANT reporter-identity finding (read before implementing):** The two callbacks have DIFFERENT reporters. `/pushed` is reported by the FILESERVER agent (`tasks/push.py:205` `api.report_pushed` runs inside `push_file` on the fileserver after rsync). `/mismatch` is reported by the COMPUTE agent (`tasks/functions.py:292/298` `api.report_push_mismatch` runs inside `process_file` on the compute agent after sha256/scratch verification). Therefore the D-07 equality check `reporter.id == backend.agent_ref` is CORRECT and meaningful on `/mismatch` (compute reporter) but would REJECT the legitimate fileserver reporter on `/pushed`. Implement D-07's reporter equality gate on `/mismatch` only; deliver `/pushed`'s MCOMP-06 property via record-don't-rederive routing (resolve the compute agent/scratch/queue from the recorded backend_id). This is flagged to the operator in the planning return.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compute agent → /mismatch | untrusted, token-authed callback; a stale/wrong/duplicate compute agent could report for another agent's file |
| fileserver agent → /pushed | token-authed callback from the pusher; routing decision must use the recorded backend, not "the active compute agent" |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-07 | Spoofing / Elevation | /mismatch reporter mis-attributing another agent's file (the MCOMP-06 security core, D-07) | mitigate | resolve cloud_job.backend_id → ComputeBackend.agent_ref; `agent.id != backend.agent_ref` → 403 + no terminalize; never re-stamp backend_id from the token (Task 2) |
| T-73-08 | Tampering / DoS | Wrong-agent process_file routing finds no scratch copy (retry storm) | mitigate | /pushed routes process_file to the RECORDED backend.agent_ref, not select_active_agent (Pitfall 4, Task 1) |
| T-73-09 | Tampering | /mismatch re-drive pushing to a null/empty destination | mitigate | Landmine 1: re-driven payload stamps dest_* from the recorded backend (Task 2) |
| T-73-10 | Information disclosure | backend_id / agent identity in logs | mitigate | log only {file_id, reporter, expected} id projections — no SecretStr/token (existing discipline) |
| T-73-SC | Tampering | npm/pip/cargo installs | accept | Zero new dependencies; pyproject untouched |
</threat_model>
