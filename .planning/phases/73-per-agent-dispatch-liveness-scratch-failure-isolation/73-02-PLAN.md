<objective>
Switch the fileserver's rsync push destination from the single-global `cfg.push_ssh_*`/`cfg.cloud_scratch_dir` reads to the payload-carried per-backend destination stamped in Plan 01. This is the fileserver leg of MCOMP-03: N compute agents each get files pushed to their OWN host/scratch, resolved per file from the payload.

Purpose: Complete record-don't-rederive on the transport side — `_build_rsync_argv` reads the recorded destination. This is D-04 in practice: retire the fileserver's single push destination env — the fileserver stops reading `push_ssh_host` + `cloud_scratch_dir` as the remote target (superseded by the payload; no ≤1 fallback), while the compute agent's own local `cloud_scratch_dir` field stays on AgentSettings (its janitor still reads it locally).

Output: payload-driven `_build_rsync_argv`, reduced `_require_push_config` required set (secrets kept).
</objective>
