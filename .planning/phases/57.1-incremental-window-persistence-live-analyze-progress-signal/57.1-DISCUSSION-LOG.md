## Counter-only vs full window rows mid-flight (PROG-01)

| Option | Description | Selected |
|--------|-------------|----------|
| Counter-only | Mid-flight advances only fine_windows_analyzed/total; detail rows stay atomic at completion. Trivial PROG-02 idempotency. | ✓ |
| Full window rows incrementally | Write each analysis_window detail row live; real partial-row reconciliation on crash. | |
| Counter + rows incrementally | Both; max fidelity, max cost/complexity. | |

**User's choice:** Counter-only
**Notes:** The in-flight bar needs only a count. Preserves Phase 31/32 atomicity for detail rows. Reframes PROG-01 → "progress-counter persistence" (reconciliation noted for planner).

## Transport across local/A1/k8s lanes (PROG-01)

| Option | Description | Selected |

## Update cadence / throttle (PROG-01)

| Option | Description | Selected |
|--------|-------------|----------|
| Every window, time-throttled (~5s), always flush final | Per-window callback, skip POST if <~5s since last; always send final. | ✓ |
| Every window, unthrottled | POST after every window; ≤~90/file; chatty on short files. | |
| Every N windows | Batch every N; visible jumps; force final flush. | |

**User's choice:** Every window, time-throttled (~5s), always flush final
**Notes:** Cheap given Phase 31 caps (≤~90 windows/file); throttle avoids burst on short files. Interval is a 3–5s tunable.
