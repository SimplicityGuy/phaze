Wire the per-window progress signal end-to-end through the ONE shared seam and BOTH non-A1 lanes (local + A1 share the pebble path; k8s is separate), per the transport decision recorded in `57.1-01-SPIKE-FINDINGS.md`.

The lane owns transport: the pebble lanes bridge a picklable queue → parent drainer → `agent_client.post_analysis_progress` (throttled ~5s parent-side, kill-safe teardown); the k8s lane moves the blocking `analyze_file` into `asyncio.to_thread` so the loop is free for `run_coroutine_threadsafe` POSTs. Ships **fine-only** counts. A dropped progress POST is best-effort — the completion `put_analysis` already writes the final count, so the bar reaches 100% regardless (D-04 final-flush belt-and-suspenders).
