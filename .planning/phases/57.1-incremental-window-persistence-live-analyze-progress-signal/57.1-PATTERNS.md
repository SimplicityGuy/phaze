### `src/phaze/routers/agent_analysis.py` — NEW `post_analysis_progress` handler (route, request-response)

**Analog:** `put_analysis` (`agent_analysis.py:125-234`) — copy the upsert *mechanism* and auth shape, but STRIP everything past the aggregate upsert (no window replace, no `FileState` flip, no ledger clear, no S3 delete). This is a **sibling handler, not a call into `put_analysis`** (RESEARCH Pattern 1 decision).

**Imports / router (lines 29-56)** — already present in the file; the new handler reuses them verbatim:
```python
from typing import Annotated

from sqlalchemy.dialects.postgresql import insert as pg_insert
from sqlalchemy.ext.asyncio import AsyncSession
from phaze.models.analysis import AnalysisResult
from phaze.routers.agent_auth import get_authenticated_agent
```

**Auth + handler signature** (copy from `put_analysis` `:125-131` and `report_analysis_failed` `:237-243`):
```python
@router.post("/{file_id}/progress", status_code=status.HTTP_200_OK, response_model=AnalysisProgressResponse)
async def post_analysis_progress(
    file_id: uuid.UUID,                                              # PATH only — AUTH-01
    body: AnalysisProgressPayload,
    agent: Annotated[Agent, Depends(get_authenticated_agent)],      # agent_id from dep, NEVER body
    session: Annotated[AsyncSession, Depends(get_session)],
) -> AnalysisProgressResponse:
```

**Core upsert pattern — COUNTER-ONLY** (lift the `pg_insert…on_conflict_do_update` idiom from `:180-196`, but the SET clause covers only the two/four count fields and the PK stamp; NO `windows`, NO `mood/style` summary, NO overflow funnel):
```python
payload = {
    "fine_windows_analyzed": body.fine_windows_analyzed,
    "fine_windows_total": body.fine_windows_total,
    "file_id": file_id,
    "id": uuid.uuid4(),          # AnalysisResult.id default is Python-only; pg_insert bypasses it (:180-182)
}
stmt = pg_insert(AnalysisResult).values([payload])
stmt = stmt.on_conflict_do_update(
    index_elements=["file_id"],   # the analysis.file_id UQ natural key (models/analysis.py:18)
    set_={
        "fine_windows_analyzed": stmt.excluded.fine_windows_analyzed,
        "fine_windows_total": stmt.excluded.fine_windows_total,

await session.execute(stmt)
await session.commit()           # NO FileRecord update, NO AnalysisWindow delete/insert, NO ledger/S3
return AnalysisProgressResponse(agent_id=agent.id, file_id=file_id)
```

- `:204-211` window delete/insert — progress writes NO detail rows (D-01).
- `:219-220` `update(FileRecord)...state=FileState.ANALYZED` — progress NEVER flips state (KEY RISK).
- `:227` `clear_ledger_entry(...)` and `:231` `_delete_staged_object_if_cloud(...)` — completion-only side effects.

### `src/phaze/services/agent_client.py` — NEW `post_analysis_progress` method (service)

**Analog:** `post_exec_batch_progress` (`:520-540`) — the exact best-effort-POST-returns-None shape; also `heartbeat` (`:542-549`) and `put_analysis` (`:274-283`).

**Pattern** (lazy schema import per the file convention `# noqa: PLC0415`; goes through the `self._request` funnel so it inherits the tenacity retry policy D-11 + exception hierarchy D-12 for free — do NOT hand-roll a retry loop):
```python
async def post_analysis_progress(self, file_id: uuid.UUID, payload: AnalysisProgressPayload) -> None:
    """POST /api/internal/agent/analysis/{file_id}/progress -- counter-only mid-flight bump (Phase 57.1).

    Best-effort: swallow AgentApiError after retries (D-16) at the CALL SITE — a dropped
    progress POST must never fail the job; the completion put_analysis writes the final count.
    file_id rides the path only (AUTH-01); no body agent_id.
    """
    await self._request(
        "POST",
        f"/api/internal/agent/analysis/{file_id}/progress",
        json=payload.model_dump(mode="json"),
    )
    return None
```
> `post_exec_batch_progress`'s docstring (`:525-534`) is the template for the "caller swallows AgentApiError after retries (D-16)" discipline — copy that wording.

```python
    fine_windows: list[FineWindow] = []

        try:
```

### `src/phaze/tasks/functions.py` — local + A1 lane bridge (task, event-driven)

**Analog:** `process_file` (`:89-240`), specifically the `run_in_process_pool` call (`:165-174`) and the `asyncio.to_thread(compute_sha256, ...)` offload (`:130`). `ctx["api_client"]` (`:97`) is the parent-side client; `ctx["process_pool"]` (`pool.py:44`) is the pebble pool.

**Constraint (Pitfall 4 / `pool.py:50`):** `pool.schedule(func, args=..., kwargs=...)` **pickles** args to the child. You CANNOT pass a closure or `agent_client` as `progress_cb`. Pass a **picklable channel** (`multiprocessing.Manager().Queue()` proxy); the child cb does `q.put_nowait((analyzed, total))`; a **parent-side drainer** task reads it and calls `ctx["api_client"].post_analysis_progress(...)`. **Throttle (D-04 ~5s, `monotonic()` keyed) + final flush live parent-side, in the drainer.** The drainer MUST be kill-safe: terminate on the child future's completion (sentinel) so a SIGKILLed child never hangs it (Assumption A2 — the #1 spike unknown).

The best-effort swallow mirrors the existing `report_analysis_failed` error discipline (`:216-232`): a progress POST failure must never fail the file.

### `src/phaze/job_runner.py` — k8s one-shot lane bridge (runner, event-driven)

**Analog:** same file — `compute_sha256` is ALREADY offloaded via `asyncio.to_thread` (`:203`); `client = construct_agent_client(cfg)` (`:176`) is the loop-side client.

**Constraint (Pitfall 3 / `:218-221`):** today `analyze_file(...)` runs **directly inside `async def run`, blocking the loop** — a progress coroutine scheduled onto it cannot run until analysis returns. **Fix:** wrap the analyze call in `asyncio.to_thread(analyze_file, str(tmp_path), models_dir, fine_cap=..., coarse_cap=..., progress_cb=cb)` (exactly the `:203` offload pattern) so the loop stays free; the cb does fire-and-forget `run_coroutine_threadsafe(client.post_analysis_progress(...), loop)` with errors swallowed. Capture `loop = asyncio.get_running_loop()` before the `to_thread` call. Preserve the existing exit-code contract: progress failures are best-effort and must NOT change the `EXIT_ANALYSIS`/`EXIT_CALLBACK` mapping (`:218-251`).

### `src/phaze/services/pipeline.py` — convergence-gate KEY-RISK fix (service, CRUD read)

**Analog:** `get_proposal_pending_batches` itself (`:1104-1126`). The leak is at `:1122`:
```python
.where(exists(select(AnalysisResult.id).where(AnalysisResult.file_id == FileRecord.id)))
```
Pre-57.1 an `analysis` row existed only at completion, so `exists(analysis)` was a sound "analysis complete" proxy. D-03 upserts a **partial** row at START — a `METADATA_EXTRACTED` file with a partial row now satisfies BOTH gate clauses and leaks into `generate_proposals` with NULL aggregates.

**Fix (RESEARCH §Q4 recommended — requires migration 028):** tighten the analysis-half to require the completion discriminator:
```python
.where(exists(select(AnalysisResult.id).where(
    AnalysisResult.file_id == FileRecord.id,
    AnalysisResult.analysis_completed_at.isnot(None),   # in-flight rows have completed_at IS NULL
)))
```
**Mandatory regression test:** a `METADATA_EXTRACTED` file + a partial analysis row (analyzed<total, NULL bpm) returns in **zero** batches (analog: `tests/test_services/test_proposal_queries.py`).

> **Search/sort (LOW risk, `search_queries.py:71,83-84`):** an `outerjoin(AnalysisResult)` + `where(AnalysisResult.bpm >= bpm_min)` already excludes a partial row (NULL bpm comparison → unknown→false). No code change — add an assertion to LOCK the invariant.

## Shared Patterns

### Bearer auth (apply to the new endpoint)
**Source:** `src/phaze/routers/agent_auth.py:62-84` (`get_authenticated_agent`)
**Apply to:** the new `post_analysis_progress` route — identical to every `/api/internal/agent/*` handler.
```python
agent: Annotated[Agent, Depends(get_authenticated_agent)]   # 401 missing bearer, 403 unknown/revoked
```
`agent_id` comes from `agent.id` (the dep), NEVER the body; `file_id` rides the PATH only (AUTH-01). `extra="forbid"` on the payload makes a forged `agent_id`/`file_id` a 422.

### Idempotent counter upsert (apply to the new endpoint)
**Source:** `src/phaze/routers/agent_analysis.py:180-196` (the `pg_insert…on_conflict_do_update` idiom)
**Apply to:** the progress upsert — single race-free statement on the `analysis.file_id` UQ; PK `id` stamped explicitly (`AnalysisResult.id` default is Python-only, `pg_insert` bypasses it).

### Best-effort agent→control POST through the retry funnel
**Source:** `src/phaze/services/agent_client.py:520-540` (`post_exec_batch_progress`) — `await self._request("POST", ...); return None`, caller swallows `AgentApiError` after retries (D-16).
**Apply to:** `post_analysis_progress` and BOTH lane bridges — a dropped progress POST must never fail the job; the completion `put_analysis` writes the final count regardless (the bar reaches 100% from completion).

### Off-loop offload (apply to the k8s lane)
**Source:** `src/phaze/job_runner.py:203` (`await asyncio.to_thread(compute_sha256, tmp_path)`)
**Apply to:** wrapping the blocking `analyze_file` call so the event loop stays free for the cb's `run_coroutine_threadsafe` POSTs.
