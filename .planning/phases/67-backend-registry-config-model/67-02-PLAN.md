## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process env → config layer | PHAZE_BACKENDS_CONFIG_FILE pointer + tomllib file read |
| assembled registry → startup log sink | Effective-registry projection logged at boot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-02-01 | Denial of service | present-but-empty / silently-empty registry (Phase-30 class) | mitigate | Container `model_validator` fails fast on empty resolved registry; absent-file → implicit-local (never a wedged empty backlog) |
| T-67-02-02 | Spoofing / Tampering | cluster-specific bucket reachable by wrong cluster | mitigate | Scope cardinality invariant: `cluster-specific` ≤1 kueue backend, fail-fast on 2 (D-09) — a correctness/security control, not a label |
| T-67-02-03 | Information disclosure | secret leakage via effective-registry startup log | mitigate | `log_effective_registry` logs an `id/kind/rank/cap` projection only; never a whole model, SecretStr, or `*_file` mount path |
| T-67-02-04 | Tampering | two config sources fighting over `backends` | mitigate | `backends`/`buckets` sourced ONLY from the TOML file; not exposed as env/JSON (Pitfall 6) |
| T-67-02-SC | Tampering | package installs | accept | Zero new dependencies (stdlib tomllib + pinned pydantic); no install task |
</threat_model>
