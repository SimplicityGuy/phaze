| config layer → S3/kube dispatch services | Bucket identity/creds and cluster connection/manifest config crossing into aioboto3/kr8s clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-04-01 | Elevation of privilege / scope creep | Backend-protocol dispatch leaking into Phase 67 | mitigate | Rewire flat reads to the transitional accessor ONLY; introduce NO Backend type/protocol; the accessors reduce the ≤1-non-local registry and are never exercised all-local (Pitfall 1) |
| T-67-04-02 | Spoofing / Tampering | wrong-cluster bucket reachability | mitigate | active_bucket resolves the single kueue backend's assigned bucket; the Plan-02 scope-cardinality invariant already blocks a cluster-specific bucket bound to two clusters (D-09) |
| T-67-04-03 | Information disclosure | S3/kube credentials in logs | mitigate | Creds stay `SecretStr` on active_bucket/active_kube; `.get_secret_value()` only at the client-construction call, never logged (existing T-53-01/T-54-01 discipline preserved) |
| T-67-04-04 | Denial of service | multi-backend/multi-bucket ambiguity silently picking wrong target | mitigate | active_kube/active_bucket raise (not silently pick) on >1 non-local backend / >1 resolved bucket (Plan 02) — fail-fast until Phase 69/70 |
| T-67-04-SC | Tampering | package installs | accept | Zero new dependencies; no install task |
</threat_model>
