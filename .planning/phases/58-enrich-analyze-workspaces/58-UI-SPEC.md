# Phase 58 — UI Design Contract

> Visual and interaction contract for the first four real stage **workspaces** that live inside the
> v7.0 shell: **Discover · Metadata · Fingerprint · Analyze**. Generated by gsd-ui-researcher.
>
> **Authority & inheritance:** This contract **INHERITS `.planning/phases/57-shell-dag-rail/57-UI-SPEC.md`
> verbatim** as its baseline design system. Spacing, typography, color tokens, the shell chrome, the
> DAG rail, the header status strip, the ⌘K modal, the OOB single-poll fanout, `$store.pipeline`, and
> the focus/ARIA baseline are all **locked in Phase 57 and are NOT re-derived here.** This spec
> specifies ONLY what is NEW for the four workspaces.
>
> The canonical visual/behavioral target remains
> `docs/superpowers/specs/2026-06-28-ui-redesign-assets/prototype.html` (the `head()` / `table()` /
> `lane()` helpers and the `discover` / `metadata` / `fingerprint` / `analyze` screen functions).

> the production C3 tokens (`phaze-bg` / `phaze-panel` / `phaze-border`, `blue-500`), **the production
> tokens win** — "evolution, not reskin" (the same rule Phase 57 established).

## NEW Pattern 1 — Workspace Scaffold (the reusable shell every workspace uses)

Every stage workspace renders a **fragment** into `#stage-workspace` (content-only, `{% include %}`,
NEVER `{% extends %}` — the Phase-57 fragment contract). The fragment is built from a shared scaffold
that mirrors the prototype's `head()` + body composition:

```
<section>                                    {# the workspace fragment root #}
  <header px-6 py-4 border-b phaze-border>    {# WORKSPACE HEAD #}
    <div>
      <h1 tabindex="-1" Jura 18 uppercase tracking-[0.15em]>{TITLE}</h1>   {# focus target #}
      <p Inter 14 text-gray-500>{LIVE SUB-COUNT}</p>                       {# x-text on $store.pipeline #}
    </div>
    <div flex gap-2>{STAGE ACTION BUTTONS}</div>                          {# right-aligned #}
  </header>
  {BODY}                                       {# lane grid (Analyze) and/or file table #}
</section>
```

**Contract:**
- The `<h1>` is the **focus landing target** after a rail swap (Phase-57 `htmx:afterSwap` /
  `htmx:historyRestore` handler moves focus here). Exactly one `<h1>` per workspace.
- The **live sub-count** binds via `x-text` to existing `$store.pipeline` keys (e.g. Discover →
  `discovered`; Metadata → `metadataDone` / `metadataTotal`; Analyze → `analyzeActive`). It refreshes
  on the single OOB poll — **no per-workspace timer, no new store key** (WORK-05; Phase-57 store rule).
- **Stage-action buttons** are the Phase-57 **secondary control** style (NOT the blue primary): Jura
  uppercase `tracking-wider`, `h-9 px-3 rounded-lg`, `bg phaze-panel`, `border phaze-border`,
  `hover:bg-white/5`, `text-gray-300`. The blue primary `bg-blue-600` style is reserved for "+ Scan"
  (rail) per Phase 57 — workspace actions are secondary.

The capacity numeral uses `font-medium` (Inter **500**) — NOT `font-semibold` — to stay within the
two-weight Phase-58 contract (see Typography note).

| Lane card | Border | Bar / capacity color | Capacity value (live) | Sub-label |

## Copywriting Contract

Per-workspace, operator-facing, terse — Jura uppercase eyebrows/labels + lowercase-sentence body (the
C3 voice). Live numerals come from `$store.pipeline`; the strings below are the static frame.

### Primary actions (stage-action buttons)

| Workspace | Buttons (Jura uppercase) |
|-----------|--------------------------|
| Discover | **SCAN** · **RECOVER** |
| Metadata | **EXTRACT SELECTED** · **EXTRACT ALL** |
| Fingerprint | **FINGERPRINT ALL** |
| Analyze | **ROUTE RULES** · **PAUSE** |

> **Single-word CTA decision (SCAN · RECOVER · PAUSE):** These three are deliberately left as bare
> verbs, with no noun object, under the locked **C3 operator-console voice** (terse, machine-room
> labels on a single-user admin surface). In their on-screen context the object is unambiguous and
> already named by the workspace `<h1>` and the adjacent stage rail node — `SCAN` acts on the Discover
> stage's scan target, `RECOVER` on its orphaned-work backlog, `PAUSE` drains the Analyze stage. None
> is a generic/ambiguous label (no bare "Submit"/"OK"/"Click here"), and the verb-only form keeps the
> stage-action button row scannable at a glance, matching the prototype's terse `head()` action chips.
> This is a deliberate, documented exception to the verb+noun CTA convention for these specific
> operator controls — not an oversight. The longer two-word actions (`EXTRACT SELECTED`/`EXTRACT ALL`,
> `FINGERPRINT ALL`, `ROUTE RULES`) keep their objects because their scope (selected-vs-all, rules) is
> NOT otherwise visible from context.
