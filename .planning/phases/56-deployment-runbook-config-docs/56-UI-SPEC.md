# Phase 56 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note (read first)

This is an **ops-only phase**. The UI footprint is deliberately tiny and **fully locked in
`56-CONTEXT.md`**. There are exactly **two** UI touch-points, both of which **reuse existing
partials and tokens** — this phase introduces **zero new design tokens, zero new components,
and zero new interactive controls**.

1. **LocalQueue-unreachable alert (D-05)** — a new alert *state* on the existing Phase 54
   cloud-state card family. Reuses the `inadmissible_card.html` amber-alert pattern verbatim,
   driven by a new degrade-safe flag (mirror of `cloud_job.inadmissible`). Rendered on the
   pipeline DAG dashboard; pushed via `hx-swap-oob` on the existing 5s `/pipeline/stats` poll.
2. **Ephemeral k8s-lane note (D-07)** — a single **static informational note** on the Agents
   page (`admin/agents.html`). No data binding, no poll, no pill. PLUS the guarantee that the
   k8s lane never registers as a heartbeating `Agent`, so the 5-state `classify()`
   (`services/agent_liveness.py:68`) **never renders a perpetually-DEAD pill** for it.

Everything else in Phase 56 is docs/runbook/config with **no UI** (cluster-admin runbook,
homelab change-prompt, `docs/k8s-burst.md`, `docs/configuration.md`, `docs/deployment.md`,
the LocalQueue startup probe code, config validators). No UI is invented for those.

The **full synthetic "k8s burst" identity card** with workload-derived liveness is
**DEFERRED to v7.0 RECORD-03** — explicitly out of scope here.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (server-rendered Jinja2 + HTMX 2.x + Tailwind CSS via CDN + Alpine.js) |
| Preset | not applicable (no shadcn; React/Next/Vite gate does not apply) |
| Component library | none — Jinja partials under `src/phaze/templates/**` are the component layer |
| Icon library | none — inline Unicode glyphs only (`⚠` for alerts, `ℹ` optional for the info note), matching `inadmissible_card.html` |
| Font | Jura (brand display: logo/nav — `.font-jura`), Inter (body/UI — `body`), both via Google Fonts CDN |

Aesthetic = C3 "Evolved phaze": Jura display font, blue accent (`#00b0d8`), wave logo, dark
theme. v6.0 must not diverge. Existing in-repo partials are the source of truth for all tokens.

## Spacing Scale

Tailwind 4px-based scale (already in use across the cloud-state cards and Agents page). This
phase reuses these exact utilities; it adds no new spacing values.

| Token | Value | Tailwind | Usage (existing) |
|-------|-------|----------|------------------|
| xs | 4px | `space-y-1`, `mt-1`, `px-1` | Tight stacking inside alert bodies |
| sm | 8px | `mt-2`, `px-2`, `gap-2` | Pill horizontal padding, meta gaps |
| (md-) | 12px | `space-y-3`, `py-3` | Card vertical rhythm, table cell padding |
| md | 16px | `p-4` | Card padding, table cell horizontal padding |
| lg | 24px | `space-y-6` | Page-level section stacking (`agents.html`) |

Exceptions: `py-0.5` (2px) — vertical padding on the 5-state status pill ladder. This is an
**inherited** project-wide pill geometry (`_status_pill.html`), NOT introduced by this phase.

## Typography

All sizes/weights are inherited from existing partials. This phase adds no new type roles.

| Role | Size | Weight | Line Height | Where (existing) |
|------|------|--------|-------------|------------------|
| Page heading | 24px (`text-2xl`) | 600 (`font-semibold`) | tight (`leading-tight`) | `agents.html` `<h1>` |
| Card heading | 18px (`text-lg`) | 600 (`font-semibold`) | default | alert/card `<h2>` |
| Body | 14px (`text-sm`) | 400 (`normal`) | default | card body `<p>`, note copy |
| Meta / pill | 12px (`text-xs`) | 600 (`font-semibold`) | default | pill labels, footers |

Weights: **normal (400)** for body + **semibold (600)** for headings/labels are the two
primary weights. `font-bold` (700) is reserved for the large count numerals in the
sky/violet count cards (existing pattern) and is NOT used by either Phase 56 touch-point.

## Color

Inherited palette. 60/30/10 split maps to the established dark theme; semantic hues come from
the Phase 49/50/54 cloud-state cards.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0c12` (`phaze-bg`) / `#ffffff` (light) | Page background, surfaces |
| Secondary (30%) | `#10141c` (`phaze-panel`), borders `#232832` (`phaze-border`) | Cards, table rows, note panel |
| Accent (10%) | `#00b0d8` (`blue-500`/`blue-600`) | Brand accent, focus rings, skip links, info-note leading glyph |
| Warning/alert | `amber-50/950` bg, `amber-300/700` border, `amber-700/800` text | LocalQueue-unreachable alert (reuse `inadmissible_card.html`) |
| Destructive | `red-100/950` bg, `red-700/400` text | NOT used in this phase (no destructive actions) |

Accent reserved for: brand logo/wordmark, primary navigation active state, keyboard focus
rings, skip links, and the optional `ℹ` leading glyph / accent text on the static info note.
NOT applied to the alert (alert is amber=warning, per the inadmissible precedent).

**Severity color decision (locked):**
- LocalQueue-unreachable **alert** = **amber** warning hue, identical to `inadmissible_card.html`
  (it is the same "Kueue misconfiguration" failure class). NOT red — the app stays up (D-05
  is warn+surface, non-fatal).
- Ephemeral-lane **note** = **neutral** info treatment: `border border-gray-200
  dark:border-phaze-border rounded-lg p-4`, body `text-sm text-gray-600 dark:text-gray-400`,
  optional `ℹ` glyph in `text-blue-600 dark:text-blue-400`. Deliberately NOT amber/red so it
  reads as steady-state information, never as an error. Matches the page intro `<p>` tone.

## Interaction Contract

| Element | Behavior |
|---------|----------|
| LocalQueue-unreachable alert | Renders **empty** (no visible body) when reachable — loud only on failure, exactly like `inadmissible_card.html` (healthy = silent). Lives OUTSIDE `#pipeline-stats`; re-pushed as `hx-swap-oob="true"` fragment on the existing 5s poll. Stable `<section id>` on both first-load and OOB renders. Degrade-safe read surfaces as "reachable" (silent) and NEVER raises into the poll. `role="alert"` + `aria-labelledby`. |
| Ephemeral k8s-lane note | Fully static. Rendered once on page load. No `hx-trigger`, no Alpine state, no localStorage. Does not participate in the Agents table 5s self-refresh. |
| Status pills (k8s lane) | None rendered — the k8s lane has no `Agent` row, so `classify()` is never called for it and no `dead`/`never` pill can appear. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — this phase adds no buttons or interactive controls |
| LocalQueue alert heading | `⚠ K8s LocalQueue unreachable` |
| LocalQueue alert body | `K8s LocalQueue unreachable — verify PHAZE_KUBE_LOCAL_QUEUE / cluster connectivity.` |
| Alert "healthy" state | Renders empty (no text) when the LocalQueue is reachable — silent by design |
| Ephemeral-note heading | (optional inline) `K8s burst lane` |
| Ephemeral-note body | `The Kubernetes burst lane runs as ephemeral, per-file Jobs — it does not register as a heartbeating agent here. Its live activity is visible as in-flight Kueue workloads on the pipeline dashboard.` |
| Empty state | Reuses existing `agents_table.html` "No agents registered yet" copy unchanged — not modified by this phase |
| Error state | The LocalQueue alert above is the only error surface; body names the exact env var + remediation per CONTEXT D-05 |
| Destructive confirmation | None — no destructive actions in this phase |

Copy rules honored: alert states the **problem + what to check** (env var + connectivity);
the note explains **where to look instead** (pipeline dashboard workloads) rather than leaving
the operator wondering why no k8s agent appears. All strings are static and pass through Jinja
autoescape — no operator free-text or PII is interpolated (mirrors the T-54-11 invariant).

| Requirement | Spec |
|-------------|------|
| Alert semantics | `role="alert"` + `aria-labelledby` on the LocalQueue card (matches `inadmissible_card.html`) |
| Note semantics | Plain prose in a `<p>`/`<section>`; if an `ℹ` glyph is used it is decorative (`aria-hidden="true"`) |
| Color independence | Amber alert pairs the hue with the `⚠` glyph + explicit text; never color-only |
| Contrast | Inherited amber-on-dark / gray-on-dark pairings already meet the project's WCAG-AA baseline |
