<objective>
Deterministic per-file bucket staging (MKUE-02, D-06): lift the module-global `active_bucket` read
out of `s3_staging` onto an explicit per-call `bucket: BucketConfig` parameter, make
`KueueBackend.dispatch` pick the bucket via `pick_bucket` and RECORD it on `cloud_job.staging_bucket`,
and make every downstream presign-GET / inline-delete / complete / abort READ that recorded bucket
(never re-derive). This is a mechanical "lift a global to a parameter" refactor — the private
`_client(bucket)` factory already takes a `BucketConfig`; only the public verbs still resolve it
globally. It must land atomically with ALL callers so the suite stays green.

Purpose: each Kueue cluster stages to a bucket from its REG-05 set; multi-bucket sets select one
deterministically per file; the control plane stays the sole presigner (DIST-01).
Output: bucket-parameterized `s3_staging`, `staging_bucket` recorded at dispatch, all call sites
reading the recorded bucket, `active_bucket` retired.

NOTE (D-05, deferred): do NOT touch the compute `agent_ref` heuristic — out of scope (PROV-01).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| control plane → S3 (per bucket) | every presign/upload/delete is minted control-side from a per-bucket `BucketConfig`; creds are `SecretStr`, control-plane-only |
| pod/agent → S3 | pods/agents receive only presigned, file_id-scoped, TTL-bounded URLs; never bucket creds (DIST-01) |
| operator config → bucket endpoint | `BucketConfig.endpoint_url` is operator-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-02 | Information Disclosure | staged object on an Internet-reachable ("public") bucket | mitigate | objects private; access ONLY via short-TTL presigned URLs minted control-side (`cfg.s3_presign_get_ttl_sec`); pods credential-free; DIST-01 sole-presigner preserved |
| T-70-02-02 | Tampering | presign/delete choosing the wrong bucket | mitigate | callers READ the recorded `cloud_job.staging_bucket` and resolve its `BucketConfig`; NEVER re-derive via pick_bucket (config-set drift would mis-point a re-derive) |
| T-70-02-03 | Tampering | SSRF via a malicious `endpoint_url` | mitigate | per-bucket http(s)+host validator from Phase 67 reused unchanged; no free-text enters keys/paths (file_id is a server UUID) |
| T-70-02-04 | Information Disclosure | bucket creds in logs (N buckets) | mitigate | creds are `SecretStr`; s3_staging stays ORM-free and logs no BucketConfig; secret hygiene (T-68-04) preserved |
| T-70-02-05 | Tampering | ORM import creeping into s3_staging | mitigate | router/caller resolves the BucketConfig; import-boundary/purity test keeps s3_staging model-free |
</threat_model>
