<objective>
Per-cluster failure isolation (MKUE-03, D-07): a single flaky Kueue cluster whose `is_available()` /
`in_flight_count()` probe or `dispatch()` call raises or times out must NOT abort the whole `*/5` drain
tick. Wrap each backend's per-tick snapshot in its own try/except (a raise → unavailable, 0 slots,
logged) and widen the dispatch guard so a generic kube/S3 raise is a clean hold of that candidate.
This extends Phase 68's "`is_available` never raises" discipline to the N-cluster snapshot loop
(Pitfall 8) — the reconcile path is already per-row guarded from Phase 69.

Purpose: one down cluster degrades to 0 slots for the tick; every healthy cluster and local still get work.
Output: a hardened `stage_cloud_window` snapshot + dispatch loop, with an N≥2-backend isolation test.
</objective>
