### Change 2: Lifespan extensions

Update the lifespan async context manager to wire two new resources and close them on shutdown. The new lifespan body:

```python
@asynccontextmanager
async def lifespan(_app: FastAPI) -> AsyncGenerator[None]:
    """Manage application lifespan: verify DB, create SAQ queue + task_router + redis on startup; dispose on shutdown."""
    async with engine.begin() as conn:
        await conn.execute(text("SELECT 1"))
    # SAQ default-queue (existing) -- used by routers/scan.py
    _app.state.queue = Queue.from_url(settings.redis_url)
    # AgentTaskRouter -- per-agent queue enqueuer (Plan 04, D-20)
    _app.state.task_router = AgentTaskRouter(redis_url=settings.redis_url)
    # Redis client for tracklists idempotency cache (Plan 07, D-27).
    # decode_responses=True matches the project's redis_client.py convention.
    _app.state.redis = redis_async.Redis.from_url(settings.redis_url, decode_responses=True)
    yield
    # Shutdown in reverse order
    await _app.state.task_router.close()
    await _app.state.redis.aclose()
    await _app.state.queue.disconnect()
    await engine.dispose()
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FastAPI lifespan → AgentTaskRouter + Redis | Both resources are constructed once at startup with the operator-configured `settings.redis_url`. No untrusted input to the constructors. |
| `request.app.state.task_router` → agent_files.py | Same trust as lifespan. The handler doesn't re-validate state; if `app.state.task_router` is missing, the handler raises AttributeError and FastAPI returns 500 — operator-visible. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-12-D | Denial of Service | Lifespan shutdown leaks Redis connections if `aclose()` or `task_router.close()` raise | mitigate | Wrap each shutdown call in try/except IF the project convention is to log + continue. Current convention (from main.py) is to let them raise — acceptable because lifespan shutdown errors don't affect already-served requests; container restart is the recovery path. |
| T-26-12-E | Elevation | A future include_router accidentally exposes an internal-agent endpoint without `Depends(get_authenticated_agent)` | mitigate | Code review at PR catches this. Plans 05-08 each have tests that assert 401 on missing auth — those tests fail if the dep is omitted. |
| T-26-12-I | Information Disclosure | OpenAPI docs at `/docs` expose the agent endpoints without auth (they're listed because they're registered) | accept | All 10 endpoints carry the `bearerAuth` securityScheme via Phase 25's HTTPBearer scheme — Swagger UI renders the lock icon. Listing endpoints without authentication is not disclosure of secrets, only of the API shape. |
| T-26-12-T | Tampering | `app.state.task_router` is accessible from any router handler — a future router might pass an untrusted agent_id | mitigate | This is a code review concern. Plan 04's threat model documents the rule: every caller MUST derive agent_id from the auth dep or a trusted DB column, never from request body. Schema `extra="forbid"` ensures agent_id can't be in the body. |
</threat_model>
