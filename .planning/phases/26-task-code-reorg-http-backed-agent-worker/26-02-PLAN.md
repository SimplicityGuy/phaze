<objective>
Build `PhazeAgentClient` — the single httpx wrapper every file-bound SAQ task on the agent uses to POST/PUT/PATCH state changes back to the application server. The client funnels every call through a tenacity retry loop that retries 5xx and transient network errors three times with exponential-jitter backoff but **never retries 4xx** (auth or validation errors must surface immediately).

Purpose: Without this, no file-bound task can run on an agent. The 4xx/5xx split is the most security/operability-sensitive bit of code in the phase: a 401 retried three times floods auth logs and delays visible failure; a 5xx not retried turns every transient blip into a permanent job failure. The predicate at the retry funnel is the single source of truth for this contract — every endpoint method goes through `_request`, so the policy applies uniformly.

Output: `src/phaze/services/agent_client.py` (single class + 4 exception types + retry predicate + 10 endpoint methods) and `tests/test_services/test_agent_client.py` (respx-mocked contract tests). Plan 03's new schemas land in parallel; this plan emits forward-reference type annotations via `TYPE_CHECKING` so the file compiles regardless of merge order.

Implements decisions: D-09 (DiscogsographyClient pattern), D-10 (one method per endpoint, Pydantic in/out), D-11 (tenacity funnel, 0.5/1/2/4s backoff, 4xx no-retry, 5xx retry), D-12 (4-class error hierarchy), D-13 (DEBUG on success, WARNING on failure, token NEVER logged), D-31 (respx contract tests), D-32 (call_count assertions), D-33 (mypy strict opt-in).

  ```python
  try:
      async for attempt in AsyncRetrying(
          stop=stop_after_attempt(3),
          wait=wait_exponential_jitter(initial=0.5, max=4.0),
          retry=retry_if_exception(_should_retry),
          reraise=True,
      ):
          with attempt:
              response = await self._client.request(method, path, **kwargs)
              response.raise_for_status()
              logger.debug("agent_api method=%s path=%s status=%d", method, path, response.status_code)
              return response
  except httpx.HTTPStatusError as e:
      status_code = e.response.status_code
      logger.warning("agent_api method=%s path=%s status=%d error=HTTPStatusError", method, path, status_code)
      if status_code in (401, 403):
          raise AgentApiAuthError(f"{method} {path} -> {status_code}") from e
      if 400 <= status_code < 500:
          raise AgentApiClientError(f"{method} {path} -> {status_code}: {e.response.text}") from e
      raise AgentApiServerError(f"{method} {path} -> {status_code} after retries") from e
  except (httpx.ConnectError, httpx.ReadTimeout, httpx.WriteTimeout) as e:
      logger.warning("agent_api method=%s path=%s error=%s", method, path, type(e).__name__)
      raise AgentApiServerError(f"{method} {path} network failure after retries") from e
  raise AssertionError("AsyncRetrying loop exited without returning or raising")
  ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent SAQ worker → application server HTTP | Every request crosses an HTTPS boundary in production (Phase 29 hardens TLS). In Phase 26 the boundary is in-cluster HTTP; the bearer token is the only auth surface. The retry funnel must not amplify auth failures into log floods. |
| PhazeAgentClient `_client` injection (test only) | The `_client` constructor param exists for respx mocking. Production code does not pass `_client`; an accidental injection in production would skip the Authorization header. CODE-REVIEW: any non-test caller of `PhazeAgentClient(..., _client=...)` is a flag for review. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-02-S | Spoofing | An attacker who can mint requests to the agent (e.g., a compromised SAQ job payload) could craft a path that misroutes the request to a different endpoint | mitigate | The 10 endpoint methods accept structured params (UUID, model) not raw paths. f-string interpolation uses UUID which has bounded format (36 chars hex+dashes) — no path-traversal vector. Pydantic validation on payload models enforces field shape at the agent boundary before the request is sent. |
| T-26-02-T | Tampering | Retry logic retries a 401 three times, flooding auth-failure logs and giving an attacker observable timing | mitigate | `_should_retry` returns False for 4xx (verified by `call_count == 1` test). Single auth failure -> single 401 log entry -> single AgentApiAuthError. No timing oracle. |
| T-26-02-I | Information Disclosure | Bearer token leaks via httpx redirect follow (server 302s to attacker-controlled URL) | mitigate | httpx.AsyncClient default is `follow_redirects=False` — confirmed by NOT setting that kwarg in `__init__`.
