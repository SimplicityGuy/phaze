# Phase 71 — UI Design Contract

> Visual and interaction contract for **BEUI-01 (N-lane grid)** and **BEUI-02 (master force-local toggle)**.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is an EVOLUTION of the v7.0 "Evolved phaze" (C3) admin UI — not a reskin.** Stack is locked
> (FastAPI + Jinja2 + HTMX 2 + Tailwind 4 self-hosted + Alpine 3). No new design system, no SPA, no new
> dependency. The card unit, tokens, poll discipline, and two-weight type contract are **preserved**; this
> phase generalizes a fixed 3-card grid to N registry-derived lanes and adds one global incident control.
> **BEUI-03 (docs) has no visual surface** — out of scope here (see §Docs note).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (no shadcn — server-rendered Jinja partials; registry safety not applicable) |
| Preset | not applicable |
| Component library | none (inline SVG glyphs + emoji lane glyphs; Alpine for state, HTMX for swaps) |
| Icon library | none — emoji lane glyphs (🖥️ / ☁️ / ⎈) + hand-authored inline `<svg>` (matches header.html) |
| Font (headings/labels) | **Jura 500** (`font-jura font-medium`) — uppercase, `tracking-wider` |
| Font (body) | **Inter 400** (default) |
| Font (numerals) | **Inter 500** (`font-mono font-medium`) — the load-bearing capacity numeral; **NOT** `font-semibold` |

**Two-weight contract (C3, preserve exactly):** headings/labels = Jura 500; body = Inter 400; capacity
numerals = Inter 500 (`font-medium`, never 600). Do not introduce a third UI weight. `base.html` loads
Jura 300/500 + Inter 400/600 for brand chrome; the workspace UI stays inside the Jura-500 / Inter-400 pair
plus the Inter-500 numeral.

## Color

C3 dark theme is dominant (`.dark`), with the light theme reachable via the existing header toggle. Values
are the repo `@theme` tokens (`assets/src/app.css`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `phaze-bg` `#0a0c12` (dark) / `white` (light) | page/workspace background |
| Secondary (30%) | `phaze-panel` `#10141c` (dark) / `white` + `gray-50` (light); borders `phaze-border` `#232832` | **lane cards**, header strip, global roll-up cards |
| Accent (10%) | `blue-500` `#00b0d8` | see reserved-for list below |
| Incident/Warning | `amber-500` family | **forced-local toggle engaged state**; offline lane word; Kueue quota-wait |
| Alert/Fault | `rose-*` / `amber-*` `role="alert"` | Kueue **Inadmissible**; localqueue-unreachable; failed rows |

**Accent (blue #00b0d8) reserved for:** focus rings (`focus-visible:ring-blue-500` — every interactive
control), the wave logo mark, the ⌘K affordance, and the **compute/A1 lane identity** (numeral + bar).
Accent is NOT used for the force-local toggle (that is amber — it is an incident/warning state, not a
brand-accent state), NOT for hover fills, NOT for generic borders.

**Lane identity color is derived from backend `kind` (generalized to N lanes):**

| Backend kind | Numeral/glyph color | Bar fill | Glyph |
|--------------|---------------------|----------|-------|
| `local` | `emerald-600 / dark:emerald-300` | `bg-emerald-400` | 🖥️ |
| `compute` (A1 / cloud-compute, 1..N) | `blue-600 / dark:blue-300` | `bg-blue-400` | ☁️ |
| `kueue` (1..N clusters) | `amber-600 / dark:amber-300` | `bg-amber-400` | ⎈ |

All `compute` lanes share blue and all `kueue` lanes share amber; they are disambiguated by the **`RANK {n}`
caption + the `{KIND · id}` title**, not by hue (WCAG 1.4.1 — never hue-only). Kueue cards keep the
`border-amber-500/30` accent border from Phase 58.

## Copywriting Contract

Tone matches existing UI-SPEC conventions: **uppercase Jura for action/status labels** (like `ROUTE RULES` /
`PAUSE` in `analyze_workspace.html`), **sentence-case Inter gray** for descriptive/empty/error copy.

| Element | Copy |
|---------|------|
| Master toggle — normal state | `CLOUD ROUTING` (aria-checked=true; neutral pill; multi-backend routing active) |
| Master toggle — engaged state | `FORCED LOCAL` (aria-checked=false; amber incident pill; all routing pinned to local) |
| Master toggle — engage confirmation (OOB status/toast) | "Routing forced to LOCAL — cloud & Kueue backends bypassed." |
| Master toggle — revert confirmation (OOB status/toast) | "Cloud routing restored — backends dispatch by rank." |
| Lane card — capacity numeral | `{in_flight}/{cap}` (e.g. `2/4`) |
| Lane card — rank caption | `RANK {n}` (uppercase Jura micro-label) |
| Lane card — available sub-label | kind descriptor, e.g. `short sets < 90 min` (local) · `long sets ≥ 90 min` (compute) · `Kueue quota wait · admitting` (kueue) |
| Lane card — offline word | `offline` (amber; replaces the sub-label; word + greyed glyph, never hue-only) |
| Lane card — Kueue admission caption | `{quota_wait} waiting · {inadmissible} inadmissible` (the `inadmissible` segment goes amber + `role="alert"` when > 0) |
| Lane grid — empty/degraded heading | `Lane status unavailable` |
| Lane grid — empty/degraded body | "Backend lane status could not be read this cycle. It refreshes on the next update." |
| Error state (per-lane probe timeout/raise) | render that ONE lane `offline` for this poll; never a page error, never 500 (D-02 isolation) |

**Destructive actions:** none. The force-local toggle is **reversible** (reverting to local is the *safe*
direction — no work is lost, files simply route/stay local), so per the R-4 copy rule it takes **no
confirmation modal** — instant-on both directions, with an OOB status line as the acknowledgement (see
§Master Toggle Interaction Contract).

## BEUI-01 — N-Lane Grid Contract

Generalizes Phase 58's fixed `grid grid-cols-3` (local/A1/k8s, included 3× by hand) into a `{% for %}`-looped
grid over the D-01 `get_backend_lane_snapshot()` list. **`_lane_card.html` stays the per-card unit** (extended,
not replaced); the redesign owns the grid container, the ordering, and three new per-card data points.

### Grid layout (D-05)

- **Replace** `grid grid-cols-3` on `#analyze-lanes` with a responsive **wrapping** grid.
- **Locked recommendation (deterministic, WCAG-reflow-safe, avoids single-card full-bleed stretch):**
  `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4 p-6`.
  - N=1 → occupies one 1/3 (lg) track, keeping card rhythm (no full-width stretch).
  - N=3 → identical to the Phase-58 layout on `lg` (visual parity for the common case).
  - N=6+ → wraps to a second row automatically; `xl:grid-cols-4` limits row width on wide screens.
- **Acceptable alternative (planner discretion, D-05):** true auto-fill —
  `[grid-template-columns:repeat(auto-fill,minmax(240px,1fr))] gap-4 p-6`. If chosen, cap single-card stretch
  with `minmax(240px, 360px)` or `justify-start`. Prefer the fixed-breakpoint form above unless N is expected
  to be large/highly variable.
- **Card box model is unchanged** (`p-4 rounded-xl h-1.5` bar) — only the container class changes.

### Card ordering (D-06)

- Cards render **rank ascending** (lowest rank = most-preferred/cheapest first; local rank-99 lands last),
  **tie-broken by `id`**. Row-major reading order therefore = the scheduler's own dispatch preference, so the
  grid reads top-left→bottom-right as "what gets used first." **Ordering is server-side** (the snapshot list is
  pre-sorted); the template loops it verbatim.

### New per-card data (vs the Phase-58 card)

Three data points are new relative to `_lane_card.html`; extend the card contract to render them:

1. **Rank** — `RANK {n}` micro-label (Jura 500, `text-[10px]` uppercase `tracking-wider`,
   `text-gray-400 dark:text-gray-500`), placed **inline after the lane title** in the title row (secondary,
   muted — must not compete with the title or the numeral).
2. **In-flight / cap** — the capacity numeral becomes **`{in_flight}/{cap}`** (Inter 500 mono, lane color).
   The capacity bar fill % = `in_flight / cap` (clamped 0–100). This replaces the Phase-58 per-lane bespoke
   numerals (`N running` / `N online` / `N pending`), which are superseded by the uniform in-flight/cap
   contract (BEUI-01).
3. **Per-lane admission attribution (D-03)** — for `kueue` lanes, a caption below the bar:
   `{quota_wait} waiting · {inadmissible} inadmissible`. The `inadmissible` segment renders amber +
   `role="alert"` (word-labelled, never hue-only) when > 0, preserving the **quota-wait-vs-Inadmissible**
   distinction **per lane** (the explicit BEUI-01 requirement). `local`/`compute` lanes show their kind
   sub-label instead.

### Lane states (word + glyph always — WCAG 1.4.1)

| State | Treatment |
|-------|-----------|
| `available` | full lane-identity color, numeral `{in_flight}/{cap}`, bar filled to utilization |
| `offline` | `opacity-60`, numeral `0` in grey, **explicit word `offline`** (amber sub-label), greyed glyph. Triggered by a failed/timed-out `is_available()` probe **for that poll only** (D-02) — isolated, never stalls the grid |

**"not configured" is retired for the N-lane grid** — every rendered lane is registry-derived (it exists
because the operator declared it), so the Phase-58 `not configured` path no longer applies. Keep only the
`available` / `offline` word contract. (If a future declared-but-structurally-incomplete lane must be shown,
reuse the greyed `offline`-style treatment with its own word.)

### Empty / degrade-safe state (D-01)

- The snapshot is degrade-safe (`[]` on any DB error, matching `_safe_count`). When `lanes == []`, the grid
  renders a **single full-width muted panel** (same `rounded-xl border phaze-border p-4` shell) with the
  empty-state heading + body copy above — **not** a page error, **not** a 500, and it **must not collapse the
  layout**. It self-heals on the next poll when the read succeeds.
- (Note: REG-04 fails fast on a truly empty registry, so the steady-state `[]` is the transient DB-read-error
  case, not "no backends declared.")

### Poll & swap mechanism (D-04, WORK-05/R-2 single-poll discipline)

- The lane list is seeded **identically** in `build_dashboard_context()` (full-page) AND
  `pipeline_stats_partial()` (the 5s OOB re-push) — the same seeded-identically discipline the cloud counts
  already follow. **No second poll loop, no new read endpoint, no `hx-trigger` on the card.**
- **Swap mechanism = OOB swap of the whole `#analyze-lanes` grid** (server-rendered loop), NOT N dynamic
  Alpine `$store.pipeline` keys. Rationale (Pitfall 2): N is dynamic and every datum (rank/in-flight/cap/
  admission/online) changes per poll; k8s counts already arrived as OOB card swaps rather than store keys, so
  swapping the grid as a unit is the consistent, N-safe mechanism and avoids inventing per-lane store keys.
  `_lane_card.html` stays pure presentation (no timer, no `setInterval`).

### The 6 global cloud-state cards (D-07 — Claude's discretion, RECOMMENDATION)

**Recommendation: KEEP the six existing global cards as a cross-lane roll-up BELOW the lane grid, unchanged
styling and unchanged OOB-swap ids** (admission / inadmissible / localqueue / awaiting / pushing / analyzing).
Lowest risk: no restyle, the WORK-03 `role="alert"` cards + their OOB swap targets stay byte-stable, and the
operator keeps the cross-lane total at a glance while per-lane detail now lives on the cards (D-03). Folding
them entirely into the lanes and deleting the globals is the higher-risk alternative and is **not** the
default. **This is discretionary — the planner/executor may fold, but the roll-up is the sanctioned baseline.**

## BEUI-02 — Master Force-Local Toggle Interaction Contract

A **live, one-click, reversible** control that pins all analysis routing to local for incident response — the
`backends`-era equivalent of the old `cloud_target=local` no-op gate (D-08). Persisted in a DB control row
(D-09, mirrors the per-stage pause/priority control table), read by the router the same way `paused` is read.

### Visual form & placement (D-10)

- **Control type: a stateful pill toggle** (a `<button>` with `role="switch"` + `aria-checked`), matching the
  header strip's existing pill controls (the Agents pill: `h-8`, `rounded-lg`, `px-3`, Jura `tracking-wider`).
  Chosen over a bare button because the control must **advertise the current routing mode at a glance from any
  page**, and over a raw checkbox/switch-track because the header vocabulary is pills.
- **Placement:** inside the header `ml-auto` status-strip cluster (`shell/partials/header.html`), **immediately
  left of the Agents pill**, so the global incident control sits with the other global status indicators and is
  one click from anywhere (its scope is global/cross-stage). It rides the existing strip — **no new timer/poll**.

### The two states (loud when engaged)

| State | `aria-checked` | Visual | Label |
|-------|----------------|--------|-------|
| **Normal** (multi-backend routing) | `true` | neutral pill: `border-gray-200 dark:border-phaze-border`, `text-gray-700 dark:text-gray-300`, small blue/emerald dot | `CLOUD ROUTING` |
| **Engaged** (forced local — incident) | `false` | **loud amber incident pill**: `bg-amber-500/15`, `border-amber-500/40`, `text-amber-700 dark:text-amber-300`, warning glyph (inline `<svg>` triangle) | `FORCED LOCAL` |

The engaged state is deliberately **prominent** (amber fill + warning glyph + changed label) because it is an
active incident state — an operator glancing at any page must immediately see that cloud/Kueue routing is
bypassed. Do not render engaged as a quiet grey "off."

### Confirm-vs-instant (D-10, R-4)

- **Instant-on, both directions, no blocking modal.** Reverting to local is the safe direction (no work lost —
  files route/stay local), and un-forcing is equally safe, so per the R-4 reversible-no-confirm rule neither
  transition gets a confirmation dialog.
- The toggle **`hx-post`s to a new thin write endpoint** (the sanctioned Review-&-Apply-era thin-endpoint
  pattern) on click; the server flips the DB control row and returns the new pill state (swapped in place) plus
  an **OOB status line/toast** carrying the engage/revert confirmation copy above. The persistent pill itself
  is the durable at-a-glance indicator; the toast is the transient acknowledgement.
- No optimistic client mutation — the authoritative pill state comes from the server response (mirrors the
  Phase-38 pause/resume `@htmx:after-request` authoritative-write pattern), so a failed write never leaves the
  pill lying about routing state.

### Cross-surface signal (RECOMMENDATION — discretionary)

When forced-local is engaged, the non-local lane cards keep rendering their live probe/in-flight state but do
NOT receive new dispatch. **Recommended (discretionary):** append a muted `bypassed` word to the sub-label of
`compute`/`kueue` cards while engaged, so an operator understands why cloud lanes show no new in-flight growth.
This is a nicety, not a requirement — the header pill is the authoritative signal. Mark as planner discretion.

### Accessibility

- `role="switch"`, `aria-checked` reflects engaged/normal, `aria-label="Force all analysis routing to local
  (incident revert)"`.
- Keyboard-operable (native `<button>` — Enter/Space), `focus-visible:ring-2 focus-visible:ring-blue-500` to
  match every other header control. The engaged amber pill meets AA contrast against the header
  `phaze-panel` background (verify `amber-300` on `#10141c` at checker time).
- Parity with the v7.0 a11y baseline: word-labelled state (never color-only), visible focus, live-region-safe
  toast (the OOB status uses a polite `aria-live` region, not an alert that steals focus).
