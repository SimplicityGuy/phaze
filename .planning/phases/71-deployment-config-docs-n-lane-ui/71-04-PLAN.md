<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /pipeline/routing/force-local | untrusted form input crosses into a state-changing write |
| internal reverse-proxy realm → endpoint | same internal-realm trust as all /pipeline/* controls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-07 | Tampering (input) | force-local POST body | mitigate | `engage: Annotated[bool, Form()]` boolean coercion; no free-text; DB default false (V5, mirrors pipeline_stage_control discipline) |
| T-71-09 | Denial of service (stall lever) | force-local toggle | accept | the toggle only reverts to LOCAL — the safe direction, no work lost, reversible; behind the internal realm; no new attack surface beyond the existing pause control |
| T-71-05 | Tampering / XSS | pill/toast render | mitigate | Jinja autoescape; if a value hits an Alpine JS context use `|tojson`; toast is polite aria-live, not an alert that steals focus |
| T-71-10 | Spoofing / state lie | optimistic client mutation | mitigate | no optimistic mutation — authoritative pill state comes from the write response (Phase-38 @htmx:after-request pattern), so a failed write never leaves the pill lying about routing |
| T-71-SC | Tampering | package installs | accept | no packages installed this phase |
</threat_model>
