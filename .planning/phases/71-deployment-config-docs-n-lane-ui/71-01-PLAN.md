|----------|-------------|
| remote Kueue cluster → control-plane | `KueueBackend.is_available` makes a live kr8s call; a hung/slow cluster is untrusted latency crossing into the shared 5s poll |
| DB row values → rendered/logged output | `cloud_job` / registry values flow into the snapshot dicts and logs |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-71-01 | Information disclosure | lane snapshot dicts + probe error logs | mitigate | emit ONLY `{id,kind,rank,cap,in_flight,available,quota_wait,inadmissible}`; probe failure logs `backend_id` only — never `config`/`SecretStr`/kube token/S3 key (SP-5, T-68-04 precedent) |
| T-71-02 | Denial of service | `_probe_availability` fan-out over a hung Kueue probe | mitigate | per-probe `asyncio.wait_for(_PROBE_TIMEOUT_SEC)` + try/except → offline; whole fan-out bounded to ~one timeout, can never stall the shared poll |
