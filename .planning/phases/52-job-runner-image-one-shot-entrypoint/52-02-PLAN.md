<objective>
Build the single genuinely-new module — `src/phaze/job_runner.py` — the fire-once orchestrator
that executes the v6.0 burst flow for one file and translates each step's outcome into a distinct
process exit code, then `sys.exit(code)`. This is the structural divergence from v5.0
`process_file` (which reports failure via callback and returns a dict so SAQ marks the job
COMPLETE): the one-shot must instead exit non-zero so Kueue/Workload reads the failure from pod
status (D-01 / KJOB-04).

It reuses every solved capability — `configure_logging` (D-03), `construct_agent_client` with a
baked CA (KJOB-05), `compute_sha256` (KJOB-02 integrity), `analyze_file` windowed/streaming
(KJOB-03), the relocated `analysis_wire` converters, `request_download_url` + `put_analysis`
(Plan 01) — and adds only the process model: import-boundary discipline, the exit-code mapping,
and the `__main__` guard.

Purpose: deliver the execution-unit logic, fully unit-tested against a fake control plane
(respx) with NO live cluster or bucket.
Output: `job_runner.py`, `tests/test_job_runner.py`, fixtures in `conftest.py`, and the
import-boundary case appended to `tests/test_task_split.py`.
</objective>

|----------|-------------|
| Object store (presigned GET) → Job pod | Untrusted downloaded bytes cross here; verified by sha256 before analysis |

| Job pod process → host/Postgres | Pod is Postgres-less; the entrypoint must not import phaze.database |
