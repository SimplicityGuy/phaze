<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → Codecov | CODECOV_TOKEN crosses to the upload action |
| third-party actions → CI | every `uses:` is attacker-influenceable if unpinned |
| PR trigger → secrets | fork PRs must never receive repo secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-03-01 | Info Disclosure | CODECOV_TOKEN placement | mitigate | Token lives ONLY in the combine job env; zero occurrences in any matrix leg (asserted in Task 2 verify) |
| T-63-03-02 | Tampering | third-party actions | mitigate | All `uses:` reuse the existing frozen 40-char SHA pins; actionlint + check-jsonschema hooks enforce |
| T-63-03-03 | Elevation of Privilege | PR trigger model | mitigate | ci.yml keeps `pull_request` (NOT `pull_request_target`); this plan does not touch triggers, so fork PRs still run without secret exposure |
| T-63-03-04 | Info Disclosure | `.coverage.<bucket>` artifacts | accept | Coverage data contains source file paths + line hit-counts only — no secrets/PII; artifacts are internal to the run |
| T-63-03-SC | Tampering | package installs | accept | No new packages here; pytest-xdist gated in Plan 01 |
</threat_model>
