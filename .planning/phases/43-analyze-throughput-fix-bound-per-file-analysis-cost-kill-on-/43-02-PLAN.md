<objective>
Bound per-file essentia cost with a cap + even stride: when a file's natural window count exceeds the cap (60 fine / 30 coarse), stride evenly across the WHOLE file instead of analyzing every window. This makes analysis cost constant regardless of set length (root cause: cost is O(duration); 3h+ DJ sets blow the 4h timeout). Emit coverage so a sampled file can be re-deepened later (Phase 44).

Purpose: Constant-cost analysis that still spans the entire file; sparklines/aggregates stay valid because no algorithm needs contiguous windows (debug Cost model + RESEARCH §Q2).
Output: pure `_stride_to_cap` downsampler preserving original `window_index`; cap-aware fine/coarse iterators; `analyze_file` returns five coverage fields consumed by Plan 03 (columns) and Plan 04 (forwarding).
</objective>
