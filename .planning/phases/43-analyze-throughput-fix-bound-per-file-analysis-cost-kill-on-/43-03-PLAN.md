<objective>
Persist analysis state and coverage on the control plane. Today `put_analysis` upserts the row but NEVER sets `files.state`, so all 11428 files are stuck `discovered` (incl. the 79 with analysis rows) — which also makes any re-trigger re-enqueue everything. This plan: sets `ANALYZED` on a successful PUT; adds `FileState.ANALYSIS_FAILED` + a worker-callable failure endpoint; persists the five coverage fields into dedicated columns (Alembic rev 021).

Purpose: Make analysis outcomes durable and visible (analyzed / sampled / failed), fix the latent re-enqueue-all bug, and give the Postgres-free worker an HTTP path to report terminal failure + coverage.
Output: migration 021; `AnalysisResult` + `FileState` model changes; extended `AnalysisWritePayload` + new `AnalysisFailurePayload`; `put_analysis` state-advance + coverage columns; new POST `/{file_id}/failed`; `report_analysis_failed` client method.
</objective>
