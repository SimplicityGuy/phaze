<objective>
Replace the un-killable `ProcessPoolExecutor` with a `pebble.ProcessPool` so a runaway essentia child is SIGKILLed and the worker recycled on a hard per-task timeout. This is the gating fix for the analyze-throughput incident: today `concurrent.futures` cannot cancel an already-started child, so a timed-out job leaks compute and starves the 4-slot pool (live: 483% CPU / 28.6 GiB).

Purpose: Make the inner analysis timeout deterministic and reclaim pool slots, independent of SAQ-level cancellation (which cannot reach the OS child).
Output: pebble dependency added; `pool.py` rewritten to own a killable pool with kwargs passthrough; shutdown hook updated; three new `AgentSettings` knobs (inner timeout + the two window caps consumed by Plan 02/04).
</objective>
