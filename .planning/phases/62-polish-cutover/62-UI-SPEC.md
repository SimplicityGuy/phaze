# Phase 62 — UI Design Contract

> Visual and interaction contract for the FINAL v7.0 phase (Polish & cutover). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note (read first):** This is a **presentation-only polish phase**, not a new-feature phase. The visual language is **already locked** by Phases 57–61 (C3 "Evolved phaze": Jura headings + Inter body, blue accent `#00b0d8`, inline wave logo, dark `phaze-bg` primary + light toggle; three-column shell; DAG rail). This contract does **not** introduce a new design language. It locks only the **small net-new surface** this phase adds:
> 1. **CUT-04** — the collapsed narrow-width icon rail (`max-lg:`) + a per-stage **inline-SVG icon set** (D-08, net-new visual work).
> 2. **CUT-01** — concrete, checkable **accessibility affordance contracts** (focus ring, ⌘K accessible name, dialog/combobox roles, skip link, rail landmarks) — the assertions the CUT-01 pytest structural guards enforce.
> 3. **CUT-02** — the ONLY visual change is the residual `base.html` header (logo + theme toggle) after the dead tab-bar nav is removed (D-04a).
> 4. **CUT-03** — docs, no visual surface.

## Spacing Scale

Tailwind default scale (multiples of 4). Values already in use across the shell — do not introduce new tokens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Theme-toggle icon padding, group-heading micro-gaps |
| sm | 8px (`gap-2`, `px-2`, `mx-2`) | Rail group boxes, node label↔icon gap (`gap-2.5`=10px on sub-nodes) |
| md | 12–16px (`px-3`, `py-2.5`, `gap-3`, `p-4`) | Top-level rail node padding, rail `py-4` outer, `+ Scan` gap |
| lg | 24px (`py-6`, `gap-6`) | Workspace/section padding (inherited) |
| xl | 32px (`py-8`) | `base.html` main content vertical padding (kept page) |

**Exceptions (declared):**
- Rail node vertical rhythm uses Tailwind half-steps already in the shell — `py-2.5` (10px) on top-level nodes, `py-1.5` (6px) on sub-nodes, `gap-2.5` (10px). Preserve verbatim; do NOT re-grid.
- **Collapsed rail (CUT-04): the icon button must present a ≥40px hit target.** In the `max-lg:w-16` strip, center each glyph in a `max-lg:justify-center` node whose padding yields ~40×40px (e.g. keep `py-2.5`, swap horizontal padding to `max-lg:px-0`). This is the one CUT-04-specific spacing rule. (Full 44px WCAG touch target is **not** required — touch/tablet is deferred SHELL-06; this is a desktop-primary narrow-width, pointer-driven collapse.)

## Typography

Locked by app.css + shell templates. Exactly the sizes/weights already shipped — no new scale.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Eyebrow / group heading | 10–11px (`text-[10px]`/`text-[11px]`) | 500 | 1.2 | Jura, `uppercase tracking-[0.2em]`/`tracking-[0.25em]` |
| Meta / count | 12px (`text-xs`) | 400 | 1.4 | Inter mono (`font-mono`) for numeric counts |
| Body / label | 14px (`text-sm`) | 400 (label) / 500 (CTA, brand) | 1.5 | Inter |
| Heading (stage title `h2`) | 20px (`text-xl`) | 500 | 1.2 | Jura (`.font-jura`) |

**Weights (2):** Regular **400** (body, labels, counts) + Medium **500** (CTAs, brand mark, eyebrows, headings). No other weights.

**Contract rule:** collapsed-rail labels are **not deleted** — they remain in the DOM as `max-lg:sr-only` (readable by screen readers, visually hidden). Typography of the visible expanded rail is unchanged.

## Color

60/30/10 split, dark theme primary (light toggle inherited, not first-class per deferred RECORD-05). Values from `@theme` in app.css.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-phaze-bg` `#0a0c12` | App background, workspace canvas |
| Secondary (30%) | `--color-phaze-panel` `#10141c` + border `--color-phaze-border` `#232832` | Rail, group boxes, cards, header; light-mode counterparts `bg-gray-50`/`bg-white`/`border-gray-200` |
| Accent (10%) | `--color-blue-500` `#00b0d8` (600 = same hex) | see reserved list below |
| Semantic — group | amber (`amber-500/25` border, `amber-500/[0.06]` fill, `amber-300`/`amber-600` text) | "Review & Apply" rail group ONLY |
| Semantic — status | emerald (`emerald-400`/`emerald-500/15`) done; `blue-400 animate-pulse` active; `gray-400/600` idle | Rail status dots (expanded view) |
| Destructive | `red-600` (inherited) | **Not used in this phase** — no destructive UI action introduced (template deletion is dev-time, not a user action) |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. Active rail node — `aria-[current=page]` tint `bg-blue-500/10` + inset bar `shadow-[inset_3px_0_0_var(--color-blue-500)]`.
2. Focus-visible rings — `focus-visible:ring-blue-500` on all interactive controls (rail nodes, ⌘K, record controls, `+ Scan`, new icon buttons).
3. Primary `+ Scan` CTA — `bg-blue-600 text-white hover:bg-blue-500`.
4. Brand wave-logo mark (`text-blue-500 dark:text-blue-400`).
5. `⌘K` active-option highlight + `aria-activedescendant` target.

Accent must NOT bleed onto the collapsed icon glyphs' default state — glyphs render `currentColor` at the node's normal text color (gray), taking accent ONLY via the active `aria-current` node treatment already listed.

## Net-New Visual Surface (CUT-04) — Collapsed Narrow-Width Rail

### Breakpoint & layout (D-07 — pure CSS, no JS, no persistence)

| Property | Value |
|----------|-------|
| Breakpoint | Tailwind **`max-lg:`** = `@media (width < 64rem)` (**< 1024px**) — verified via Context7, Tailwind v4.3.2 |
| Expanded (≥1024px) | **UNCHANGED** — today's `w-[280px]` dot + label + count layout, exactly as shipped in `rail.html`. |
| Collapsed (<1024px) | Icon-only strip. **Width `max-lg:w-16` (64px)** — chosen collapsed width (Claude's discretion, D-07). |

**Collapsed transform rules (applied per rail node in `rail.html`, class additions only):**
- Aside: `w-[280px]` → add `max-lg:w-16`.
- Node label spans: add `max-lg:sr-only` (screen-reader-readable, visually hidden — **NOT** `max-lg:hidden`, which would strip it from the a11y tree). This is a hard contract, enforced by the CUT-01 guard.
- Numeric count spans (`x-text=...`): add `max-lg:hidden` (visual-only data, safe to drop from the collapsed view).
- Group eyebrows / group-box chrome ("Enrich · parallel", "Identify", "Review & Apply"): add `max-lg:hidden` (or restyle to a thin divider) — labels are redundant when only icons show.
- Node inner layout: add `max-lg:justify-center max-lg:px-0` so the glyph centers in the 64px strip with a ~40px hit target (see Spacing exception).
- Each navigable node gains a native `title="{label}"` (hover/focus tooltip) so the collapsed icon still surfaces its name to sighted pointer/keyboard users; the `sr-only` label + `title` together keep it fully labelled.
- Active-state (`aria-[current=page]` tint + inset bar) and focus-visible ring are **preserved verbatim** in both states.

**Overlays at narrow width (verify, no structural change expected):** the ⌘K palette (`fixed inset-0`, centered) and the record slide-in (`fixed inset-y-4 right-4 w-[760px] max-w-[94vw]`) are viewport-anchored, independent of rail width — they remain usable/legible < 1024px. Confirm visually; no class change anticipated.

### Per-stage icon set (D-08 — inline SVG, no dependency)

The rail today uses **status dots, not glyphs** — an icon-only collapse REQUIRES this net-new set. **No prototype icon set exists** (prototype uses `✓`/`∿`/colored dots), so glyph choice is Claude's discretion.

**SVG wrapper contract (every glyph, verbatim):**
```
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.5"
     stroke-linecap="round" stroke-linejoin="round"
     class="w-5 h-5 shrink-0" aria-hidden="true"> … </svg>
```
- 24×24 viewBox, 1.5px stroke, `currentColor`, `w-5 h-5` (20px rendered) — identical idiom to the theme-toggle icons (`base.html`) and wave logo.
- `aria-hidden="true"` on every glyph — the node's text label (kept as `sr-only` when collapsed) is the accessible name. The glyph is decorative to AT.
- **Source:** copy the outline `<path d="…">` **verbatim from heroicons v2 (MIT-licensed), inlined** — no `@heroicons` package, no icon font. Inlining is consistent with the no-build / no-new-dep ethos.
- In the **expanded** view the glyph sits where the status dot is today (leading, before the label). Keep the existing status dot semantics if desired OR let the glyph carry status via color — executor's choice, but glyph must be present in BOTH states (it is the only visual in collapsed).

**Glyph assignment (one per node — all distinct, all exist in heroicons v2 outline):**

|-----------|----|--------------------------|
| + Scan (CTA) | — | `plus` (accent button; the "+ Scan" text collapses to the plus glyph) |
| Discover | `discover` | `folder-open` |
| Metadata | `metadata` | `document-text` |
| Fingerprint | `fingerprint` | `finger-print` |

The two below-the-line links (`/audit/`, `/admin/agents`) are plain `<a>` (not `/s/` stages, D-04) — they get the same collapse treatment (glyph + `max-lg:sr-only` label + `title`).

## Interaction & Accessibility Contract (CUT-01)

**This is audit-and-close-gaps, NOT an ARIA rebuild (D-01a).** Phase 61 already built the hard ARIA. State everything below as **concrete, checkable contracts** — these become the assertions in the new `tests/test_a11y_guards.py` (pure-filesystem structural guards, mirroring `test_dead_template_guard.py` / `test_base_html_sri.py`; no browser, no new dep, per D-01). Target WCAG 2.1 AA; floor is parity-or-better (D-02).

### Focus-state contract (codify the existing treatment as the standard)
- Every interactive control carries: `focus:outline-none focus-visible:ring-2 focus-visible:ring-blue-500`.
- Where the control sits on a panel surface, add offset: `focus-visible:ring-offset-1 dark:focus-visible:ring-offset-phaze-panel` (as `+ Scan` already does).
- Ring color **`blue-500` `#00b0d8`**, width **2px**, offset **1px** (dark offset against `phaze-panel`).
- Applies uniformly to: rail nodes, below-line links, `+ Scan`, the new collapsed icon buttons, ⌘K input + options, record slide-in controls, theme toggle.
- **Guard:** assert every navigable rail node / interactive control template carries a `focus-visible:` class.

### ⌘K command palette (`cmdk_modal.html` + `palette_results.html`)
Locked contract (confirm existing + close the ONE gap):
- Dialog wrapper: `role="dialog" aria-modal="true" aria-label="Command palette"` — **present, keep.**
- Focus trap: `x-trap.inert.noscroll` (via `@alpinejs/focus`) — **present, keep.**
- Input: `role="combobox"`, `aria-controls="cmdk-results"`, `aria-expanded="true"` (acceptable hardcode for an always-open listbox), `:aria-activedescendant="activeId"` — **present, keep.**
- Results container: `role="listbox" aria-label="Search and command results"` — **present, keep.**
- Result rows: `role="option"` (in `palette_results.html`) — **present, keep.**
- **GAP TO CLOSE (the one real CUT-01 fix):** the combobox `<input x-ref="input">` has **no accessible name** (placeholder ≠ name). **Add `aria-label="Search files and commands"`.**
- **Guard:** assert input has `role="combobox"` + `aria-controls` + `aria-expanded` + `:aria-activedescendant` **and** an accessible name (`aria-label`); listbox + option roles present; dialog wrapper role/modal/label present.

### Record slide-in (`record_host.html` + `record_body.html`)
- Panel: `role="dialog" aria-modal="true" aria-label="File record"` (refined to the filename via `textContent` — XSS-safe, do NOT switch to `|e` in JS context) — **present, keep.**
- Focus trap `x-trap.inert.noscroll`; focus moves to `<h2 tabindex="-1">` on open — **present, keep.**
- **Guard:** assert panel role + `aria-modal="true"` + `aria-label` + presence of `x-trap`.

### DAG rail landmarks + current state (`rail.html`)
- `<aside aria-label="Pipeline navigation">` + `<nav aria-label="Pipeline stages">` — **present, keep.**
- Active node carries `aria-current="page"` (driven by the shell `stage` context var); active visual driven from that attribute via the `aria-[current=page]:` variant — **present, keep.**
- **Guard:** assert `nav` + `aside` landmarks with non-empty `aria-label`; every rail `<button>` uses the `{% if stage == ... %}aria-current="page"{% endif %}` idiom.
- **Collapse-safe guard (CUT-04 ↔ CUT-01 join):** assert rail node label spans use `max-lg:sr-only` and are NOT removed / `max-lg:hidden` — the collapsed strip must stay screen-reader-navigable (D-08).

### Skip link (`shell.html:153`)
- `<a href="#stage-workspace" class="sr-only focus:not-sr-only …">` is the **first focusable element** in `<body>` and targets an existing id in the shell include graph — **present, keep.**
- **Guard:** assert the skip link exists, is first focusable, and its `#stage-workspace` target id exists in the rendered shell.

### CUT-02 residual — `base.html` header (D-04a)
- After removing the dead tab-bar nav block (`base.html:161-251`), the header retains **only**: the **wave-logo home link** (`<a href="/" aria-label="Phaze home">`) + the **theme toggle** (`aria-label="Toggle theme"`). Nothing else. This keeps the KEPT audit/agents pages (which `{% extends base.html %}`) from becoming dead-ends (logo = "back to shell").
- No new affordance is added (the logo home link already serves "return to app" — D-04a rejected a separate back button).
- **Note (verify, not a named CUT-01 surface):** the `base.html` skip link currently targets `#proposals-table` (legacy). If the KEPT pages no longer render that id after cutover, correct the skip-link target to a valid id on those pages (or the page `<main>`); this is a supersession-hygiene follow-through, not a rebuild.
