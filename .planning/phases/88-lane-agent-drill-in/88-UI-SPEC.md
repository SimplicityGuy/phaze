# Phase 88 — UI Design Contract

> Visual and interaction contract for the Lane / Agent Drill-In. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds two clickable, poll-surviving, keyboard-accessible drill-in detail views over the existing
> v7.0 server-rendered admin console. **The stack is LOCKED** (Jinja2 + HTMX 2.0.10 + Alpine.js 3.15.12 +
> Tailwind v4 self-hosted, no build step, no SPA, no charting lib). **Every token below is REUSED from an
> existing partial** — this phase introduces zero new colors, zero new fonts, and zero new component
> primitives. The contract's job is to nail down the *interaction, state, and a11y* behaviour of the new
> triggers and the shared detail pane so the executor implements it without further design decisions.

## Copywriting Contract

This phase is **read-only** — the "CTA" is the drill-in activation + the pane dismiss, not a form submit.

| Element | Copy |
|---------|------|
| Primary action — lane trigger | Activate lane detail. Accessible name: **`aria-label="Open lane {kind} {id} detail"`** (e.g. "Open lane kueue k8s-a detail"). Visible affordance: `cursor-pointer` + hover + focus ring; no added button chrome (the card *is* the button). |
| Primary action — agent trigger | Activate agent activity. Accessible name: **`aria-label="Open agent {name} activity"`**. |
| Pane dismiss (CTA) | Visible **`✕` Close** control in the pane header, `aria-label="Close detail"`. Esc also dismisses (D-09). |
| Empty — nothing selected (resting pane) | Heading: **"No lane selected"** / **"No agent selected"**. Body: **"Select a lane to see its queues, in-flight work, and recent completions."** / **"Select an agent to see its stage activity, queue depths, and liveness."** |
| Empty — lane offline | **"Lane offline — no live queue data on this poll."** (mirrors the card's word `offline`; amber caption, never hidden) |
| Empty — agent owns 0 files | **"This agent owns no files yet."** (shown in place of the stage-matrix counts) |
| Empty — no recent completions | **"No completions in the last {N}."** |
| Empty — no recent scan batches | **"No recent scan batches."** |
| Empty — no agents at all | Reuse the **existing** `agents_table.html` empty state verbatim ("No agents registered yet …") — do not restyle. |
| Error — degraded read (PERF-01 / D-00b) | Per-value fallback renders **`—`** or **`0`** (never blank, never a 500). If a whole section degrades: **"Live data unavailable — retrying on the next tick."** |
| Error — live-tick refresh failed (D-03) | Red `role="alert"` caption, mirroring the agents-table pattern: **"Detail refresh failed — retrying every 5s."** Prior values stay on screen (stale, not blanked). |
| Loading (first fetch) | Reuse the console `.htmx-indicator` opacity convention; brief — no spinner copy required. |
| Destructive confirmation | **None — this phase has no destructive actions.** |

Free-text safety: all `id`/`kind`/`name` values are operator-declared and stay **Jinja-autoescaped** (T-71-05 precedent). All counts/queue depths are server-rendered scalars.

### Detail-pane states
| State | Behaviour |
|-------|-----------|

## Keyboard & A11y Contract (DRILL-03 / D-09)

**HARD requirement.** The pane is **non-modal** (D-01 rejected modal/dialog): it sits side-by-side with the
list, so it must NOT trap focus and must NOT use `aria-modal`/`x-trap` (that would inert the list the
operator is meant to keep glancing at). Contrast `record_host.html`/`cmdk_modal.html`, which ARE modal —
borrow their **Esc + focus-return** discipline, not their focus-trap/backdrop.

**Triggers:**
- `role="button"`, `tabindex="0"`, descriptive `aria-label`, `aria-current="true"` when selected.
- **Enter + Space** both activate. Preferred wiring: htmx-native `hx-trigger="click, keyup[key=='Enter']"` (no per-card Alpine) **plus** an inline `onkeydown` that `preventDefault()`s Space-scroll and re-fires the activation. Space MUST activate and MUST NOT scroll the page.
- Visible `:focus-visible` ring (accent token above).

**Pane:**
- `role="region"`, `aria-labelledby="{pane-header-id}"`, `aria-live="polite"` (so tick updates + open are announced without stealing focus).
- **On open:** move focus to the pane header (`<h2 tabindex="-1">`), matching `record_host.html`'s heading-focus pattern. **This is load-bearing** — it moves focus OFF the trigger, which the 5s poll replaces in the DOM; parking focus on the pane avoids a stranded/lost focus when the trigger node is swapped.
- **Dismiss:** a visible **`✕ Close`** control (`aria-label="Close detail"`) **and** Esc. Esc handler is guarded so it only fires when the pane is open AND no modal (`[aria-modal="true"]` visible) is layered above it (record/cmdk own Esc when open).
- **On dismiss:** clear the `?param` (via `hx-push-url` to the base path on Close, or `history.replaceState`) AND **return focus to the originating trigger by STABLE ID** — `document.getElementById('lane-trigger-'+id)?.focus()` / `'agent-trigger-'+id` — **never a captured element handle** (the original node is gone after any intervening poll swap; the re-rendered trigger keeps the same id).

**WCAG 1.4.1 (never hue-only):** every lane kind, agent kind, and stage bucket carries a WORD + GLYPH (already in the reused partials). Selection = ring + `aria-current`, not hue. Kueue Inadmissible keeps `role="alert"` + the word.

## Poll-Survival Mechanism (DRILL-01/02/03 · D-01/D-02/D-03)

Both DRILL-03 escape hatches are used **together** (D-02), not either/or:

1. **Pane lives OUTSIDE the polled region.** The lane pane sits outside `#analyze-lanes` (host: `analyze_workspace.html`); the agent pane sits outside `#agents-table-section` (host: `agents_table.html`). Physical placement (right column vs below the grid) + narrow-screen collapse is the planner's call (D-01 discretion) — **recommended: right-side column on `lg+`, stacked BELOW the polled list on narrow (`< lg`)** — but it MUST be a distinct swap target the 5s `outerHTML` poll never touches. Target id: `#detail-pane` (`hx-target="#detail-pane" hx-swap="innerHTML"`).
2. **Selection carried in a URL param** (`?lane={backend_id}` / `?agent={agent_id}`) set via `hx-push-url` on the trigger. The polled list endpoints **read the param** to re-apply the selected-highlight after each swap; a **page reload with the param re-opens the detail** (server renders the pane populated when the param is present).
3. **Open pane self-refreshes on its OWN bounded tick** (`hx-trigger="every 5s"` scoped to the selected id), consistent with the console's 5s cadence; the tick is a single bounded `_safe_count`/SAVEPOINT-degrade read (D-00b) and is removed with the pane on dismiss (no orphan loop).

Every derived read (both endpoints + both ticks) is bounded and degrade-safe — **no whole-corpus scan per poll** (PERF-01). The per-agent stage grouping is a single indexed `GROUP BY` aggregate over `stage_status_case`, **never** row-by-row materialization of the agent's files (D-04).
