## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → bulk-approve endpoint | request body/query is UNTRUSTED — must not drive which rows are approved |
| client → edit PATCH | operator-supplied `proposed` value crosses into a persisted Text field later used to build a physical move path |
| DB → response HTML | filenames/paths/tags cross into rendered toast/row markup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60-01 | Tampering / EoP | D-02 bulk-approve + D-03 tag-bulk | mitigate | server RE-QUERIES the fixed predicate at submit (`confidence >= 0.9`; tag no-blank+≥1-change); NO client `proposal_ids`/id-list is read — a stale/forged selection cannot mass-apply (the REVIEW-02 core); SQLAlchemy-parameterized `where`, no raw SQL |
| T-60-02 | Tampering | D-05 edit PATCH `proposed` value | mitigate | reject empty, any `..` segment, NUL/control chars, and (filename facet) `/`; path facet `strip('/')`+collapse `//` mirroring store_proposals; ORM parameter binding (no string interpolation); prevents path-traversal into the later physical move |
| T-60-03 | Repudiation | REVIEW-05 audit integrity | mitigate | integration test asserts exactly ONE TagWriteLog per write + one dedupe resolution per resolve; tag reversal reuses execute_tag_write (before_tags) so the audit trail stays append-only + coherent |
| T-60-XSS | Tampering / Info-disclosure | tags/proposals response partials | mitigate | Jinja2 autoescape; NEVER `\| safe` on any filename/path/tag/toast value |
| T-60-SC | Tampering | package installs | accept | Phase 60 installs nothing (no new pip deps) — no supply-chain surface |
