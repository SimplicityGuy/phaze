    ```python
    NOT a router module -- exports `get_authenticated_agent` as a FastAPI dependency

        `ix_agents_token_hash_active (token_hash) WHERE revoked_at IS NULL`
        (migration 014). Revocation = set `revoked_at = NOW()`; the next request
        misses the predicate and returns 403, NO server restart required (AUTH-04).
    """

    from __future__ import annotations

    import hashlib
    from typing import TYPE_CHECKING, Annotated

    from fastapi import Depends, HTTPException, status
    from fastapi.security import HTTPAuthorizationCredentials, HTTPBearer
    from sqlalchemy import select

    from phaze.database import get_session
    from phaze.models.agent import Agent


    if TYPE_CHECKING:
        from sqlalchemy.ext.asyncio import AsyncSession


    bearer_scheme = HTTPBearer(
        scheme_name="bearerAuth",
        description="Per-agent bearer token. Format: phaze_agent_<32 urlsafe-base64 bytes>.",
    )
    """The HTTPBearer instance that drives OpenAPI security-scheme emission.

    `scheme_name="bearerAuth"` lands the auto-generated `components.securitySchemes.bearerAuth`
    entry on `/openapi.json` (FastAPI introspection). Every route depending on
    `get_authenticated_agent` inherits the lock icon in /docs.
    """


    def hash_token(token: str) -> str:
        """Return `sha256(token).hex()` of the entire wire token (prefix included).

        Per phase-25 D-02: the server NEVER strips the `phaze_agent_` prefix before
        hashing. A future prefix change is therefore a versioning event that
        invalidates all existing tokens -- the right behaviour.
        """
        return hashlib.sha256(token.encode("utf-8")).hexdigest()


    async def get_authenticated_agent(
        credentials: Annotated[HTTPAuthorizationCredentials, Depends(bearer_scheme)],
        session: Annotated[AsyncSession, Depends(get_session)],
    ) -> Agent:
        """Resolve the calling agent from the bearer token.

        Raises:
            HTTPException(403): well-formed bearer whose hash is unknown OR whose
                row has `revoked_at IS NOT NULL`. Both cases are indistinguishable
                to the client by design (phase-25 D-06).

        The missing/malformed-header 401 case is raised by `HTTPBearer` BEFORE this
        function runs; that response includes `WWW-Authenticate: Bearer` per RFC 6750.
        """
        token_hash = hash_token(credentials.credentials)
        # Predicate must be `Agent.revoked_at.is_(None)` so SQLAlchemy renders
        # `revoked_at IS NULL`, matching migration 014's partial index predicate
        # (`ix_agents_token_hash_active ... WHERE revoked_at IS NULL`).
        stmt = select(Agent).where(Agent.token_hash == token_hash, Agent.revoked_at.is_(None))
        agent = (await session.execute(stmt)).scalar_one_or_none()
        if agent is None:
            raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, detail="Forbidden")
        return agent
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API (bearer token presentation) | Untrusted bearer string crosses into the API process. `HTTPBearer` parses the header; `hash_token` reduces the secret to a deterministic hex; the SELECT is the actual authorization check. |
| Auth dep → database (token-hash lookup) | The token-hash is parameterized into a SQLAlchemy `select().where(...)` — no string interpolation, no injection surface. The partial index covers exactly the auth predicate. |
| Server → client (401 vs 403 response) | The choice between 401 and 403 carries information; 403 must be indistinguishable for "unknown token" vs "revoked token" to avoid an enumeration oracle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-02-S | Spoofing | An attacker presents a bearer token whose hash collides with a known agent's hash via brute force on the 256-bit SHA-256 space | accept | SHA-256 hex space is 2^256; uniform-random 32-byte secrets mean the realistic brute-force surface is the secret entropy itself (2^256). This is the same calculus banking/SSO bearer-token systems accept. Mitigation in Phase 29 is HTTPS + LAN binding so the wire token is never network-observable in the first place. |
| T-25-02-I | Information Disclosure | 401-vs-403 status code oracle leaks "does this agent_id exist" by returning 401 only for genuinely-absent bearers | mitigate | Per D-06: 401 ONLY for missing/malformed `Authorization` header; 403 for BOTH "unknown hash" AND "revoked agent" — indistinguishable to the client. Test 3 (`test_unknown_token_returns_403`) asserts the 403 with body `{"detail": "Forbidden"}` (no extra fields that could leak existence). |
| T-25-02-I | Information Disclosure | Logging the bearer token plaintext in an exception or trace exposes it to log-pipeline operators | mitigate | The auth dep does NOT log `credentials.credentials` (the plaintext token) anywhere. The only log signal callers may emit downstream is `agent.id` after a successful resolution. CONTEXT.md "Claude's Discretion" confirms: keep token material out of every log line. This plan does not add any `logger.info(...)` calls. |
| T-25-02-T | Tampering | A future contributor caches the auth result in-process (e.g., LRU-by-hash) defeating immediate revocation (AUTH-04) | mitigate | Single-paragraph docstring on `get_authenticated_agent` explicitly forbids caching ("Per-request verification is a single indexed SELECT ... no in-process cache"). Test 4 (`test_revoke_blocks_next_call`) is the regression guard — adding any cache would cause it to fail. |
| T-25-02-T | Tampering | The auth SELECT uses `== None` instead of `.is_(None)`, rendering `revoked_at = NULL` which is always false; every request would return 403 (DoS), or worse, the predicate gets folded out and revoked agents authenticate | mitigate | E711 ruff rule rejects `== None`. The docstring on `get_authenticated_agent` explicitly cites `.is_(None)` and notes the byte-for-byte match with migration 014's partial-index predicate. Test 4 + test 5 are the active regression guards. |
| T-25-02-D | Denial of Service | An attacker floods the endpoint with bearer tokens, exhausting database connections via the per-request SELECT | accept | Private LAN deployment + bearer-known operators per CONTEXT.md "Deferred Ideas" (rate limiting deferred). Postgres connection pool sized in `database.py` is the operational bound; the index lookup is microsecond-scale so realistic agent fleet (<10) cannot saturate the connection pool. Revisit if multi-tenant arrives (OPS-06). |
