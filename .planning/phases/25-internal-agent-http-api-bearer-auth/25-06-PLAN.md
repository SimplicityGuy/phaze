Recommended approach (chosen): update `schemas/agent_files.py` to read the setting at import time via a module-level constant. SEE ACTION below for the exact pattern.

- It depends on Plans 03/04/05 each producing their `agent_*.router` exports.

<!-- Schema literal swap (Plan 03 deferred to Plan 06) -->
From src/phaze/schemas/agent_files.py (current Plan-03 literal):
```python
files: list[FileUpsertRecord] = Field(min_length=1, max_length=1000)
```
This plan converts the `max_length=1000` literal to use the setting. Pydantic's
`Field(...)` is evaluated at class-definition time, so we read the setting at
module import. Pattern: import `settings`, define a module-level alias, use the
alias in `Field(...)`. The cleanest form:

```python
from phaze.config import settings

_CHUNK_MAX: int = settings.agent_file_chunk_max

class FileUpsertChunk(BaseModel):
    model_config = ConfigDict(extra="forbid")
    files: list[FileUpsertRecord] = Field(min_length=1, max_length=_CHUNK_MAX)
```

Trade-off: changing `agent_file_chunk_max` via env var at runtime requires a
process restart (the schema is bound at import). For a single-user app this is
fine and matches every other settings-driven knob in the codebase.
